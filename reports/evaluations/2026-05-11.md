## Key Technical Decisions

1. **Per-channel quantisation scale**: Changed from per-tensor (single scalar) to per-channel `[1, 2, d_head]` shape. This is the standard approach in modern KV quantisation (KVQuant, FlashInfer). Enables accurate 8-bit quantisation with < 1% attention output error on random N(0,1) data.

2. **int16 storage**: Fixed int8 overflow for 8-bit quantisation (values 0–255 exceed int8 range −128..127). int16 stores all values correctly.

3. **Theoretical compression ratio**: `compression_ratio()` reports `1 − avg_bits/16` based on water-filling bit_allocation. This is the standard information-theoretic metric used by RateQuant and similar work.

4. **CEGAR key alignment**: `compile_corpus` and `get_segments` both use the same SHA-256 hash of chunk token content, ensuring cache hits are correctly attributed.

5. **r_h metadata serialisation**: `save_pipeline()` stores `codec_bit_allocation` and `codec_head_variances` alongside WiCER artefacts, enabling zero-overhead loading (no re-calibration required).
