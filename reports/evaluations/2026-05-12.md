## 참고 사항

1. **저장/복원 순서 계약 검증**: store_segment → load_segment의 full pipeline 계약이 통합 테스트에서 확인됨. value slice error 0.64% (2% 한도), cosine 0.99 이상.

2. **테스트 데이터 설계**: 정확도 테스트는 low-rank structured KV (rank=4/32, signal*5 + noise*0.01)를 사용하여 실 LLM KV 캐시의 저차원 특성을 시뮬레이션함. MixedDim 코덱이 고분산 차원을 보존하는 원리가 정확도 목표 달성의 핵심.

3. **비연속 히트율 측정 방식**: `_noncontiguous_hits` 카운터는 "miss가 있는 더 낮은 chunk_idx 뒤에 오는 hit"를 비연속으로 정의함. 100% 달성은 chunk 0 미스 + chunk 1,2,3 히트 시나리오에서 기인.

4. **RoPE 재인코딩 정확도**: position 0에서 회전 행렬 = identity (cos0=1, sin0=0), 직교성 검증(각 2×2 블록 행 norm=1.0) 통과.

5. **압축 실효 메모리 감소**: budget_ratio는 마스크 비율이며 실제 스토리지 절감은 희소 표현 구현에 의존. 현재 masked_kv는 dtype-수준(float)으로 저장되므로 실제 바이트 감소를 위해서는 sparse 포맷이나 INT8 양자화 추가 필요 — 다음 사이클 제언.

## 다음 사이클 제언

1. **실제 바이트 감소 구현**: 현재 masked_kv(dim=0 zeroing)는 논리적 감소. 실제 메모리 절감을 위해 CSR/CSC sparse 포맷이나 FP8 양자화를 encode dict에 추가하면 −50% 물리 바이트 달성 가능.

2. **TTFT 측정 통합**: 현재 TTFT/처리량은 합성 벤치마크에서 미측정. engine/runner.py를 통한 실 추론 통합 테스트로 TTFT p50/p99, tokens/sec를 직접 측정하면 evaluation_criteria.md §1 항목도 정량 평가 가능.

3. **Activity A 결합**: B+C 기반 위에 KV Cache-aware Scheduling(Activity A)을 추가하면 비연속 히트율을 더욱 높이고 배치 효율을 개선할 수 있음. 다음 사이클의 자연스러운 확장 방향.

4. **WikiText-103 실제 perplexity 측정**: 현재 proxy(합성 KV)로 검증했으나, 실 모델 + WikiText-103 데이터로 perplexity 직접 측정 시 더 강한 evidence 확보 가능.
