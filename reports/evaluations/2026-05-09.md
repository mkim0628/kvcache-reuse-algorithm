## 측정 수치

- **Cache Hit Rate (비연속)**: 100% (실험 조건, N=30, noise=0.02, 중간부 쿼리)
- **eOptShrinkQ Key MSE**: 1.05% / Val MSE: 0.23% (perplexity proxy; 기준 < 5%)
- **Index Search Time (N=1000, dim=32)**: ~25.5ms (linear ~23.8ms; 비율 0.93×)
  - 주: 소규모(N=1000)에서는 Python 오버헤드로 index가 linear과 유사. N이 클수록 tree pruning 효과 증가 (test 설계 수준에서 허용)
- **Test Count**: 547 passed / 547 total
- **SpecKV Gamma (FP16/INT8/NF4 입력)**: 5 / 2 / 3 (NF4 ≤ FP16 확인)
- **ContextIntensiveAccuracyGuard**: 고밀도 min_bits=4.0, 저밀도 min_bits=1.0

## 미해결 이슈

1. **실 GPU 처리량·TTFT 측정 미실시**: §1·§2·§3 수치 항목은 시뮬레이션 기반 Pass이며, 실 모델(GPT-2/LLaMA) GPU 벤치마크가 없다. 다음 사이클에서 `experiments/run_experiment.py` 실행 필요.
2. **실 모델 perplexity/태스크 정확도 측정 미실시**: §4 필수 항목은 proxy(MSE) 기준 충족이지만 wikitext2 perplexity 실측은 수행되지 않았다.
3. **O(log N) 실증적 우위 미확인**: N=1000 규모에서 Python overhead로 index가 linear과 대등. N=10K+ 스케일에서의 실측 검증 권장.
4. **SemanticBoundarySegmentCache ↔ TriangleInequalitySegmentIndex 완전 통합**: 현재 통합 테스트에서 수동 연결 방식이며, 자동 파이프라인 통합 인터페이스 미구현.
