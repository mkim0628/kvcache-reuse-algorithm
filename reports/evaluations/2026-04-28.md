## 미해결 이슈

1. **실제 LLM 모델 연동 미실시**: 현 구현은 합성 KV 텐서 기반 시뮬레이션. 실제 GPT-2/LLaMA 모델에서 perplexity를 직접 측정하면 INT8 정확도 영향을 더 엄밀히 검증 가능.
2. **throughput tokens/sec 측정 없음**: 현재 TTFT만 측정. 실제 토큰 생성 처리량 측정을 위해 `src/engine/batch_runner.py` 구현 필요.
3. **Scheduling Overhead 미측정**: Activity A 미구현. 스케줄링 오버헤드 TTFT +5% 이내 항목 적용 불가.

## 다음 사이클 제언

1. **Activity A 추가**: B+C 위에 cache-hit-weighted 스케줄러를 추가하면 비연속 히트율을 추가로 높일 수 있다. 특히 partial-hit(일부 세그먼트만 캐시된) 요청을 warm 노드로 라우팅하는 전략.
2. **실제 모델 perplexity 검증**: WikiText-2에서 GPT-2 모델의 실제 perplexity를 측정해 ±1% 이내 보존 여부를 직접 검증.
3. **NVFP4 지원 실험**: 현재 INT8(75% savings) → FP4(87.5% savings)로 확장 시 −80% 이상 메모리 절감 가능. 정확도 영향 측정 필요.
4. **청크 크기 최적화**: 현재 chunk_size=64 고정. 64/128/256 비교 실험으로 히트율-레이턴시 트레이드오프 분석.
