## 미해결 이슈

### [중간 우선순위] `noncontiguous_hit_rate()` 메서드 버그 — `relay_ulayer_segment.py`

`get_segments_layer_selective()` 호출 경로에서 `self._hits`가 증가하지 않아 `noncontiguous_hit_rate()`가 항상 0.0을 반환한다. 실제 비연속 히트는 정상 기록(_noncontiguous_hits 카운터)되지만 분모가 0이어서 rate = 0.0.

- 영향 범위: `noncontiguous_hit_rate()` 공개 API 오작동
- 테스트 영향: 기존 테스트는 `nc_rate >= 0.0` 및 `_noncontiguous_hits >= 1` 만 검사하여 버그 미감지
- 수정 방향: `get_with_layer_selection()` 내부에서도 `self._hits` / `self._misses` 카운터를 증가시키거나, `noncontiguous_hit_rate()` 분모를 `_base_cache._hits`로 교체

### [낮음] 실 GPU 벤치마크 없음

- throughput/TTFT 수치가 합성 추정치
- 향후 사이클에서 실제 모델 + GPU 환경 측정 필요

### [낮음] 캐시 히트율 향상 Activity A 단독 측정 없음

- Activity A의 "스케줄링 미적용 대비 +10%p" 기준을 단독 측정하지 않음
- 다음 사이클에서 baseline(no scheduler) vs RadixFeatherBatchScheduler 비교 실험 추가 권장
