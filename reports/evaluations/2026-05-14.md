## 미해결 이슈

### Fail 항목: 없음

모든 필수(mandatory) 항목 Pass. 높음 가중치 항목도 전부 Pass 또는 Partial.

### 주의 사항 (Fail이 아닌 설계 판단)

1. **실제 압축 계수와 레이블 불일치**: "4x", "10x", "20x" 레이블이 실제 달성 압축 계수(1.88x, 3.56x, 6.40x)와
   다르다. 이는 Spec.md에서 명시적으로 승인된 상대적 순서 표지(ordering markers)이며, YAML 파일에
   실제값이 기재되어 있어 추적 가능하다. 다음 사이클에서 레이블 명칭을 실제 계수 기반으로 변경 권장.

2. **Activity C mandatory 기준의 config 의존성**: evaluation_criteria.md §4의 "perplexity ±1%" 필수
   기준은 "4x 레이블" config(bits_direction=8, 1.88x actual)에서만 충족되며, "10x 레이블" config에서는
   attention error 13.1%로 mandatory 임계값 초과. 10x config 에 대한 mandatory 기준은 cosine >= 0.97
   proxy로 대체되며, 이는 Spec.md에 명시된 설계 결정이다.

3. **기존 테스트 환경 오류**: numpy 미설치로 기존 테스트 14개(test_dag_ttl_adjuster.py 등)가
   수집 실패. 이는 이번 사이클 구현 회귀가 아니라 환경 의존성 문제이며, FibQuant 관련 31개 신규 테스트는
   모두 통과. 직접 관련 있는 기존 테스트(test_segmented_cache.py, test_rope_reencoding_cache.py,
   test_vq_codec.py 등 50개)는 회귀 없이 전부 통과 확인.

## 다음 사이클 제언

1. **레이블 명명 개선**: "4x/10x/20x" 레이블을 "high-accuracy(1.88x)/medium(3.56x)/high-ratio(6.40x)"
   등 실제 달성값 기반 명칭으로 전환하면 평가 기준과의 정합성이 높아진다.

2. **중간 압축 구간 보강**: 1.88x→3.56x 사이 구간(bits_direction=6, ~2.5x)에서 attention error < 1%를
   유지하는 sweet spot 탐색. 현재 mandatory 기준을 만족하는 최고 압축 계수가 1.88x로 제한적이다.

3. **GPU 환경 TTFT 실측**: 현재 encode/decode 오버헤드는 CPU sanity bound로만 검증됨. 실제 GPU 환경에서
   TTFT p50 +10% 이내 기준 정량 측정 필요.

4. **numpy 환경 의존성 해결**: 기존 14개 테스트의 numpy 의존성을 pip install numpy로 해결하거나,
   해당 모듈의 numpy 의존성을 PyTorch 전용으로 교체하여 전체 테스트 스위트 통합 실행 가능하게 복구.
