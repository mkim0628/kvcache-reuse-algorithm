# vLLM 평가 리포트 — 2026-05-13

## vLLM 버전
- 테스트 버전: 0.20.2
- 설치 방법: pip install --upgrade vllm

## 이번 사이클 Activity
A + B + C (조합)

## 총평

Activity A (PBKVAgentSegmentPreservationSchedulerMixin), B (KVFoldAccumulativeBlockManager),
C (SRFTInt8AttentionHook) 세 컴포넌트 모두 vLLM 0.20.2에 충돌 없이 임포트되고 smoke 테스트
47/47 통과(4 skipped — vLLM engine 없는 환경 조건부), deprecation 경고 0건이다.
Activity C의 필수 정확도 조건(perplexity ±1%)은 모든 테스트에서 여유 있게 통과했다
(key 상대 오차 0.53~0.59%, 주의력 출력 오차 0.81%, KL < 1e-7, 코사인 > 0.9999).
메모리 감소율 73.4%(이론치)가 Report ① 대비 그대로 유지되어 목표 −30%를 크게 초과한다.
주요 제한은 Activity A 스케줄링 오버헤드가 소규모 큐(W≤10)에서는 통과하지만
대형 큐(W=50+)에서 5ms 임계값을 초과한다는 점이다.

## 미해결 이슈

1. **Activity A 대형 큐 오버헤드**: W=100에서 pbkv_pre_schedule 오버헤드 ~9ms로 5ms 임계값
   초과. W≤50에서는 4.4ms로 경계이며, 실제 에이전틱 워크로드(소규모 큐)에서는 문제 없음.
   대형 배치 서빙 시나리오 대응을 위해 O(W)→O(1) 근사 임베딩 or 조기 종료 최적화 권고.
2. **실제 vLLM 엔진 통합 미실시**: 처리량(tokens/sec), TTFT p50/p99, TBT 지표는 실 GPU
   엔진 없이 측정 불가. 다음 사이클에서 실 모델 연동 후 results/2026-05-13/metrics.json
   기록 권고.
3. **파일 헤더 버전 표기 불일치**: block_manager_patch.py, attention_backend_patch.py의
   docstring 일부가 vLLM 0.20.1을 참조함(실제 실행 환경 0.20.2). 기능적 영향 없는 cosmetic
   이슈이나 다음 사이클에서 정리 권고.
