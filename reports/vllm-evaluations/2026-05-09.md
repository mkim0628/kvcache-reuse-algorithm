## 하위 호환 버그 분석

### VllmRedundancyAwareEvictionPolicy.select_evict_keys 타이브레이킹 버그

**위치**: `vllm_integration/attention_backend_patch.py`, `select_evict_keys()` 메서드

**증상**: importance_score=1.0인 seg0의 eviction_score = `(1 - 1.0) * redundancy = 0.0`이 맞지만, n_evict=3을 5개 후보에서 선택 시 다른 세그먼트들도 embedding 유사도가 낮아 score ≈ 0에 수렴하면 tie-breaking에 따라 seg0가 top-3에 포함됨.

**영향**: 2026-05-04 backward-compat 테스트 실패 → install.sh 조기 종료 → 2026-05-09 in-script 검증 미실행.

**수정 방향**: `select_evict_keys()`에서 score=0.0 항목 필터링 추가:
```python
return [k for k, s in scored if s > 0.0][:n_evict]
```

### TriangleIndexKVCacheManagerMixin MRO 설계 결함

**위치**: `vllm_integration/install.sh`, 2026-05-09 smoke test `_MinimalManager`

**증상**: `_MinimalManager(TriangleIndexKVCacheManagerMixin)` 가 `get_computed_blocks`를 override하면, Python MRO상 `_MinimalManager.get_computed_blocks`가 호출되어 mixin의 `get_computed_blocks` 호출 불가. `total_lookups` 카운터 미증가.

**영향**: install.sh in-script `triangle_index_stats()` 테스트 assertion 실패. 실제 vLLM 통합 시 `TriKVManager(TriangleIndexKVCacheManagerMixin, KVCacheManager)` 구조에서는 문제 없음.

**수정 방향**: `_MinimalManager`에서 `get_computed_blocks` override 제거하고 `super().get_computed_blocks()` 체인 활용.
