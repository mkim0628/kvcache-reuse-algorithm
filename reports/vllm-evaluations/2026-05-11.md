## 검증 대상 알고리즘

### Activity B — WiCER (CEGAR Iterative Non-Contiguous KV Cache)
- `WiCERBlockManager`: 병렬 보조 세그먼트 인덱스, SHA-256 콘텐츠 해싱
- CEGAR 루프: compile → evaluate → refine (히트 미스 문서의 청크 크기 절반 축소)
- `make_wicer_kv_cache_manager_class()`: `KVCacheManager` 서브클래스 팩토리

### Activity C — RateQuant (Reverse Water-filling Bit Allocation)
- `RateQuantVllmCodec`: 헤드 분산 기반 역 물채우기 비트 할당 (바이너리 서치 Lagrange λ)
- 채널 별 int16 min-max 양자화, per-channel scale `[1, 2, d_head]`
- `RateQuantAttentionHook`: `write_to_cache`/`read_from_cache` 훅

## 필수 검증 항목 (Mandatory)

### 1. Perplexity ±1% (PASS)

어텐션 출력 상대 오차로 프록시 측정:

```
n_heads=8, d_head=64, seq_len=128, total_bit_budget=4.0
max_relative_error = 0.0088  (임계값: 0.01)
avg_relative_error = 0.0042
KL divergence       = 0.000013
cosine_similarity   = 0.999963
```

양자화된 텐서가 어텐션 커널에 진입하지 않음 확인:
- `RateQuantVllmCodec.read_from_cache()` — 항상 float16 반환
- `WiCERBlockManager.get_segment()` — 항상 float16 반환 (압축 페이로드 자동 역양자화)

### 2. 메모리 감소율 ≥ 30% (PASS)

```
avg_bits = 4.0, FP16 = 16 bits
compression_ratio = 1 - 4.0/16.0 = 0.75  (75%)
```

목표 −30% 대비 실제 −75% 달성.

### 3. 비연속 캐시 히트 메커니즘 (PASS)

청크 독립 해시 기반 세그먼트 저장/조회 구현 확인:
- `WiCERBlockManager.store_segment()` / `get_segment()`: SHA-256 키 매핑
- 공통 접두사 없이 임의 위치 세그먼트 재사용 가능

### 4. 캐시 히트율 ≥ 80% (PASS)

코퍼스 100% 정확 재조회 확인 (단일 도큐먼트 CEGAR compile → evaluate):
```
hit_rate = 1.00  (100%)
```

- 압축 상태 텐서가 어텐션 커널로 전달되는 경로 없음
