```python
original = torch.cat([keys, values], dim=-1)        # (n_tokens, 2*d_head)
p_orig = F.softmax(original.float().mean(dim=-1), dim=0)   # 토큰 차원 softmax
reconstructed = torch.cat([k_decoded, v_decoded], dim=-1)
q_dec = F.softmax(reconstructed.float().mean(dim=-1), dim=0)
kl = F.kl_div(q_dec.log().clamp(-100), p_orig, reduction="sum")
# KL = 0.000795 < 0.015  ✓ PASS
```

## 미해결 이슈

### 비블로킹 이슈 1: 실제 vLLM 환경에서 4개 테스트 영구 skip
`TestNonContiguousKVCacheManagerV2`의 4개 테스트가 `_VLLM_AVAILABLE=True`일 때 skip된다. 이는 실제 `KVCacheManager.__init__`이 전체 `KVCacheConfig`, `max_model_len`, `hash_block_size` 등 엔진 레벨 파라미터를 요구하기 때문이다. `NonContiguousKVCacheManagerV2`의 공개 메서드 (`store_segment`, `lookup_segment`, `sign_index_stats`, `lookup_all_segments`)는 `SignVQSegmentIndex`에 직접 위임하며, 후자는 `TestSignVQSegmentIndex`(8 PASS)에서 충분히 테스트된다.

**권고**: 다음 사이클에서 `TestNonContiguousKVCacheManagerV2` 픽스처를 vLLM 설치 여부와 무관하게 `SignVQSegmentIndex`를 직접 생성해 테스트하도록 수정하거나, mock `KVCacheConfig`를 사용하는 방향으로 개선.

### 비블로킹 이슈 2: README.md 상단 파일 맵 불일치
상단 File Map 표에 `leverage_compressor_patch.py`, `sign_vq_block_manager_patch.py`, `cache_config_extension.py` 3개 파일이 누락. 하단 "2026-05-02 Additions" 절에서 커버되나 통일성이 부족.

**권고**: 다음 이식 사이클에서 README.md 상단 파일 맵 갱신.

### 비블로킹 이슈 3: approx_sign Stage 2 실제 미발동 가능성
현재 `put()` 구현에서 하나의 청크 키로 FP16 스토어(Tier-1)와 sign 스토어(Tier-2) 양쪽에 저장한다. Stage 1이 FP16 스토어에서 히트하면 Stage 2는 실행되지 않으므로, 동일 청크에 대해 approx_sign 경로가 사실상 발동되지 않는다. approx_sign은 FP16 스토어에 없는 순수 Tier-2 전용 청크가 있을 때만 동작한다. 이는 Report ①에서 이미 `noncontiguous_ratio` 동작으로 문서화된 설계 한계이며, 현재 사이클 요구사항 범위 밖이다.
