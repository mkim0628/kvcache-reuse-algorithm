## 총평

이번 사이클(A+C Cross-1)의 vLLM 이식 코드가 모든 필수 평가 항목을 충족했다.
Activity A의 캐시 히트율이 +57.5%p 향상됐고, Activity C의 perplexity proxy MSE가 0.47~0.50%로
±1% 기준 이내이며, KV 메모리 감소율이 39~87%로 목표치(−30%)를 초과 달성했다.
TTFT p50 오버헤드는 +0.47%로 허용 기준(+5%) 이내이다. 미해결 항목으로 TTFT p99
burst 시나리오에서 +286.9% 초과와 요청 공정성(burst p99/normal p50 = 4.0×)이 있으나,
이는 필수 항목이 아니며 Report ①에서 식별된 것과 동일한 이슈다. 3회차(최종)로서
현재 상태 그대로 리포트를 확정한다.

## 미해결 이슈

### 1. TTFT p99 burst 오버헤드 (+286.9%) — 높음 가중치, Fail

- **원인**: 선점형 스케줄러 도입 후 KV 복원 경로를 거치는 요청의 p99가 25.77ms로
  baseline 6.66ms 대비 기준치 초과. Report ①에서 동일하게 식별됨.
- **영향 범위**: 필수 항목이 아니며, TTFT p50은 기준 이내. burst 시나리오에서도
  선점형 p99(26.04ms)는 baseline burst p99(38.70ms) 대비 −32.7% 개선됨.
- **개선 방향**: 복원 vs 재계산 동적 비용 모델; SLA Tier-A 범위 확장;
  eOptShrinkQ decode 비동기 프리패치.

### 2. 요청 처리 공정성 (burst p99 / normal p50 = 4.05×) — 높음 가중치, Fail

- **원인**: fairness_max_wait이 배치 스텝 수 기준(정수)이나,
  실제 대기 시간(ms)으로 환산 시 2× 기준(12.86ms) 초과.
- **개선 방향**: pko_fairness_max_wait을 ms 단위로 변경;
  burst 시나리오 전용 SLO 기준 분리.

### 3. __init__.py re-export 부재 — 낮음 가중치

- **현상**: vllm_integration 패키지 `__init__.py`가 주석만 포함하고 클래스를 re-export하지
  않아 `from vllm_integration import PreemptiveKVOffloadSchedulerMixin` 형태의 임포트가
  ImportError를 발생시킴. 서브모듈 직접 임포트로 우회 가능.
- **개선 방향**: `__init__.py`에 주요 클래스 re-export 추가.

## 다음 사이클 제언

1. **TTFT p99 SLO 보장**: pko_fairness_max_wait을 ms 단위로 변환하고,
   복원 경로에 비동기 프리패치를 적용해 p99를 +15% 이내로 억제.
2. **__init__.py re-export 정비**: 외부에서 사용하기 쉽도록 공개 API를 `__init__.py`에 명시.
3. **vLLM 버전 업그레이드 주의사항**: vLLM 0.21+ 시 `vllm.v1.worker.gpu_model_runner`의
   KV cache 블록 테이블 API가 변경될 가능성이 있으므로, block_manager_patch.py의
   `_sdm_block_map` 접근 방식을 버전별 분기 처리 고려.
4. **GPU 환경 실측**: 현재 측정값 대부분이 CPU-only 합성 워크로드 기반이므로,
   실제 GPU 환경(A100/H100)에서의 TTFT, 처리량, KV 메모리 실측이 필요하다.
5. **메모리-처리량 트레이드오프 곡선**: 51% 메모리 절감을 활용한 배치 크기 증가로
   처리량 추가 이득을 정량화해 하이퍼파라미터 튜닝 근거 마련.
