## 미해결 이슈

### [중간 우선순위] `load_batch()` 비연속 히트 오버카운팅 버그 — `block_manager_patch.py`

`load_batch()`의 현재 구현은 `miss_ids` 리스트를 hit 발생 시 초기화하지 않는다. 따라서 한 번 miss가 발생하면 이후 모든 연속된 hits가 "비연속"으로 분류된다. 실제로 [A, MISS, B, C, MISS2, D] 패턴에서 기대값은 NC=2 (B, D) 이지만 구현은 NC=3 (B, C, D)을 반환한다.

- 영향 범위: `noncontiguous_hit_rate()` 과부풀림; 실제 NC 비율보다 높게 보고
- 수정 방향: hit 발생 후 `miss_ids` 를 빈 리스트로 초기화하도록 변경

```python
# 현재 (버그)
if miss_ids:  # hit after a miss → non-contiguous
    self._noncontiguous_hits += 1

# 수정 필요
had_prior_miss = len(miss_ids) > 0
miss_ids.clear()  # reset after hit
if had_prior_miss:
    self._noncontiguous_hits += 1
```

이 버그는 Report ① 의 `noncontiguous_hit_rate()=0.0` 버그와 다른 코드 경로(block_manager_patch의 `RelayUShapeAuxStore.load_batch()`)에서 발생한다.

### [낮음] 3D KV 텐서 입력 시 eviction 미작동

`LookaheadEvictionAttentionHook.write_to_cache()`는 dim<4인 입력에 대해 identity passthrough를 적용한다. vLLM FlashAttentionImpl 실제 KV는 4D 이상이므로 운영 환경에서는 문제없으나, 외부 사용자가 3D KV로 호출하면 eviction이 작동하지 않는다는 점을 문서화 필요.

### [낮음] 실 GPU 벤치마크 없음

처리량/TTFT 수치가 합성 추정치 (실 GPU 없는 환경). 향후 사이클에서 실제 모델 + GPU 환경 측정 필요.

### [낮음] LookaheadModule untrained 상태

현재 LookaheadModule이 key-norm 블렌딩 폴백을 사용한다. 정확도 단위 테스트는 합성 sparse/high-norm 데이터로 통과하지만, 실제 LLM 추론 환경에서 trained lookahead 가중치 사용 권장.
