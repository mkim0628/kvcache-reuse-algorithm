## 변화 요약
- **Activity A 이론-보장 안정성 및 3D 병렬 복원 이중 등장**: CacheFlow의 3D 병렬화(토큰·레이어·GPU 세 축 동시 활용)와 Flow-Controlled Scheduling의 안정성 이론(필요조건 증명 + 활성화 예산 파라미터)이 이전 사이클의 모든 A 기법이 다루지 않은 두 새로운 설계 차원을 동시에 열었다. 특히 CacheFlow는 vLLM+LMCache 기반 구현이므로 6단계 vLLM 이식에 즉시 활용 가능하다.
- **Activity B 재계산 제로 맥락 독립 패러다임 전환**: KV Packet(arXiv 2604.13226)이 "선택적 재계산으로 맥락 의존성 보정"에서 "소프트-토큰 어댑터로 맥락 의존성 자체 제거"로의 질적 전환을 실증했다. 이전 사이클의 모든 B 기법의 정확도 천장(accuracy ceiling)을 근본적으로 높이는 최초 재계산-프리 비연속 재사용 기법이다.
- **Activity C VQ·시간적 계층화·엔트로피 코딩 세 신규 기둥 등장**: VQKV(훈련 없이 VQ 82.8% 압축), TTKV(시간 기반 HBM/DRAM 계층화 5.94× 트래픽 감소), KVTC(ICLR 2026, 20× 압축 ±1% 정확도)가 이전 사이클 C 기법 전체(스칼라 양자화·저랭크·중요도 퇴거)와 이론적 기반이 다른 세 신규 기둥을 동시에 형성한다.

## 이번 사이클 초점
**B+C (KV Packet 소프트-토큰 어댑터 비연속 재사용 + VQKV/KVTC VQ·엔트로피 코딩 압축)** — 보조로 **A** CacheFlow 3D 병렬 복원 스케줄러, **A+B** KV Packet + CacheFlow 통합, **A+C** TTKV 시간적 계층화 + Valve 온-오프라인 KV 메모리 회수

**선택 이유**:
1. 이전 두 사이클(2026-05-08 A+C, 2026-05-09 A+B)에서 A 조합에 집중했으므로, 이번 사이클에서는 B+C 통합에 초점을 맞춘다. KV Packet의 소프트-토큰 어댑터(B)와 VQKV의 VQ 압축(C)은 "재계산 없는 맥락 독립 세그먼트 + 고압축 저장"이라는 B+C의 가장 직접적인 결합 경로를 제공한다.
2. KVTC의 PCA 기저 1회 계산 + DP 비트할당 + 엔트로피 코딩이 기존 eOptShrinkQCodec과 직교하는 새 Activity C 기둥을 형성하며, 20× 압축 + ±1% 정확도가 연구 목표를 10× 초과 달성한다.
3. TTKV의 시간 기반 계층화가 Activity B 세그먼트의 접근 빈도와 결합하면 "자주 재사용되는 비연속 세그먼트 → HBM, 드물게 재사용 → DRAM VQ 압축"하는 B+C 동적 압축 정책이 가능하다.
4. CacheFlow의 3D 병렬 복원이 비연속 세그먼트 다수를 병렬로 복원할 때 직접 적용되므로 A 보조 타겟으로 설정한다.

## Activity A 아이디어

### 아이디어 A-1: CacheFlow3DParallelRestoreScheduler — 토큰·레이어·GPU 3축 KV 복원 병렬화 + 배치-인식 한계편익 두-포인터 스케줄러
- **가설**: 기존 KV 복원 방법들이 토큰·레이어·GPU 세 축 중 하나씩만 활용하고 배치 단위 요청 간 자원 경쟁을 무시해 Disaggregated Prefill 환경에서 KV 복원이 TTFT 병목이 되는 문제를, "통합 3D 병렬화 추상화(토큰-축 청크 재계산+I/O 중첩, 레이어-축 선후행 레이어 연산 중첩, GPU-축 분산 병렬 복원) + 배치-인식 두-포인터 스케줄러(재계산 비용 감소 한계편익이 높은 요청을 우선 처리)"를 적용하면, 이전 사이클의 모든 A 기법(PPDAppendPrefillRouter, ContiguousChunkIOScheduler, PreemptiveKVOffloadScheduler 등)이 복원의 세 축 중 하나씩만 최적화하거나 배치 자원 경쟁을 다루지 않은 것과 달리 TTFT를 10~62% 단축할 수 있다.
- **접근 방법**:
  1. `CacheFlow3DParallelRestoreScheduler` 클래스 (`src/scheduler/cacheflow_3d_restore.py`): `BaseScheduler` 상속
  2. 3D 병렬화 추상화 레이어:
     - 토큰-축: FlashAttention 선호 블록 크기(512 토큰)에 청크 정렬. 청크 단위 재계산과 I/O를 CUDA 스트림 분리로 오버랩.
     - 레이어-축: 레이어 L 연산 중 레이어 L+1의 KV를 백그라운드에서 I/O. 레이어 간 파이프라인 깊이 파라미터(`pipeline_depth`, 기본 2).
     - GPU-축: 멀티-GPU 환경에서 비연속 세그먼트를 GPU 집합에 분산 복원. `distribute_segments(segment_list, gpu_pool)` 인터페이스.
  3. 배치-인식 두-포인터 스케줄러:
     - 왼쪽 포인터: 재계산 비용이 가장 낮은(=이미 캐시에 가까운) 요청 → 즉시 복원 우선
     - 오른쪽 포인터: 복원 지연 허용 가능한(=TTFT SLO 여유 있는) 요청 → 복원 지연
     - 한계편익 계산: `marginal_benefit(req) = (recompute_cost_saved) / (restore_resource_used)`. 처리 후 비용을 동적 갱신.
  4. ContiguousChunkIOScheduler(05-06 기구현)와의 결합: ContiguousChunk 단위 I/O 정렬 원칙을 토큰-축 청크 크기(512 토큰)와 통합. 기존 청크 단위를 512 토큰으로 정렬 업그레이드.
  5. PPDAppendPrefillRouter(05-09 기구현)와의 결합: D 노드 append-prefill 요청의 복원 병목을 3D 병렬화로 추가 단축. append-prefill 요청 우선순위를 두-포인터에 반영.
  6. 이전 A 기법과의 차별점: ContiguousChunkIOScheduler(05-06)가 청크 경계-I/O 정렬(단일 축)에 집중하고, PreemptiveKVOffloadScheduler(05-08)가 선점 타이밍과 CPU 전송 오버랩에 집중한 반면, CacheFlow3DParallelRestoreScheduler는 토큰·레이어·GPU 세 축을 동시에 병렬화하는 완전히 새로운 복원 추상화 계층을 도입한다.
- **대상 환경**: 단일 노드 / 멀티 노드 양쪽
- **멀티 노드 고려사항**: GPU-축 병렬화가 Disaggregated Prefill의 P/D 노드 경계를 교차해 비연속 세그먼트를 복수 P 노드에서 병렬 복원하는 아키텍처로 자연스럽게 확장. TTKV(Activity C)의 HBM/DRAM 계층 정보를 복원 우선순위(HBM 계층 = 복원 비용 낮음, DRAM 계층 = 복원 비용 높음)에 연동하는 "계층-인식 CacheFlow" 변형 가능.
- **예상 효과**: TTFT −10~62% (CacheFlow arXiv 2604.25080 실증), 처리량 +20~40% (3D 병렬화로 복원 병목 제거), Scheduling overhead TTFT +3% 이내
- **구현 난이도**: medium
- **우선순위**: 2

### 아이디어 A-2: StabilityBoundedActivationBudgetScheduler — 활성화 예산 파라미터 기반 이론-보장 KV 안정성 스케줄러
- **가설**: LLM 추론에서 디코드 길이가 사전에 알 수 없고 요청별 KV 사용량이 동적으로 증가해 KV 캐시 오버플로우와 시스템 불안정이 발생하는 문제를, "활성화 예산(activation budget)으로 타임스텝마다 활성 집합에 진입 가능한 요청 수를 제한"하고 이 조건이 "기대 워크로드 < KV 캐시 용량"이라는 수학적 안정성 필요조건을 항상 만족하도록 온라인으로 조정하면, 이전 사이클의 OnlineLatencyOptimalScheduler(이론 최적 레이턴시에 집중, 안정성 조건 미형식화)와 WAITThresholdBatcher(경험적 퇴거 없는 처리량에 집중)와 달리 "임계 부하 조건에서도 수학적으로 보장된 시스템 안정성"을 달성할 수 있다.
- **접근 방법**:
  1. `StabilityBoundedActivationBudgetScheduler` 클래스 (`src/scheduler/stability_activation_budget.py`): `BaseScheduler` 상속
  2. 활성화 예산 계산: `activation_budget_T = kv_cache_capacity × (1 - max_expected_kv_per_request_fraction)`.
     - `max_expected_kv_per_request_fraction`: 히스토그램 기반 상위 95th 퍼센타일 출력 길이로 추정 (온라인 업데이트).
  3. 타임스텝 활성 집합 제어: 각 스텝에서 활성 요청의 KV 사용량 합이 `activation_budget_T` 미만인 경우에만 새 요청을 활성 집합에 진입 허용.
  4. 안정성 조건 모니터링: `E[workload] < kv_cache_capacity` 조건을 실시간 추적. 위반 시 `activation_budget_T`를 자동 하향 조정(안전 마진 증가).
  5. WAITThresholdBatcher(05-08 기구현)와의 통합: WAIT 배치 임계값을 활성화 예산의 하한으로 설정 → WAIT가 배치 시작 타이밍을 결정하고, StabilityBudgetScheduler가 활성 집합 크기를 제한하는 이중 안전 계층 구성.
  6. Flow-Controlled Scheduling의 이론적 보장 적용: (1) `E[workload] < capacity` 필요조건 지속 만족 검증, (2) 일정한 활성화 예산으로 안정성 달성, (3) 과부하 조건에서 무제한 레이턴시 방지.
  7. 이전 A 기법과의 차별점: OnlineLatencyOptimalScheduler(05-06)가 레이턴시 최소화 이론 최적성에 집중하고 WAITThresholdBatcher(05-08)가 경험적 퇴거 없는 처리량에 집중한 반면, StabilityBoundedActivationBudgetScheduler는 "임계 부하 조건에서 시스템 불안정이 발생하지 않음"을 수학적으로 보장하는 안정성 이론 기반 설계를 처음으로 도입한다.
- **대상 환경**: 단일 노드 / 멀티 노드 양쪽
- **멀티 노드 고려사항**: 멀티 노드에서는 `kv_cache_capacity`를 클러스터 전체 합으로 모델링. 노드별 독립 안정성 조건과 클러스터-레벨 안정성 조건을 계층적으로 관리. Valve(arXiv 2604.07874)의 동적 메모리 예약과 결합해 온라인 요청의 KV 성장 예산을 실시간 보장.
- **예상 효과**: 시스템 안정성 보장 (임계 부하에서 KV 오버플로우 0%), 처리량 +15~25% (안정적 활성 집합 크기 유지), TTFT p50 오버헤드 +3% 이내
- **구현 난이도**: low
- **우선순위**: 3

### 아이디어 A-3: ValveOnlineOfflineKVReclaimScheduler — 온-오프라인 동시 운영 기반 페이지 폴트 없는 KV 메모리 회수 스케줄러
- **가설**: 프로덕션 LLM 추론이 트래픽 폭주로 과도하게 프로비저닝되어 오프-피크 시간 GPU 자원이 낭비되는 문제를, "채널 제어 컴퓨트 격리(요청당 최대 1회 서브-밀리초 선점) + 페이지 폴트 없는 KV 메모리 회수(선점 KV를 안전하게 오프라인 배치로 재할당) + 동적 메모리 예약(온라인 요청 KV 성장 예산 실시간 보장)"이라는 세 가지 GPU 런타임 기법을 결합하면, 이전 사이클의 PreemptiveKVOffloadScheduler(05-08)가 CPU 오프로딩과 I/O 오버랩에 집중한 것과 달리 "선점 레이턴시 + 선점률을 공동으로 유계(jointly bounded)하면서 오프라인 배치 작업과 온라인 서빙을 동시 운영"하는 새 차원으로 GPU 이용률을 34.6% 향상시킬 수 있다.
- **접근 방법**:
  1. `ValveOnlineOfflineKVReclaimScheduler` 클래스 (`src/scheduler/valve_kv_reclaim.py`): `BaseScheduler` 상속
  2. 채널 제어 선점: 요청당 최대 1회 선점 보장. 선점 발생 시 서브-밀리초(< 1ms) 완료 제약. 선점 횟수 × 레이턴시 공동 유계 파라미터: `max_preemptions_per_request=1`, `max_preempt_latency_ms=1.0`.
  3. 페이지 폴트 없는 KV 회수: 선점된 요청의 KV 메모리를 오프라인 배치 작업에 재할당할 때 GPU 페이지 폴트 없이 직접 재매핑하는 메모리 관리 인터페이스. `reclaim_kv_for_offline(request_id, offline_batch_id)`.
  4. 동적 메모리 예약: 각 온라인 요청의 예상 KV 성장(현재 토큰 수 × 평균 생성 비율)을 실시간 추정해 예약. `reserve_kv_budget(request_id, estimated_kv_growth)`.
  5. 온-오프라인 배치 공존 관리: 오프라인 배치(재훈련, 평가)를 KV 메모리 여유 구간에 삽입. 온라인 요청 폭주 시 오프라인 배치 즉시 중단 + KV 메모리 환원.
  6. PreemptiveKVOffloadScheduler(05-08)와의 차별점: 05-08 기법이 GPU→CPU KV 전송과 백그라운드 오버랩에 집중한 반면, ValveOnlineOfflineKVReclaimScheduler는 KV 메모리를 CPU로 옮기지 않고 GPU 내에서 온라인/오프라인 간 직접 재할당하는 "페이지 폴트 없는 GPU-내 재매핑"이라는 새로운 시스템 계층 최적화를 도입한다.
  7. Valve의 프로덕션 검증: 8,054 GPU 클러스터에서 GPU 이용률 34.6% 향상, TTFT <5% 증가, TPOT <2% 증가를 실증한 설계 원칙을 단일 노드 구현에 적용.
- **대상 환경**: 단일 노드 / 멀티 노드 양쪽
- **멀티 노드 고려사항**: 멀티 노드에서는 오프라인 배치 작업을 KV 메모리 여유가 가장 큰 노드에 우선 배치. 온라인 요청 폭주 시 오프라인 배치를 클러스터 전체에서 즉시 중단하는 중앙화 제어 인터페이스.
- **예상 효과**: GPU 이용률 +34.6% (Valve arXiv 2604.07874 실증), TTFT 증가 <5%, TPOT 증가 <2%, 처리량 +30~45% (온-오프라인 동시 운영으로 GPU 낭비 제거)
- **구현 난이도**: medium
- **우선순위**: 1

## Activity B 아이디어

### 아이디어 B-1: KVPacketSoftAdapterCache — 소프트-토큰 어댑터 래핑 기반 재계산-프리 맥락 독립 비연속 KV 재사용
- **가설**: 비연속 KV 세그먼트 재사용의 근본적 한계가 "어텐션 분포가 맥락에 따라 달라지는 맥락 의존성"에 있으며, 이전 사이클의 모든 B 기법(QueryCentricRecomputeCache, CrossChunkDependencyHybridReuse, SemanticBoundarySegmentCache, TriangleInequalitySegmentIndex 등)이 모두 "선택적 재계산으로 맥락 의존성 보정"에 의존한 것과 달리, 각 문서 KV 블록을 경량 훈련 가능 소프트-토큰 어댑터(trainable soft-token adapter)로 래핑해 자기지도 증류(self-supervised distillation)로 훈련하면 맥락 불연속을 재계산 없이 해소하는 "맥락 독립 KV 패킷"을 구성할 수 있으며, TTFT를 재계산 기준선 대비 대폭 단축할 수 있다.
- **접근 방법**:
  1. `KVPacketSoftAdapterCache` 클래스 (`src/cache/kv_packet_soft_adapter.py`): `CacheStore` 인터페이스 상속
  2. 어댑터 구조: 각 KV 패킷(문서 KV 블록)에 경량 소프트-토큰 어댑터(`SoftTokenAdapter`) 부착.
     - 어댑터 크기: d_head × r (r=8~16, 전체 KV 크기의 <1%).
     - 어댑터 역할: 캐시된 KV가 새 맥락에서 어텐션 시 발생하는 분포 이동을 소프트 토큰으로 보정.
  3. 자기지도 증류 훈련:
     - 입력: 문서 KV(원본 맥락에서 계산됨)
     - 목표: 다양한 새 맥락에서 재계산한 KV와 어댑터 보정 KV 간 MSE 최소화
     - 훈련 비용: 문서당 소수 다양한 맥락(8~16개) × 단일 패스 → 총 1K 스텝 이하
  4. 추론 시 조합: `pack(doc_kv_list) → [adapter_1, kv_block_1, adapter_2, kv_block_2, ...]` 단순 연결. 추가 재계산 없이 서빙.
  5. `TriangleInequalitySegmentIndex`(05-09 기구현)와의 통합: KV 패킷을 기존 세그먼트 인덱스에 등록. 어댑터 크기가 미미해 임베딩 계산에 영향 없음.
  6. 모델 파라미터 수정 불필요: 어댑터만 훈련. 어댑터는 범용(universal)으로 한 번 훈련 후 다양한 맥락에 재사용.
  7. 이전 B 기법과의 차별점:
     - CrossChunkDependencyHybridReuse(05-09)가 교차-청크 의존성을 분류해 선택적 재계산하는 반면, KVPacketSoftAdapterCache는 재계산 자체를 어댑터로 대체해 FLOPs 제로를 달성한다.
     - SemanticBoundarySegmentCache(05-09)가 세그먼트 내부 의미 코어로 청킹하는 반면, 이 기법은 세그먼트 자체를 맥락 독립 패킷으로 변환한다.
     - ProphetKV/QCFuse(05-06) 계열이 "20% 재계산으로 96~101% 정확도"를 달성한 반면, 이 기법은 재계산 0%로 동등 정확도를 목표로 한다.
- **예상 효과**: 비연속 히트율 +25~35%p (맥락 독립으로 어떤 맥락에서도 즉시 재사용), TTFT −50~70% (재계산 제거), 연산량 (FLOPs) 재계산 대비 ~0%, accuracy delta ±0.5% 이내 (KV Packet arXiv 2604.13226 실증: Llama-3.1, Qwen2.5 F1 유지)
- **구현 난이도**: medium
- **우선순위**: 1

### 아이디어 B-2: ThreeTierPersistentKVSegmentCache — HBM/DRAM/디스크 3-티어 에이전트 KV 영속화 + 시간 기반 자동 계층화
- **가설**: 에지·클라우드 멀티에이전트 환경에서 에이전트별 KV를 서버 재시작 후에도 유지하고 메모리 압력을 해소하는 동시에 접근 지연을 최소화하는 문제를, "최신 에이전트 KV → HBM 고정밀도, 중간 오래된 KV → DRAM 4비트 양자화, 가장 오래된 KV → 디스크 4비트(safetensors) 영속화"의 3-티어 자동 계층화를 시간적 거리 기준으로 구현하면, 이전 사이클의 StaticDynamicSegmentCache(05-08)가 갱신 패턴 기반 정적/동적 분리에 집중하고 NVMeDirectKVOffloadScheduler(05-09)가 에지 NVMe 이중 경로에 집중한 것과 달리, "에이전트 KV의 시간 기반 3-티어 영속화 + 직접 주입 복원(재프리필 제거)"이라는 새로운 에이전트 메모리 관리 차원을 달성할 수 있다.
- **접근 방법**:
  1. `ThreeTierPersistentKVSegmentCache` 클래스 (`src/cache/three_tier_persistent_kv.py`): `CacheStore` 인터페이스 상속
  2. 3-티어 구조:
     - Tier 0 (HBM): 최근 N 스텝 이내 활성화된 에이전트 KV. FP16 정밀도. 즉시 접근.
     - Tier 1 (DRAM): N~M 스텝 전 마지막 활성화. Q4(4비트) 양자화(`BatchQuantizedKVCache`). 재진입 시 어텐션 레이어 직접 주입.
     - Tier 2 (디스크): M 스텝 이전 마지막 활성화 또는 명시적 영속화 요청. Q4 + safetensors 파일로 저장. 서버 재시작 후도 복원.
  3. 자동 계층 이동 정책: `time_since_last_access`를 기준으로 HBM→DRAM→디스크 자동 강등. 접근 시 디스크→DRAM→HBM 자동 승격.
     - 파라미터: `tier0_ttl_steps=100`, `tier1_ttl_steps=1000` (YAML 외부화)
  4. 직접 주입 복원: DRAM/디스크 KV를 어텐션 레이어에 직접 주입 (재프리필 없음). Persistent Q4 KV Cache(arXiv 2603.04428) 방식 재현.
  5. `BatchQuantizedKVCache`: 배치 Q4 추론 지원. 복수 에이전트 KV를 동시에 Q4로 처리하는 배치 인터페이스.
  6. TTKV(Activity C, C-2)와의 통합: TTKV의 HBM/DRAM 계층 정보를 3-티어 계층화 정책에 반영. "시간 기반(3-티어)" × "토큰 중요도 기반(TTKV)" 이중 계층 정책.
  7. 이전 B 기법과의 차별점:
     - StaticDynamicSegmentCache(05-08)가 갱신 이벤트 기반 무효화 범위 최소화에 집중한 반면, 이 기법은 시간적 거리 기반 3-티어 영속화에 집중한다.
     - NVMeDirectKVOffloadScheduler(05-09)가 NVMe-Direct 경로(파일시스템 우회)에 집중한 반면, 이 기법은 "에이전트별 KV 상태의 서버 재시작 후 복원" 내구성(durability)을 추가 차원으로 도입한다.
- **예상 효과**: 에이전트 재프리필 지연 −15.7초 (4K 컨텍스트 기준, Persistent Q4 KV Cache 실증), 에이전트 메모리 지속성 (서버 재시작 후 상태 복원), 메모리 −60~75% (Q4 압축 + 3-티어 분산), 비연속 히트율 +20~30%p (영속화된 에이전트 KV 즉시 재사용)
- **구현 난이도**: medium
- **우선순위**: 2

## Activity C 아이디어

### 아이디어 C-1: RSimVQCodec — 잔차 단순 벡터 양자화 + pre-RoPE 공간 코드북 학습 기반 훈련 불필요 고충실도 KV 압축
- **가설**: 기존 스칼라 양자화와 저랭크 근사가 높은 압축률과 높은 재구성 충실도를 동시에 달성하지 못하는 한계를, Residual Simple Vector Quantization(RSimVQ)으로 KV 벡터를 다중 코드북에 매핑하고 정수 인덱스만 저장하면서, RoPE의 이종 주파수 특성으로 Key 캐시의 post-RoPE 표현이 이질적으로 변하는 문제를 "pre-RoPE 공간에서 코드북 학습 후 적용"으로 해소하면, 이전 사이클의 모든 C 기법(eOptShrinkQCodec, TurboQuantCodec, NQKVCodec, FireQCodec, TriAttentionCodec 등)이 스칼라 양자화·저랭크·중요도 퇴거에 머문 것과 달리 훈련 없이 82.8% 압축(5.8×) + LongBench 98.6% 성능을 달성할 수 있다.
- **접근 방법**:
  1. `RSimVQCodec` 클래스 (`src/cache/rsimvq_codec.py`): `CompressionCodec` 인터페이스 구현
  2. pre-RoPE 코드북 학습:
     - 교정 데이터셋(≥100 요청)에서 pre-RoPE Key 벡터를 수집 → k-means로 M개 코드워드 클러스터링.
     - 코드북 크기: 기본 `M=256` (8비트 인덱스), 잔차 코드북 수: `n_residuals=4`.
     - 한 번만 학습하면 모든 미래 캐시에 재사용 (VQKV 원리).
  3. RSimVQ 인코딩:
     - Key 벡터에서 RoPE 적용 전 표현 복원 (가역적 RoPE 역변환으로 pre-RoPE 접근).
     - pre-RoPE 공간에서 잔차 VQ 인코딩: `code_1 = nearest_codeword(k_pre_rope)`, `code_2 = nearest_codeword(k_pre_rope - decode(code_1))`, ..., `code_n`.
     - 저장: 정수 인덱스 배열 (FP16 대비 ~12.5% 크기).
  4. 온-디맨드 재구성:
     - 슬라이딩 윈도우 내 최신 토큰만 로컬 FP16 캐시로 유지.
     - 오래된 토큰은 on-demand 재구성: `decode(codes) → k_pre_rope → apply_rope(pos) → k`.
  5. Value 코드북: Value 벡터는 RoPE 미적용이므로 표준 RSimVQ 적용 (pre-RoPE 변환 불필요).
  6. eOptShrinkQCodec(05-08 기구현)과의 비교:
     - eOptShrinkQCodec이 BBP 위상전이 자동 랭크 선택 + TurboQuant 잔차 양자화(스칼라)인 반면, RSimVQCodec은 벡터 단위 코드북 양자화(VQ)로 스칼라 양자화 대비 높은 재구성 충실도를 동일 비트수에서 달성한다. 두 기법은 직교하므로 결합 가능(저랭크 공유 성분은 eOptShrinkQ, 잔차는 RSimVQ).
  7. TriAttentionCodec(05-06 기구현)과의 비교:
     - TriAttentionCodec이 pre-RoPE 삼각함수 시리즈로 중요도 추정 + 퇴거에 집중한 반면, RSimVQCodec은 퇴거 없이 모든 토큰을 고압축으로 저장해 정확도를 유지한다.
- **예상 효과**: 메모리 −82.8% (5.8× 감소, VQKV arXiv 2603.16435 실증), 동일 메모리에서 4.3× 긴 컨텍스트 지원, accuracy delta ±1.4% (LongBench 98.6% 성능 유지)
- **accuracy-preserving 근거**: VQKV(arXiv 2603.16435)가 LLaMA3.1-8B에서 82.8% 압축률로 LongBench 98.6% 성능 유지를 실증. 훈련 불필요(training-free)로 즉시 배포 가능. pre-RoPE 공간 코드북 학습이 Key RoPE 이질성 문제를 근본 해소해 post-RoPE 공간 VQ 대비 재구성 오류 감소. 잔차 VQ의 다중 코드북이 단일 코드북 대비 재구성 충실도를 단계적으로 향상. 검증: LongBench 8개 서브태스크 + WikiText-2 perplexity ±1% + 4.3× 컨텍스트 확장 시 perplexity 측정.
- **구현 난이도**: medium
- **우선순위**: 1

### 아이디어 C-2: TemporalTieredKVManager — 시간적 거리 기반 HBM/DRAM 계층화 + 블록-단위 스트리밍 어텐션 I/O-연산 오버랩
- **가설**: KV 캐시 토큰 중요도가 시간적 거리에 따라 비대칭적으로 변화한다(최근 = 단기 기억, 생성에 더 중요; 오래된 = 장기 기억, 소수만 현재 쿼리와 관련)는 관찰에 기반해, "최신 KV → HBM 고정밀도, 오래된 KV → DRAM 저정밀도(Q4) + 블록-단위 스트리밍 어텐션으로 저속 계층 접근 시 I/O-연산 오버랩"을 구현하면, 이전 사이클의 NVLinkCompressedTierScheduler(피어 GPU HBM 계층, 피어-간 공간 분리)와 TailorKVManager(레이어별 오프로딩+양자화 혼합, 레이어 특성 기반)와 달리 "시간적 거리라는 새로운 계층화 기준 + 저속 계층 I/O-연산 오버랩"이라는 새로운 메커니즘으로 128K 컨텍스트에서 크로스-계층 트래픽을 5.94× 감소시킬 수 있다.
- **접근 방법**:
  1. `TemporalTieredKVManager` 클래스 (`src/cache/temporal_tiered_kv.py`): `CacheStore` 인터페이스 상속
  2. 시간적 계층 정책:
     - HBM 계층: 최근 `recent_window` 토큰 (기본 2048). FP16 고정밀도. 직접 어텐션.
     - DRAM 계층: `recent_window` ~ `context_length` 토큰. Q4(4비트) 양자화. 블록-단위 스트리밍 어텐션으로 접근.
  3. 블록-단위 스트리밍 어텐션:
     - DRAM KV를 `block_size` (기본 512 토큰) 단위로 청크화.
     - GPU가 현재 블록 어텐션 연산 중 CPU가 다음 블록을 DRAM→GPU로 비동기 전송 (I/O-연산 오버랩).
     - `stream_attention(query, kv_blocks, device='cuda', dram_transfer_stream=secondary_stream)`.
  4. 동적 `recent_window` 조정: 현재 HBM 사용률이 임계값 초과 시 `recent_window` 자동 축소 → HBM 계층을 DRAM으로 강등.
  5. CacheFlow3DParallelRestoreScheduler(A-1)와의 통합: TTKV 계층 정보를 복원 우선순위에 반영. HBM 계층 KV = 복원 비용 낮음(한계편익 높음), DRAM 계층 KV = 복원 비용 높음(한계편익 낮음).
  6. ThreeTierPersistentKVSegmentCache(B-2)와의 통합: B-2의 Tier 0(HBM)/Tier 1(DRAM)/Tier 2(디스크) 3-티어와 C-2의 HBM/DRAM 2-티어를 통합하면 4-티어 메모리 계층(HBM FP16 / DRAM Q4 / DRAM 영속 Q4 / 디스크 Q4)으로 확장 가능.
  7. 이전 C 기법과의 차별점:
     - NVLinkCompressedTierScheduler(05-04)가 피어 GPU HBM을 중간 계층으로 사용(공간적 분리)한 반면, TemporalTieredKVManager는 단일 노드에서 시간적 거리 기반으로 HBM/DRAM을 분리(시간적 분리)한다.
     - TailorKVManager(05-05)가 레이어 entropy 기반 오프로딩/양자화 혼합인 반면, TemporalTieredKVManager는 토큰의 시간적 나이(age) 기반 계층화로 설계 원칙이 완전히 다르다.
- **예상 효과**: 크로스-계층 트래픽 −5.94× (TTKV arXiv 2604.19769 실증, 128K 컨텍스트), 레이턴시 −76%, 처리량 +2× (128K 기준), 유효 컨텍스트 길이 2× 이상 확장
- **accuracy-preserving 근거**: TTKV(arXiv 2604.19769)가 128K 컨텍스트 태스크에서 레이턴시 76% 단축, 처리량 2× 향상을 실증하면서 정확도를 유지. 최신 토큰(단기 기억)을 HBM 고정밀도로 유지해 생성 품질 보존. 오래된 토큰(장기 기억)을 DRAM Q4로 저장하되 블록-단위 스트리밍으로 어텐션 계산에 포함해 장기 의존성 손실 방지. 검증: LongBench 128K 서브태스크 + WikiText-2 perplexity ±1% + RULER 128K 컨텍스트.
- **구현 난이도**: medium
- **우선순위**: 2

### 아이디어 C-3: PCAEntropyKVTC — PCA 기저 1회 계산 + DP 최적 비트할당 + DEFLATE 엔트로피 코딩 3단계 KV 변환 압축
- **가설**: KV 캐시 압축에서 스칼라 양자화(eOptShrinkQ, TurboQuant, NQKVCodec 등)와 저랭크 근사(eOptShrinkQCodec)가 모두 "원본 KV 공간의 상관성을 제거하지 않은 채 압축"하는 공통 한계를 가지는 문제를, "PCA 기반 특성 비상관화(교정 데이터 1회) + 동적 프로그래밍 최적 비트할당(잔여 재구성 오류 최소화) + DEFLATE 엔트로피 코딩(심볼 패킹)"의 3단계 변환 코딩 파이프라인을 적용하면, 이전 사이클의 어떤 C 기법도 시도하지 않은 "정보 이론적 최적에 가까운 완전 파이프라인"으로 20× 압축 + ±1% 정확도 이내를 달성할 수 있다.
- **접근 방법**:
  1. `PCAEntropyKVTC` 클래스 (`src/cache/pca_entropy_kvtc.py`): `CompressionCodec` 인터페이스 구현
  2. Stage 1 — PCA 기반 비상관화:
     - 교정 데이터셋에서 단 1회 PCA 기저 행렬 V 계산: `torch.linalg.svd(kv_calibration_matrix)`.
     - 모든 미래 KV에 동일 기저 재사용: `kv_decorrelated = kv @ V`. 프롬프트별 별도 분해 없음 → 추론 오버헤드 거의 없음.
     - 레이어별 기저 행렬 저장: `configs/pca_basis_layer_{l}.pt`.
  3. Stage 2 — DP 최적 비트할당:
     - 동적 프로그래밍으로 총 비트 예산 `B_total`을 d_kv 차원에 최적 배분. 목표: 잔여 재구성 오류 최소화.
     - `dp_bit_allocate(variances, B_total) → bit_widths[]`. 후반 주성분은 종종 0비트 할당 → 자동 차원 감소.
     - 비트 범위: 0~8비트. 비트폭 0 = 해당 주성분 저장 생략 (PCA 자동 조기 차원 절단).
  4. Stage 3 — DEFLATE 엔트로피 코딩:
     - DP 할당된 양자화 심볼에 DEFLATE 알고리즘 적용 (Python `zlib` 또는 C extension 활용).
     - 추가 압축으로 최종 저장 크기를 DP 양자화 대비 추가 감소.
  5. eOptShrinkQCodec(05-08 기구현)과의 결합:
     - eOptShrinkQCodec이 BBP 위상전이로 저랭크 공유 성분을 추출 → PCAEntropyKVTC가 잔차를 추가 압축.
     - 결합 파이프라인: `eOptShrinkQ(kv) → [shared_component + residual]` → `PCAEntropyKVTC(residual)`.
     - 두 기법의 직렬 결합이 단독 적용 대비 추가 압축률 달성.
  6. 이전 C 기법과의 차별점:
     - eOptShrinkQCodec(05-08)이 BBP 위상전이 자동 랭크 선택 + TurboQuant 잔차 양자화(스칼라 방식)인 반면, PCAEntropyKVTC는 PCA 비상관화 + DP 비트할당 + 엔트로피 코딩이라는 완전히 다른 3단계 파이프라인이다.
     - KVTC(ICLR 2026, arXiv 2511.01815)의 설계 원칙을 Activity C 구현 프레임워크에 통합한 최초 구현.
- **예상 효과**: 메모리 −95% (20× 압축, KVTC ICLR 2026 실증), TTFT −8× (장문 KV 오프로딩 대비, KVTC 실증), accuracy delta ±1% 이내 (AIME25/GSM8K/MMLU/LongBench 포함 6개 벤치마크)
- **accuracy-preserving 근거**: KVTC(arXiv 2511.01815/ICLR 2026, NVIDIA)가 20× 압축 시 AIME25, GSM8K, LiveCodeBench, LongBench, MATH-500, MMLU에서 기준 모델 대비 점수 1점 이내 정확도 유지를 실증. 특정 유스케이스에서 40×+ 압축 달성. Llama3, Mistral NeMo, R1-Qwen2.5에서 검증. PCA 비상관화가 KV 차원 간 상관성을 제거해 양자화 오류 분산을 줄이고, DP 비트할당이 중요 주성분에 더 많은 비트를 자동 배분해 재구성 충실도를 이론적 최적으로 유지. 검증: WikiText-2 perplexity ±1% + LongBench + AIME25 정확도 + 압축률 vs 정확도 트레이드오프 곡선.
- **구현 난이도**: medium
- **우선순위**: 3

## 크로스 Activity 시너지 아이디어

### 아이디어 Cross-1: B+C — KVPacketSoftAdapterCache + RSimVQCodec (재계산-프리 맥락 독립 + 고압축 VQ 저장)
- **조합 근거**: KVPacketSoftAdapterCache(B-1)의 소프트-토큰 어댑터가 맥락 의존성을 해소한 후, RSimVQCodec(C-1)의 VQ 압축을 각 KV 패킷에 적용하면 "맥락 독립 + 고압축" 비연속 세그먼트 저장이 실현된다. 어댑터 크기는 미미(<1%)하므로 패킷 전체를 82.8% 압축해도 어댑터 정보를 손실 없이 보존할 수 있다. 트렌드 리포트의 B+C 시너지("KV Packet + VQKV 결합")를 직접 구현한다.
- **구체 설계**:
  1. `ContextFreeCompressedKVPacket` 모듈 (`src/cache/context_free_compressed_packet.py`)
  2. 패킷 생성: `KVPacketSoftAdapterCache.create_packet(doc_kv)` → 어댑터-래핑 KV 패킷
  3. VQ 압축: `RSimVQCodec.encode(kv_packet.kv_block)` → pre-RoPE 공간 RSimVQ 인코딩 적용. 어댑터 텐서는 FP16 원본 유지 (크기 << kv_block).
  4. 저장: `(adapter_fp16, kv_rsimvq_codes, codebook_id)` 3-tuple로 저장. 어댑터 정보 손실 없음.
  5. 재사용: 검색된 KV 패킷의 VQ 코드를 on-demand 재구성 → 어댑터 적용 → 맥락 독립 어텐션 계산.
  6. `TriangleInequalitySegmentIndex`(05-09 기구현)와 통합: VQ 압축된 패킷도 임베딩(어댑터 포함 평균 벡터)으로 인덱싱 → O(log N) 검색.
  7. eOptShrinkQCodec(05-08 기구현)과의 선택적 결합: 어댑터 래핑 후 RSimVQ 대신 eOptShrinkQ 적용도 가능. 코덱 파라미터로 선택.
- **예상 복합 효과**: 처리량 +40~55%, 메모리 −82~88% (RSimVQ 82.8% 압축 + 어댑터 경량), 비연속 히트율 +30~40%p (맥락 독립으로 임의 맥락 재사용), 재계산 FLOPs ~0%, accuracy delta ±1% 이내
- **구현 난이도**: medium
- **우선순위**: 1

### 아이디어 Cross-2: B+C — ThreeTierPersistentKVSegmentCache + TemporalTieredKVManager (에이전트 KV 4-티어 시간 기반 통합 계층화)
- **조합 근거**: ThreeTierPersistentKVSegmentCache(B-2)의 HBM/DRAM/디스크 3-티어와 TemporalTieredKVManager(C-2)의 HBM/DRAM 시간 기반 2-티어를 통합하면, "HBM FP16 → DRAM Q4(블록 스트리밍) → DRAM Q4 영속(safetensors) → 디스크 Q4"의 4-티어 에이전트 KV 메모리 계층이 구성된다. 시간적 거리 기준 자동 강등/승격과 서버 재시작 후 복원이 결합된 완전한 에이전트 메모리 관리 시스템.
- **구체 설계**:
  1. `FourTierAgentKVHierarchy` 모듈 (`src/cache/four_tier_agent_kv.py`)
  2. 4-티어 매핑:
     - Tier 0: HBM FP16 — 활성 에이전트, 최신 N 스텝 이내. 직접 어텐션.
     - Tier 1: DRAM Q4 (인메모리) — N~M 스텝 전. 블록 스트리밍 어텐션 (TemporalTieredKVManager 방식).
     - Tier 2: DRAM Q4 영속 (safetensors, 메모리 맵) — M 스텝 이전. 메모리 맵 파일로 직접 어텐션 주입.
     - Tier 3: 디스크 Q4 (safetensors 파일) — 비활성 에이전트. 재활성화 시 Tier 2→1로 승격.
  3. 자동 계층 이동: `time_since_last_access` 기준 자동 강등. 접근 시 자동 승격.
  4. 블록 스트리밍 어텐션 (Tier 1→0 전환 중): TemporalTieredKVManager의 I/O-연산 오버랩 적용.
  5. 서버 재시작 복원: Tier 2/3는 파일 기반이므로 재시작 후 즉시 복원 가능.
- **예상 복합 효과**: 처리량 +30~45%, 메모리 −70~85% (4-티어 분산), 에이전트 재프리필 지연 제거 (Tier 2→1 직접 주입), 비연속 히트율 +25~35%p, accuracy delta ±1% 이내
- **구현 난이도**: medium
- **우선순위**: 2

### 아이디어 Cross-3: A+B+C — CacheFlow3DParallelRestoreScheduler + KVPacketSoftAdapterCache + RSimVQCodec + TemporalTieredKVManager (RAG 서빙 완전 최적화 파이프라인)
- **조합 근거**: 트렌드 리포트의 A+B+C 통합 시너지 포인트를 직접 구현한다. CacheFlow(A-1)가 비연속 세그먼트를 3D 병렬로 복원하고, KVPacketSoftAdapterCache(B-1)가 재계산 없이 맥락 독립 패킷을 조합하며, RSimVQCodec(C-1)이 VQ 압축으로 저장 효율을 극대화하고, TemporalTieredKVManager(C-2)가 자주 재사용되는 패킷은 HBM에, 드물게 재사용되는 패킷은 DRAM Q4에 자동 배치하는 완전 통합 RAG 파이프라인.
- **구체 설계**:
  1. RAG 요청 수신 → TemporalTieredKVManager가 검색된 K개 문서의 계층 확인 (HBM/DRAM)
  2. HBM 계층 패킷: 즉시 KVPacketSoftAdapterCache로 조합 (어댑터 적용만, 복원 없음)
  3. DRAM 계층 패킷: CacheFlow3DParallelRestoreScheduler가 3D 병렬로 DRAM→HBM 복원. 레이어-축 병렬화로 복원 중 어텐션 시작 가능.
  4. 새 문서 캐싱: RSimVQCodec으로 인코딩 → 어댑터 래핑 → TemporalTieredKVManager에 Tier 0(HBM) 등록 → 접근 빈도에 따라 자동 계층 이동.
  5. 세그먼트 인덱스: TriangleInequalitySegmentIndex(05-09 기구현) + VQ 압축 패킷의 임베딩으로 O(log N) 검색.
  6. 추측 디코딩 통합: SpecKVCompressionGammaController(05-09 기구현)가 RSimVQCodec 압축 수준(~12.5% = ~3비트 등가)에 최적 γ를 자동 선택.
- **예상 복합 효과**: 처리량 +50~70%, 메모리 −82~90%, 비연속 히트율 +35~50%p (전체 히트의 70%+ 목표), TTFT −60~75% (3D 병렬 복원 + 재계산 프리), accuracy delta ±1% 이내
- **구현 난이도**: high
- **우선순위**: 2

### 아이디어 Cross-4: A+C — ValveOnlineOfflineKVReclaimScheduler + RSimVQCodec (선점-VQ 압축 온-오프라인 메모리 공유)
- **조합 근거**: ValveOnlineOfflineKVReclaimScheduler(A-3)가 선점된 온라인 요청의 KV 메모리를 오프라인 배치에 재할당할 때, RSimVQCodec(C-1)으로 선점 KV를 즉시 VQ 압축하면 오프라인 배치에 재할당되는 메모리 크기가 5.8× 감소한다. 오프라인 배치 작업을 위한 유효 메모리가 대폭 증가하고, 온라인 요청 재개 시 압축 해제 후 어댑터 주입으로 TTFT를 최소화한다.
- **구체 설계**:
  1. `VQCompressedOnlineOfflineReclaim` 모듈 (`src/scheduler/vq_online_offline_reclaim.py`)
  2. 선점 결정 시: `RSimVQCodec.encode(kv_to_reclaim)` → VQ 코드로 5.8× 축소
  3. 오프라인 배치에 재할당되는 메모리: 원본 FP16 대신 VQ 코드 크기(~17% 수준)
  4. 재개 시: `RSimVQCodec.decode(kv_codes)` → FP16 복원 → 어텐션 계산
  5. 압축-재할당 파이프라인 오버랩: CUDA 스트림 분리로 RSimVQ 압축(compute stream)과 메모리 재매핑(memory stream) 병렬 실행
  6. Valve 설계 원칙 유지: 요청당 최대 1회 선점, <1ms 선점 지연 제약 충족 확인.
- **예상 복합 효과**: 처리량 +40~55%, GPU 이용률 +34.6% (Valve) + VQ 압축으로 오프라인 메모리 5.8× 증가, 메모리 −82% (VQ), accuracy delta ±1% 이내
- **구현 난이도**: medium
- **우선순위**: 3

## 이번 사이클 최우선 구현 타겟

**Cross-1(B+C): KVPacketSoftAdapterCache(B-1) + RSimVQCodec(C-1) + ContextFreeCompressedKVPacket 통합**, 보조로 TemporalTieredKVManager(C-2), ThreeTierPersistentKVSegmentCache(B-2), CacheFlow3DParallelRestoreScheduler(A-1) 구현

구현 순서:
1. **C-1** `RSimVQCodec`: pre-RoPE 공간 k-means 코드북 학습 + 잔차 VQ 인코딩/디코딩 + on-demand 재구성. `CompressionCodec` 인터페이스. **accuracy-preserving 검증 필수**: WikiText-2 perplexity ±1% + LongBench 8개 서브태스크 + 압축률 vs 정확도 곡선(코드북 크기 M=64/128/256 별).
2. **B-1** `KVPacketSoftAdapterCache`: 소프트-토큰 어댑터 구조 + 자기지도 증류 훈련 파이프라인 + 단순 연결(concatenation) 조합. `CacheStore` 구현. **재계산 없는 F1 동등성 검증 필수**: 어댑터 ON/OFF 비교 + F1/perplexity 측정.
3. **Cross-1** `ContextFreeCompressedKVPacket`: B-1 + C-1 통합. 패킷별 VQ 인코딩 + 어댑터 분리 저장 + TriangleInequalitySegmentIndex(05-09 기구현) 통합.
4. **C-2** `TemporalTieredKVManager`: 시간적 거리 기반 HBM/DRAM 계층화 + 블록-단위 스트리밍 어텐션 I/O 오버랩. `CacheStore` 구현. **128K 컨텍스트 처리량 + 트래픽 감소 측정 필수**.
5. **B-2** `ThreeTierPersistentKVSegmentCache`: HBM/DRAM/디스크 3-티어 + Q4 직접 주입 + safetensors 영속화. 서버 재시작 후 복원 테스트 포함.
6. **A-1** `CacheFlow3DParallelRestoreScheduler`: 3D 병렬화 추상화 + 배치-인식 두-포인터 스케줄러. ContiguousChunkIOScheduler(05-06 기구현)와 청크 크기 정렬.

이유:
- RSimVQCodec은 훈련 불필요로 즉시 배포 가능하며, 82.8% 압축 + LongBench 98.6% 성능이라는 Activity C 목표 −30%를 10× 초과하는 가장 강력한 단일 기법이다. eOptShrinkQCodec(05-08 기구현)과 직교해 결합 가능하다.
- KVPacketSoftAdapterCache는 이전 사이클까지의 모든 B 기법의 공통 한계("선택적 재계산 의존")를 "재계산 제로"로 돌파하는 패러다임 전환 기법이다. 비연속 히트율 ≥30% 목표 달성에 가장 직접 기여한다.
- Cross-1 통합(B+C)이 재계산-프리 맥락 독립 + 고압축 저장의 상승 효과로 처리량 +40~55% + 메모리 −82~88%를 동시 달성해 연구 목표 지표를 폭넓게 커버한다.
- TemporalTieredKVManager는 이전 사이클의 NVLinkCompressedTierScheduler(05-04), TailorKVManager(05-05)와 계층화 기준이 완전히 달라 새 Activity C 기둥을 형성하며, 128K 컨텍스트 지원 목표에 직접 기여한다.
