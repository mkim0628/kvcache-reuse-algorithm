## 변화 요약
1. Tutti가 CPU를 I/O 경로에서 완전 제거하는 GPU 중심 KV 객체 저장소 패러다임을 도입하여 기존 스케줄링 가정과 근본적으로 다른 새 알고리즘 범주를 열었다.
2. MixedDimKV는 기존 이진(evict/keep) 패러다임을 깨고 토큰별 연속 차원 예산 할당을 처음 제안하며, Louver는 halfspace range search라는 수학적으로 새로운 KV 인덱스 정식화를 도입했다(zero false negative 보장).
3. Sequential PLT는 모델 자신의 예측을 참조로 쓰는 예측적 델타 코딩으로 Shannon 한계 이하의 압축을 달성, 기존 어느 사이클에서도 시도되지 않은 정보이론적 접근이다.

## Activity A 아이디어

### 아이디어 A-1: TuttiGPUCentricSSDScheduler
- **가설**: CPU를 KV I/O 경로에서 완전히 배제하고 GPU io_uring으로 직접 SSD KV 블록을 pull하면, CPU 대기 레이턴시가 제거되어 TTFT를 −78% 이상 단축하고 처리량을 2× 향상시킬 수 있다.
- **접근 방법**: GPU-native KV 객체 저장소(Tutti 아키텍처 기반)를 구현한다. 스케줄러는 각 디코드 스텝의 slack(idle GPU bubble)을 측정하고, slack이 SSD I/O 예상 지연보다 크면 비동기 GPU io_uring prefetch를 트리거한다. KV 블록은 CPU DRAM을 경유하지 않고 SSD → GPU HBM으로 직송된다. 비연속 세그먼트 로딩을 위한 scatter-gather GPU DMA 연산을 지원하며, 실패 시 재계산 fallback을 유지한다.
- **대상 환경**: 단일 노드 (NVMe SSD + GPU 직결 구성)
- **멀티 노드 고려사항**: 해당 없음 (단일 노드 SSD 오프로드에 집중)
- **예상 효과**: TTFT p50 −50~70%, 처리량 +80~120%, CPU 개입으로 인한 스케줄링 오버헤드 제거
- **구현 난이도**: high
- **우선순위**: 2

### 아이디어 A-2: SparKVHybridChunkRouter
- **가설**: 경량 MLP가 어텐션 희소성 패턴을 보고 청크별로 "클라우드 스트리밍 vs 로컬 재계산" 여부를 동적으로 결정하면, 무선 네트워크 대역폭과 엣지 GPU 헤드룸 변동에 강인한 cloud-edge KV 로딩 전략을 실현할 수 있다.
- **접근 방법**: 각 청크의 어텐션 희소성 통계(top-k 토큰 집중도)를 특징으로 쓰는 2층 MLP 예측기를 학습시킨다. 런타임 어댑티브 컨트롤러가 무선 처리량(EMA)과 엣지 GPU 현재 부하를 모니터링하여 MLP 예측 임계값을 동적으로 조정한다. 캐시 히트가 발생한 청크는 스트리밍 우선, 히트 미발생 청크는 로컬 재계산으로 라우팅한다.
- **대상 환경**: 멀티 노드 (클라우드-엣지 분리 구성)
- **멀티 노드 고려사항**: 무선 링크 대역폭 변동 추적 필요; 클라우드→엣지 KV 전달 비용 모델링; 네트워크 단절 시 자동 로컬 재계산 fallback
- **예상 효과**: TTFT 1.3~5.1× 단축(네트워크 조건에 따라), 엣지 재계산 비용 −40%
- **구현 난이도**: high
- **우선순위**: 3

## Activity B 아이디어

### 아이디어 B-1: RoPEReencodingNonContiguousCache
- **가설**: RoPE의 상대 위치 불변 특성(회전 행렬 합성 가능성)을 이용해 비연속 KV 세그먼트를 임의 순서로 무손실·비용 제로로 재조합하면, prefix 불일치 요청에서도 세그먼트 캐시 히트를 정확도 손실 없이 활용할 수 있다.
- **접근 방법**: 캐싱 시 KV를 RoPE 적용 전 상태로 저장한다(position-decoupled KV store). 재사용 시 목표 위치에 맞는 회전 행렬을 GPU에서 in-place로 적용하여 재인코딩한다. AdapShot의 엔트로피 기반 probe 선택을 확장해 Many-Shot ICL 외 일반 대화 컨텍스트에도 적용한다. 세그먼트 인덱스는 content hash(위치 무관)로 키를 설정한다.
- **예상 효과**: 비연속 히트율 +25~40%(position 불일치로 놓치던 히트 복구), TTFT ±3% 이내(재인코딩 비용 미미)
- **구현 난이도**: medium
- **우선순위**: 1

### 아이디어 B-2: HalfspaceRangeSearchKVIndex
- **가설**: KV 검색 문제를 반공간(halfspace) range search 문제로 정식화하면 zero false negative(중요 KV 누락 없음)를 수학적으로 보장하면서 dense attention보다 빠른 인덱싱이 가능하다.
- **접근 방법**: Louver 논문의 반공간 range search 정식화를 구현한다. 각 KV 벡터를 d차원 반공간으로 표현하고, 쿼리를 반공간 교집합 쿼리로 변환한다. CPU+GPU 하이브리드 인덱스 구조로 구성: CPU에서 반공간 인덱스 탐색 → GPU에서 후보 KV 어텐션 계산. False negative 없음 보장이 핵심이므로 precision은 희생 가능하나 recall은 100% 유지한다.
- **예상 효과**: 비연속 세그먼트 검색 속도 FlashAttention 대비 +20~40%, zero false negative로 정확도 유지
- **구현 난이도**: high
- **우선순위**: 2

## Activity C 아이디어

### 아이디어 C-1: MixedDimPerTokenBudgetCodec
- **가설**: 토큰별로 연속적인 차원 예산(0~전체 dim)을 손실 점수 기반으로 할당하면, 이진 evict/keep 방식보다 동일 메모리 예산에서 더 높은 정확도를 보존할 수 있다.
- **접근 방법**: 각 토큰의 손실 점수 = 어텐션 중요도 × 값 벡터 크기 × 차원별 압축성(PCA 분산 비율)의 가중 곱으로 정의한다. 레이어 전체 메모리 예산 B를 제약 조건으로 bisection search를 수행해 손실 점수 임계값 λ*를 결정한다. 각 토큰은 λ* 이하 손실 점수 차원만 보관하며 나머지는 드롭한다. 훈련 없이 추론 시 즉시 적용 가능.
- **예상 효과**: 메모리 −45~55%, 디코드 레이턴시 −55%(A100 기준), KV 메모리 점유 −50%
- **accuracy-preserving 근거**: 고중요도 토큰(어텐션 집중)은 압축성 점수가 낮아 전체 차원 보존; 압축 대상은 어텐션 가중치가 낮고 값 크기도 작은 토큰의 저분산 차원에 한정됨. 이론적으로 어텐션 출력 변화량 ε = O(λ* · √dim)이고 λ*를 perplexity ±1% 기준으로 보정 가능.
- **구현 난이도**: medium
- **우선순위**: 1

### 아이디어 C-2: SequentialPLTCrossSessionCodec
- **가설**: 세션 간 반복 프리픽스를 Probabilistic Language Trie로 중복 제거하고, 잔차(residual)를 모델 자신의 예측으로 예측적 델타 코딩하면 Shannon 한계 이하의 KV 캐시 압축률을 달성할 수 있다.
- **접근 방법**: Cross-session 공통 프리픽스를 PLT(Probabilistic Language Trie)에 인덱싱한다. 캐시 저장 시 모델 다음 토큰 분포를 참조 신호로 사용해 KV 벡터 잔차를 계산하고 잔차만 저장한다(predictive delta coding). 복원 시 PLT 공통 노드 KV + 잔차 KV를 합산한다. 잔차는 모델이 거의 예측 가능한 토큰에서 극히 작아지므로 엔트로피가 Shannon 한계 이하로 내려간다.
- **예상 효과**: 세션 간 KV 중복 저장 제거 −30~50%, 잔차 압축 추가 −20%, 전체 KV 스토리지 −40~60%
- **accuracy-preserving 근거**: PLT 공통 노드는 bit-exact KV를 보존하므로 공통 경로 오차 없음. 잔차 저장은 손실 없는 delta coding이므로 복원 KV는 원본과 동일(lossless). 정보 손실이 없으므로 정확도 변화 0%.
- **구현 난이도**: high
- **우선순위**: 3

### 아이디어 C-3: MISAMoEIndexerAccelerator
- **가설**: DSA(DeepSeek Sparse Attention)의 인덱서 헤드를 MoE 라우터로 선택적으로 활성화하면, 인덱서 연산 비용을 크게 줄이면서도 핵심 헤드 전문성을 보존하여 long context 정확도를 유지할 수 있다.
- **접근 방법**: DSA 인덱서 헤드 집합 위에 경량 MoE 라우터를 추가한다. 라우터는 블록 수준 어텐션 통계(최대 어텐션 값, 희소성 비율)를 특징으로 쿼리당 top-k 인덱서 헤드를 선택한다. 비선택 헤드는 해당 쿼리에 대해 비활성화된다. 라우터 학습: 원래 full activation 결과를 교사 신호로 distillation.
- **예상 효과**: 인덱서 헤드 계산량 −60~80%(top-2/8 선택 시), long context TTFT −30%, accuracy delta ±0.5%
- **accuracy-preserving 근거**: (1) 원본 인덱서 헤드 풀 전체를 보존하므로 헤드 자체는 손실 없음; (2) MoE 라우터는 distillation으로 full activation 결과를 근사하도록 학습되어 활성화 패턴 차이가 최소화됨; (3) top-k를 k=2 이상으로 설정 시 coverage가 충분하여 perplexity 변화 ±1% 이내 실험적 확인 가능.
- **구현 난이도**: high
- **우선순위**: 4

- **예상 복합 효과**: 처리량 +120%, 메모리 −55%, 정확도 ±1%, 유효 컨텍스트 2~3×

## 이번 사이클 최우선 구현 타겟
**Cross-2 (AdapShotMixedDimSegmentPipeline, B+C)**와 **B-1 (RoPEReencodingNonContiguousCache)**를 우선 구현할 것을 권장한다.

이유:
1. B-1은 구현 난이도 medium이며 즉시 비연속 히트율을 대폭 향상시킬 수 있는 핵심 메커니즘이다.
2. C-1(MixedDimPerTokenBudgetCodec)은 훈련 없이 적용 가능하고 −55% 디코드 레이턴시라는 강력한 효과가 있다.
3. Cross-2는 B-1과 C-1의 조합으로 구현 재사용률이 높아 추가 공수가 작다.
4. C-2(SequentialPLT)와 A-1(TuttiGPU)은 시스템 수준 변경이 크므로 이후 사이클에서 순차 구현 권장.
