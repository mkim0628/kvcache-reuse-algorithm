# 트렌드 리포트 — 2026-05-08

## 전체 요약

1. **Activity A (스케줄링)**: 선점형(preemptive) KV 캐시 관리가 새로운 실용화 단계에 진입했다. TokenFlow(EuroSys 2026)는 요청 선점 + 능동적 KV 전송으로 P99 TTFT를 80.2% 감소시키고 유효 처리량을 82.5% 향상했다. GORGO는 지역(region) 간 KV 재사용과 네트워크 지연을 공동 최적화하는 첫 크로스-리전 라우터로 중앙화 GORGO-proxy 기준 중간 TTFT 2.5× 향상을 달성했다. Fluid-Guided Online Scheduling은 유체역학 근사로 KV 캐시 동적 성장을 형식화하고, 퇴거 없이 근최적 처리량을 달성하는 WAIT 알고리즘을 도출했다.

2. **Activity B (비연속 재사용)**: GORGO의 크로스-리전 최적화가 "어느 노드/리전에서 KV를 재사용할 것인가"라는 Activity B의 멀티-노드 비연속 재사용 문제의 지리적 확장이다. KEEP은 구체화 에이전트 시나리오에서 메모리 세그먼트 갱신 후 재계산이 필요한 비연속 KV 문제를 정적-동적 분리 구조로 해결한다.

3. **Activity C (압축)**: 스펙트럼(spectral) 기반 및 잔차(residual) 기반 새 압축 패러다임이 등장했다. eOptShrinkQ(2026-05)는 랜덤 행렬 이론으로 KV 캐시의 저랭크 공유 성분을 자동 추출 후 잔차를 TurboQuant로 압축하는 이중 파이프라인으로 TurboQuant 3.0비트 대비 2.2비트에서 동등 성능을 달성한다. DeltaKV는 장거리 유사 토큰에 대한 잔차 인코딩으로 KV 메모리를 원본의 29%로 압축하면서 근무손실 정확도를 유지한다. DASH-KV(ACL 2026)는 비대칭 딥 해싱으로 어텐션 복잡도를 O(N²)에서 O(N)으로 낮춘다.

4. **A+B 시너지**: GORGO의 네트워크-인식 크로스-리전 라우팅은 DualMap(이미 보고)의 이중 해시 라우팅에 지리적 네트워크 지연 차원을 추가한 직접적 확장이다.

5. **B+C 시너지**: DeltaKV의 잔차-기반 압축이 Activity B의 비연속 세그먼트 저장에 직접 적용 가능 — 공통 세그먼트를 기저로, 차이를 잔차로 인코딩하는 B+C 통합 저장 구조.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [TokenFlow: Responsive LLM Text Streaming Serving under Request Burst via Preemptive Scheduling](https://arxiv.org/abs/2510.02758)
- **출처**: arXiv 2510.02758 / EuroSys 2026 (April 27–30, Edinburgh)
- **핵심 아이디어**: 표준 LLM 서빙 시스템의 비선점형(non-preemptive) 요청 스케줄링과 반응적(reactive) KV 캐시 관리가 요청 폭주(burst) 시 낮은 자원 이용률과 높은 TTFT를 유발하는 문제를 해결한다. 실시간 토큰 버퍼 점유율과 소비율(token consumption rate)을 기준으로 요청을 동적으로 선점하고, GPU-CPU 간 KV 캐시 전송을 백그라운드에서 능동적으로 실행하여 I/O와 컴퓨팅을 오버랩한다. 선점 오버헤드를 최소화하면서 스트리밍 텍스트 생성의 응답성(responsiveness)과 안정성(steady generation)을 동시에 보장한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Llama3-8B 및 Qwen2.5-32B에서 RTX 4090, A6000, H200 다중 GPU 환경으로 검증. 유효 처리량(실제 사용자 소비 고려) 최대 82.5% 향상, P99 TTFT 최대 80.2% 감소. EuroSys 2026 채택으로 시스템 커뮤니티에서 최고 수준 인정. Activity A의 "스케줄링 오버헤드 TTFT p50 +5% 이내" 목표를 선점형 접근으로 실현하는 새로운 레퍼런스. 기존 연속 배칭(continuous batching)의 head-of-line blocking 문제를 선점으로 근본 해소한다.

#### 2. [GORGO: Maximizing KV-Cache Reuse While Minimizing Network Latency in Cross-Region LLM Load Balancing](https://arxiv.org/abs/2602.11688)
- **출처**: arXiv 2602.11688 (2026-02-12)
- **핵심 아이디어**: 다중 지역(multi-region) 분산 LLM 추론에서 기존 멀티-리전 로드밸런서가 KV 캐시 히트율만 최적화하고 클러스터 간 네트워크 지연을 무시하는 문제를 해결한다. GORGO는 (i) 접두사 캐시 국소성, (ii) 실측 리전 간 지연, (iii) 순간 어드미션/큐 상태를 공동 최적화하는 per-region 로드밸런서로, 로컬 처리 가능 시 로컬로 라우팅하고, 원격 포워딩이 유리할 때만 선택적으로 전달한다. 저지연·저대역폭 관측성(observability)으로 피어 네트워크 메트릭을 모든 로드밸런서에 지속 전파한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 중앙화 GORGO-proxy 기준 중간 TTFT 2.5×, P99 TTFT 감소 달성. 기존 "최소 부하" 또는 "접두사 유사도" 단일 기준 라우팅 대비 현저한 우위. Activity A의 "네트워크-인식 KV 라우팅" 목표에서 InfiniBand/NVLink 클러스터 내 라우팅(DualPath, Harvest 등 이미 보고)을 넘어 지리적 멀티-클러스터 수준으로 문제 범위를 확장하는 첫 논문. Activity B의 비연속 세그먼트 재사용에 네트워크 지연을 추가 제약으로 통합하는 설계 방향을 제시한다.

#### 3. [Optimizing LLM Inference: Fluid-Guided Online Scheduling with Memory Constraints](https://arxiv.org/abs/2504.11320)
- **출처**: arXiv 2504.11320 (2025-04, v2 업데이트 2026, OpenReview 채택)
- **핵심 아이디어**: LLM 추론 최적화를 다단계 온라인 스케줄링 문제로 형식화한다. KV 캐시가 생성 중 동적으로 증가하고 메모리 오버플로우가 퇴거 연쇄(eviction cascade)로 이어지는 독특한 도전을 유체역학 근사로 해결한다. 유체 동역학 근사로 다루기 쉬운 벤치마크를 도출하고, 임계값 기반 배칭(threshold-based batching)으로 시스템을 부하 균형 근처에 유지해 퇴거 없이 근최적 처리량을 달성하는 WAIT(Waiting for Accumulated Inference Threshold) 알고리즘을 제안한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 출력 길이가 알려진 경우 근최적 처리량 달성. Llama2-7B A100 단일 GPU에서 vLLM·Sarathi 대비 평균 처리량 개선. GSA(이미 보고)의 경쟁비 보장 접근과 함께 KV 메모리 제약 스케줄링의 새로운 이론 기반 알고리즘군 형성. Activity A의 "캐시 퇴거율 최소화" 목표를 WAIT 임계값으로 제어하는 직접 수단을 제공한다.

#### 4. [KEEP: A KV-Cache-Centric Memory Management System for Efficient Embodied Planning](https://arxiv.org/abs/2602.23592)
- **출처**: arXiv 2602.23592 (2026-02-27, v2: 2026-03-17)
- **핵심 아이디어**: 구체화(embodied) 에이전트의 장기 계획에서 원시 텍스트로 메모리를 저장하면 프롬프트가 폭발적으로 길어지고 프리필 지연이 증가하는 문제를 KV 캐시를 메모리 저장 단위로 활용하여 해결한다. 에이전트가 행동 실행 후 메모리를 갱신할 때 갱신된 세그먼트 이후의 KV는 모두 무효화되어 재계산이 필요한 비연속 KV 문제를 Static-Dynamic Memory Construction 알고리즘으로 해결한다. 정적(공유) 메모리 세그먼트와 동적(갱신) 메모리 세그먼트를 혼합 세분성으로 분리 관리하고, Multi-hop Memory Reuse로 재계산 범위를 최소화한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 구체화 계획 벤치마크에서 기존 방법 대비 프리필 지연 대폭 감소 및 정확도 유지. Activity A+B 통합 관점에서 "에이전트 행동 → 메모리 갱신 → 비연속 KV 재사용" 사이클을 최초로 KV 캐시 수준에서 관리하는 시스템. SideQuest(이미 보고)의 자기-지시형 퇴거와 달리 에이전트 메모리 구조 자체를 KV 캐시 친화적으로 재설계한다.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [eOptShrinkQ: Near-Lossless KV Cache Compression Through Optimal Spectral Denoising and Quantization](https://arxiv.org/abs/2605.02905)
- **출처**: arXiv 2605.02905 (2026-04-06 제출, 2026-05 공개)
- **핵심 아이디어**: 트랜스포머 어텐션 헤드의 KV 캐시가 저랭크 공유 맥락 성분과 전랭크 토큰별 잔차로 분해된다는 스파이크 랜덤 행렬 모델(spiked random matrix model)에 기반하는 2단계 압축 파이프라인. 1단계: eOptShrink(최적 특이값 수축)가 BBP 위상전이(BBP phase transition)를 이용해 자동으로 랭크를 선택하고 공유 저랭크 성분을 추출한다. 2단계: 잔차를 TurboQuant(이미 보고)로 근최적 왜곡률로 양자화한다. 이론적으로 세 가지 보장을 제공한다: (1) BBP를 통한 자동 랭크 선택, (2) 잔차의 내적 편향 근-제로 보장, (3) 좌표 비국소화로 양자화 왜곡률 근최적 보장.
- **연구 목표와의 관련성**: high
- **주목할 점**: Llama-3.1-8B, Ministral-8B에서 LongBench 16개 태스크 검증. TurboQuant 3.0비트 대비 2.2비트에서 동등 또는 우수한 성능(entry당 약 0.8비트 절감). 멀티-니들 검색에서 2.2비트로 FP16 비압축 대비 동등한 수준. Activity C의 "정확도 보존(±1% 이내)" 제약과 "메모리 −30% 이상" 목표를 2.2비트라는 매우 공격적인 압축 비율에서 달성한다. 랜덤 행렬 이론 기반의 자동 랭크 선택이 Activity C 구현에서 레이어/헤드별 하이퍼파라미터 수동 조정을 제거하는 실용 방향을 제시한다.

#### 2. [DeltaKV: Residual-Based KV Cache Compression via Long-Range Similarity](https://arxiv.org/abs/2602.08005)
- **출처**: arXiv 2602.08005 (2026-02, Hugging Face Papers 수록; GitHub: CURRENTF/Sparse-vLLM 통합)
- **핵심 아이디어**: KV 캐시 표현에 장거리(long-range) 토큰 간 유사성이 존재한다는 관찰에 기반한다. 기존 방법들이 근거리(locality) 유사성에만 집중해 전역 유사성을 무시한 반면, DeltaKV는 전역에서 검색된 참조 토큰에 대한 토큰-단위 잔차를 인코딩해 의미 충실도를 유지하면서 저장량을 대폭 줄인다. 희소 어텐션(OmniKV 등)과 통합 시 선택된 토큰만 복원하는 온-디맨드 재구성으로 처리 효율을 극대화한다. 훈련 목적함수는 MSE + NTP 하이브리드.

- **KEEP의 Static-Dynamic 분리**: 에이전트 메모리를 정적(불변)/동적(가변) 세그먼트로 분리하는 원칙이 Activity B의 비연속 세그먼트 캐시 설계에 직접 전용 가능하다. 자주 변경되지 않는 "시스템 프롬프트/공통 맥락" 세그먼트는 정적으로 취급해 완전 재사용하고, 자주 갱신되는 "사용자 입력/도구 출력" 세그먼트는 동적으로 취급해 최소 재계산하는 이원화 전략.
- **GORGO의 세그먼트-인식 가능성**: GORGO의 크로스-리전 라우팅에 "비연속 세그먼트별 히트 확률"을 추가하면 단순 접두사 매칭이 아닌 세그먼트별 히트 예상 가치로 리전을 선택하는 Activity A+B 통합 라우터 설계 가능.

### Activity C
- **eOptShrinkQ(2605.02905) — 랜덤 행렬 이론 기반 자동 압축**: 저랭크 성분 자동 추출(eOptShrink) + 잔차 TurboQuant 압축이 Activity C에서 레이어/헤드별 하이퍼파라미터 수동 조정 없이 이론적으로 최적에 가까운 압축을 달성한다. KV-CoRE(이미 보고)의 Key가 Value보다 압축 가능성이 높다는 발견과 결합하면 Key는 낮은 랭크 + 더 공격적 양자화, Value는 높은 랭크 + 보수적 양자화하는 비대칭 eOptShrinkQ 변형이 유망하다.
- **DeltaKV의 잔차-기반 패러다임**: 원본 KV 저장 대신 전역 참조에 대한 잔차를 저장하는 접근이 Activity C의 "원본 토큰 선택/결합" 패러다임을 넘어 "참조+잔차" 저장 구조라는 새로운 방향을 열었다. eOptShrinkQ의 공유 저랭크 성분 = DeltaKV의 전역 참조 개념적 연결. B+C 통합에서 "공통 비연속 세그먼트를 저랭크 공유 성분으로, 세그먼트별 차이를 잔차로 저장"하는 통합 스토리지 구조를 실험할 것.
- **GVote(ICLR 2026)의 적응 예산**: 요청마다 다른 최적 압축 비율을 몬테카를로 샘플링으로 자동 결정하는 접근이 Activity C에서 "단일 고정 압축 비율"의 한계를 실증적으로 극복한다. Activity B+C 통합에서 GVote가 각 비연속 세그먼트의 미래 접근 가치를 예측해 세그먼트별 압축 비율을 자동 조정하는 "세그먼트-레벨 GVote" 변형을 탐색할 것.
