# 트렌드 리포트 — 2026-04-28

## 전체 요약

1. **Activity A (스케줄링)**: P/D 분리(Disaggregated Prefill) 아키텍처가 성숙 단계에 진입했으며, FlowKV·CPD·BanaServe 등 캐시-인식 라우팅과 부하 균형을 동시에 달성하는 시스템이 연이어 등장하고 있다. NVIDIA Dynamo와 LMCache는 RDMA/NVLink 기반 클러스터 규모의 KV 전송을 표준화하는 방향으로 수렴 중이다.
2. **Activity B (비연속 재사용)**: CacheBlend·EPIC·InfoBlend·vLLM generalized KV reuse 이슈 등을 통해 위치-독립적(position-independent) 캐싱이 핵심 연구 주제로 부상했다. 청크 단위 실험 논문(arXiv 2603.20218)이 청크 레벨 캐싱의 정량적 비교를 제공한다.
3. **Activity C (압축)**: FP8·INT8 KV 양자화가 vLLM·lmdeploy에서 실용화됐고, NVFP4(4비트)까지 상용화 확장 중이다. 정적 퇴거(SnapKV 계열) + 동적 선택(RocketKV)의 계층적 결합이 새로운 패러다임으로 주목받는다. KV Cache Transform Coding(KVTC)은 20×~40× 압축이라는 극단적 목표를 추구한다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [FlowKV: A Disaggregated Inference Framework with Low-Latency KV Cache Transfer and Load-Aware Scheduling](https://arxiv.org/abs/2504.03775)
- **출처**: arXiv (2025-04)
- **핵심 아이디어**: 전용 전송 최적화로 KV 전송 레이턴시를 평균 0.944s → 0.053s(96% 감소)로 줄이고, Load-Aware Scheduler가 PD 노드 할당을 동적으로 조정한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: KV 전송 병목이 해소되면 Activity A의 스케줄링 결정이 전송 대기 없이 즉각 반영될 수 있어 멀티-노드 오케스트레이션 설계에 직접 적용 가능.

#### 2. [Cache-Aware Prefill-Decode Disaggregation (CPD)](https://www.together.ai/blog/cache-aware-disaggregated-inference)
- **출처**: Together AI 블로그 (2025)
- **핵심 아이디어**: 캐시 히트율을 기준으로 "cold" vs "warm" 워크로드를 물리적으로 분리 배치해 TTFT 단축과 처리량 +40% 달성.
- **연구 목표와의 관련성**: high
- **주목할 점**: 히트율 기반 라우팅이 Activity A의 "요청 재정렬 + 배치 우선순위" 개념과 동일 원리. 히트율이 낮은 노드에 cold 요청을 몰아줌으로써 캐시 오염(cache pollution)을 방지.

#### 3. [BanaServe: Unified KV Cache and Dynamic Module Migration](https://arxiv.org/html/2510.13223v1)
- **출처**: arXiv / Wiley SPE (2026)
- **핵심 아이디어**: KV 캐시 마이그레이션 + Global KV Cache Store + 레이어 수준 가중치 마이그레이션을 조합해 핫스팟(cache-induced hotspot)을 근본적으로 제거.
- **연구 목표와의 관련성**: high
- **주목할 점**: 라우터가 순수 부하 인식만으로 스케줄링할 수 있도록 캐시 배치를 분리(decoupling). Activity A의 멀티-노드 KV 마이그레이션 설계에 직접 참고 가능.

#### 4. [Prefill-as-a-Service (PrfaaS): KVCache of Next-Generation Models Could Go Cross-Datacenter](https://arxiv.org/abs/2604.15039v1)
- **출처**: arXiv (2026-04)
- **핵심 아이디어**: 데이터센터 간 KV 캐시 선택적 오프로드 + 대역폭-인식(bandwidth-aware) 스케줄링 + 캐시-인식 요청 배치를 결합. 동질(homogeneous) PD 대비 처리량 54% 향상.
- **연구 목표와의 관련성**: high
- **주목할 점**: 크로스-데이터센터 시나리오까지 확장하는 최신 연구. InfiniBand 대역폭 제약을 스케줄링에 명시적으로 반영하는 점이 novelty.

#### 5. [LMCache: An Efficient KV Cache Layer for Enterprise-Scale LLM Inference](https://arxiv.org/html/2510.09665v2)
- **출처**: arXiv (2025-10)
- **핵심 아이디어**: KV 캐시를 CPU DRAM → 로컬 SSD → 원격 스토리지 → Redis 계층으로 오프로드하며, Ethernet/RDMA/NVLink를 모두 지원하는 범용 KV 레이어.
- **연구 목표와의 관련성**: medium
- **주목할 점**: 클러스터 규모 KV 공유의 사실상 기반 인프라로 자리 잡는 중. vLLM·SGLang 연동 플러그인 형태로 배포 가능.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Prefix-aware batching | vLLM/SGLang 내 단일 노드 중심 | CPD·llm-d 등에서 멀티-노드 계층으로 확장 |
| Cache-locality scheduling | 히트율 최우선 → 부하 불균형 문제 | BanaServe·FlowKV가 히트율+부하 균형 동시 달성 |
| Multi-node KV routing | 실험적 단계 | NVIDIA Dynamo·LMCache·llm-d가 프로덕션 적용 단계 진입 |
| Disaggregated prefill | 단순 PD 분리 | 크로스-DC(PrfaaS)·계층 스토리지(LMCache)로 진화 |

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [GPU-Accelerated INT8 Quantization for KV Cache Compression](https://arxiv.org/html/2601.04719v1)
- **출처**: arXiv (2026-01)
- **핵심 아이디어**: INT8 KV 양자화를 GPU 커널 수준에서 가속. 최대 요소별 오류 0.004, 어텐션 스코어 오류 0.1 미만 달성으로 정확도 영향 최소화.
- **연구 목표와의 관련성**: high
- **주목할 점**: vLLM이 현재 FP8만 지원(Issue #33480)하는 상황에서 INT8 GPU 커널을 별도 구현해 ~50% 메모리 절감을 실현한 실용적 기여.

#### 2. [PM-KVQ: Progressive Mixed-precision KV Cache Quantization for Long-CoT LLMs](https://openreview.net/forum?id=Vem6FQvRvq)
- **출처**: OpenReview (학회 제출, 2025-2026)
- **핵심 아이디어**: Chain-of-Thought 추론에서 레이어별·단계별 혼합 정밀도(mixed-precision) 양자화를 점진적으로 적용해 정확도와 압축률을 동시에 최적화.
- **연구 목표와의 관련성**: high
- **주목할 점**: 긴 추론 시퀀스에서 초기 레이어 KV는 고정밀도, 후반 레이어는 저정밀도 할당하는 방식. Activity C의 "정확도 보존 압축" 목표에 부합.

#### 3. [KV Cache Transform Coding (KVTC)](https://openreview.net/forum?id=aNVKROYpLB)
- **출처**: OpenReview (학회 제출, 2025-2026)
- **핵심 아이디어**: 변환 코딩(transform coding) 기법을 KV 캐시에 적용해 20×~40× 초고압축을 달성하면서 추론·장문 맥락 정확도를 보존.
- **연구 목표와의 관련성**: high
- **주목할 점**: 기존 양자화(2~4× 압축) 대비 한 자릿수 더 높은 압축률. 단, 실시간 디코딩 오버헤드 검증이 필요.

#### 4. [PagedEviction: Structured Block-wise KV Cache Pruning](https://arxiv.org/html/2509.04377)
- **출처**: arXiv (2025-09)
- **핵심 아이디어**: PagedAttention의 블록 단위 구조에 맞춰 KV 퇴거를 구조화해 fragmentation 없이 대용량 KV를 프루닝. 초기 어텐션 싱크 토큰은 보존.
- **연구 목표와의 관련성**: medium
- **주목할 점**: vLLM의 PagedAttention과 직접 통합 가능한 설계. 블록 단위 퇴거가 Activity A의 캐시 관리와 결합 시 복잡도 감소.

#### 5. [NVIDIA NVFP4 KV Cache for Long Context](https://developer.nvidia.com/blog/optimizing-inference-for-long-context-and-large-batch-sizes-with-nvfp4-kv-cache/)
- **출처**: NVIDIA Technical Blog (2025-2026)
- **핵심 아이디어**: FP8에서 한 단계 더 나아간 NVFP4(4비트) KV 캐시를 Hopper/Blackwell GPU에서 하드웨어 가속. 대형 배치·장문 컨텍스트 특화.
- **연구 목표와의 관련성**: medium
- **주목할 점**: 업계 표준이 FP8에서 FP4로 이동하는 신호. 동일 메모리에서 2× 더 많은 KV 저장 가능.

### 동향 변화 감지

|------|---------|------------|
| Quantization (INT8/FP8) | vLLM FP8만 지원, INT8 미지원 | INT8 GPU 커널 논문(2026-01) + vLLM INT8 지원 요청(Issue #33480) 활성화 |

| Low-rank / transform coding | 실험적 연구 단계 | KVTC가 20×~40× 압축 달성, 실용화 임박 |

## Activity 간 시너지 포인트

- **A + B**: CPD(캐시 히트율 기반 라우팅)와 비연속 세그먼트 재사용을 결합하면 "부분 히트(partial hit)" 요청도 warm 노드로 라우팅할 수 있어 Activity A의 히트율 분류 기준을 정밀화할 수 있다. BanaServe의 Global KV Store가 비연속 세그먼트를 저장하는 인프라로 활용될 수 있다.

- **B + C**: InfoBlend·CacheBlend의 위치-독립 KV 재조합 시 압축된 KV(INT8/FP8)를 그대로 연결(concatenate)하면 메모리 절감과 재사용률 향상을 동시에 달성할 수 있다. 단, 압축 KV의 선택적 재계산 비용을 측정해야 한다.

- **A + C**: PagedEviction의 블록 단위 퇴거 정책을 Activity A의 스케줄러가 인식하면 퇴거 직전 블록을 다른 요청에 우선 재사용하도록 스케줄링 힌트를 줄 수 있다. 스케줄러-압축기 공동 설계(co-design)가 새로운 방향.

- **A + B + C**: FlowKV의 저지연 KV 전송 위에 비연속 세그먼트(B)를 INT8 압축(C) 상태로 전송하면 전송 대역폭을 추가로 절감하면서 멀티-노드 재사용률(A)을 높이는 3중 시너지가 가능하다. KVTC 수준의 고압축과 결합 시 클러스터 규모 KV 공유의 네트워크 병목이 근본적으로 해소될 수 있다.

## idea-generator에게 전달할 포인트

### Activity A
- CPD와 BanaServe의 핵심 교훈: 캐시 히트율과 부하 균형은 서로 상충하므로 "히트율 가중 부하(hit-rate-weighted load)" 같은 복합 스코어로 라우팅해야 한다.
- PrfaaS가 크로스-DC 시나리오를 다루는 만큼, 단일-DC 내 RDMA/NVLink 기반 최적 경로 선택 알고리즘이 아직 미개척 영역.
- FlowKV의 96% 전송 지연 감소는 "전송 시간을 스케줄링에서 무시 가능한 상수"로 취급할 수 있게 하는 전제 조건이 될 수 있음.

### Activity B
- vLLM Issue #25672(Generalized KV reuse)가 공식 논의 중이므로 Activity B 구현을 vLLM 파이프라인과 호환되도록 설계하면 이식(6단계) 비용이 최소화됨.
- InfoBlend의 TTFT -54% 달성 비결(선택적 토큰 재계산 최소화)을 분석해 Activity B의 정확도 보존 메커니즘에 적용 가능.
- 청크 레벨 캐싱 실험 논문(arXiv 2603.20218)을 기준 비교 대상으로 활용해 비연속 히트율 30% 이상 목표 달성 여부를 측정할 것.

### Activity C
- KVTC(20×~40× 압축)의 실시간 인퍼런스 오버헤드가 공개되지 않았으므로 실용성 검증이 우선.
- PM-KVQ의 혼합 정밀도 전략(초기 레이어 고정밀도, 후반 저정밀도)은 Activity C의 "정확도 보존" 제약(±1% 이내)과 가장 잘 부합하는 방향.
- PagedEviction의 블록 단위 퇴거를 vLLM PagedAttention과 통합하는 구현이 상대적으로 공학적 난이도가 낮아 빠른 검증이 가능한 후보.
