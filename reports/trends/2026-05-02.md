# 트렌드 리포트 — 2026-05-02

## 전체 요약

(1) **Activity A (스케줄링)**: 이론 기반 KV 캐시 스케줄링 알고리즘(비-예언적 경쟁비 보장)이 등장하고, vLLM v0.18/v0.19에서 FlexKV 오프로딩 백엔드와 비동기 스케줄러가 기본 활성화됐다. Crusoe MemoryAlloy는 클러스터 전체를 단일 KV 메모리 평면으로 추상화해 9.9× TTFT 개선을 실증했다. KV 복원(restoration) 병목을 3D 병렬화로 해소하는 CacheFlow도 주목된다.
(2) **Activity B (비연속 재사용)**: RelayCaching이 멀티에이전트 협업에서 디코딩 KV를 훈련 없이 프리필에 직접 재주입해 4.7× 속도향상을 달성했다. AAAI 2026 채택 논문 Self-Indexing KVCache는 1-bit 부호 VQ로 압축-검색을 통합하여 5× 메모리 절감을 달성했다. 멀티에이전트 공유 KV 풀 연구(PolyKV)와 레이어 간 KV 공유(Stochastic KV Routing)도 새롭게 등장했다.
(3) **Activity C (압축)**: VQKV(벡터 양자화, 82.8% 압축 + 98.6% 성능 유지), CapKV(정보이론 기반 퇴거, ICLR 수준 정확도), LookaheadKV(ICLR 2026, 경량 룩어헤드 퇴거), ReCalKV(헤드 재정렬 저랭크 압축) 등 정확도 보존 압축 방법들이 집중 등장했다.
(4) 세 Activity 전반에서 "멀티에이전트 / 멀티세션 KV 공유"가 공통 주제로 부상 중이며, 비연속 재사용(B)과 압축(C)을 멀티에이전트 오케스트레이션(A)에 통합하는 연구가 수렴되고 있다.
(5) 이론적 스케줄링 보장(Activity A)과 정보이론적 압축 보장(Activity C)이 동시에 등장하여, 시스템 전체의 정확도-효율 트레이드오프를 형식적으로 보장하는 연구 방향이 열리고 있다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [Competitive Non-Clairvoyant KV-Cache Scheduling for LLM Inference](https://arxiv.org/abs/2601.22996)
- **출처**: arXiv 2601.22996 (January 2026)
- **핵심 아이디어**: 응답 길이를 사전에 알 수 없는(non-clairvoyant) 환경에서 KV 캐시 메모리 예산 제약 하에 레이턴시를 최소화하는 최초의 상수 경쟁비(competitive ratio) 알고리즘(GSA)을 제안한다. 주기적 재시작의 기하급수적 단계 구조(geometric phase structure) + 엇갈린 파이프라인(staggered pipeline)으로 메모리 예산 위반 없이 높은 동시성을 달성한다. 경쟁비 61.92(일반), 32(대용량 메모리).
- **연구 목표와의 관련성**: high — 스케줄링 오버헤드를 TTFT p50 +5% 이내로 유지하면서 최악의 경우 보장을 수학적으로 제공하는 이론 기반 스케줄러다.
- **주목할 점**: 단순 경험적 스케줄러에서 이론 보장 스케줄러로의 전환점이 될 수 있다. 실제 요청 트레이스에서도 강건한 성능을 실험으로 확인했다.

#### 2. [CacheFlow: Efficient LLM Serving with 3D-Parallel KV Cache Restoration](https://arxiv.org/abs/2604.25080)
- **출처**: arXiv 2604.25080, UIUC & NUS (April 2026)
- **핵심 아이디어**: KV 캐시 복원(restoration)을 토큰-레이어-GPU 3개 차원의 병렬 실행 문제로 재정의한다. 재계산과 I/O 전송을 구조적 의존성에 따라 세밀하게 오버랩하는 "배치-인식 two-pointer 스케줄러"로 요청별 TTFT를 10~62% 감소시킨다.
- **연구 목표와의 관련성**: high — KV 복원 병목은 멀티턴·RAG·에이전트 파이프라인에서 비연속 캐시 재사용(Activity B)과 직결되며, 3D 병렬화가 스케줄링 오버헤드를 대폭 줄인다.
- **주목할 점**: 기존 시스템(HCache 등)이 단일 요청 트레이드오프에만 집중한 것과 달리, 배치 전체의 컴퓨트·I/O 자원 할당을 공동 최적화한다.

#### 3. [Predictive Multi-Tier Memory Management for KV Cache in Large-Scale GPU Inference](https://arxiv.org/abs/2604.26968)
- **출처**: arXiv 2604.26968 (April 2026)
- **핵심 아이디어**: GPU HBM → CPU DRAM → CXL → NVMe(GPUDirect) → RDMA 패브릭 → 병렬 파일시스템으로 이어지는 6-계층 메모리 위계를 통해 노드당 KV 용량을 40 GB에서 38 TB 이상으로 확장한다. 아키텍처 변이형(MLA 포함) 인식 크기 산정 엔진으로 최대 7.4× 배치 크기 향상, RDMA 풀이 NVMe 대비 처리량 +23% 달성.
- **연구 목표와의 관련성**: high — 멀티노드 KV 라우팅의 실질적 병목인 메모리 계층 관리를 포괄적으로 다루며, RDMA 기반 크로스-노드 공유를 지원한다.
- **주목할 점**: MLA(Multi-head Latent Attention) KV 캐시의 일반 프레임워크 미지원으로 최대 57× 메모리 초과 할당 문제를 해결한다. MLA 압축 구조를 인식하는 최초의 범용 KV 관리 프레임워크.

#### 4. [Crusoe MemoryAlloy: Reinventing KV Caching for Cluster-Scale Inference](https://www.crusoe.ai/resources/blog/crusoe-memoryalloy-reinventing-kv-caching-for-cluster-scale-inference)
- **출처**: Crusoe AI 기술 블로그 (2025–2026)
- **핵심 아이디어**: 단일 모델·런타임에 독립적인 클러스터 전체 KV 메모리 평면(unified memory plane)을 구현한다. KV 세그먼트를 노드 간 P2P 전송으로 공유하고, TTFT 추정 기반 KV-인식 게이트웨이가 캐시 히트 확률이 높은 인스턴스로 요청을 라우팅한다. 9.9× TTFT 단축, 5× 처리량 향상 달성.
- **연구 목표와의 관련성**: high — 클러스터 규모 비연속 KV 세그먼트 공유와 캐시-인식 라우팅이 Activity A+B 통합의 실용 레퍼런스다.
- **주목할 점**: P2P 메모리 패브릭 + KV-인식 로드밸런서 조합이 llm-d(중앙 인덱서 방식)와 보완적 설계 대안을 제시한다.

#### 5. [vLLM v0.18 / v0.19 릴리즈 — FlexKV 오프로딩 백엔드 및 비동기 스케줄러](https://fazm.ai/blog/vllm-update-april-2026)
- **출처**: vLLM 공식 릴리즈 (March–April 2026)
- **핵심 아이디어**: v0.18(2026-03 말)에서 FlexKV(Tencent & NVIDIA 협업)가 Mooncake Transfer Engine을 통한 분산 KV 재사용을 지원하는 KV 오프로딩 백엔드로 추가됐다. v0.19(2026-04-02)에서는 비동기 스케줄러가 기본 활성화되고 Model Runner V2가 도입됐다.
- **연구 목표와의 관련성**: high — vLLM 이식(6단계) 대상인 분산 KV 오프로딩과 스케줄러가 mainline에 공식 통합됐다.
- **주목할 점**: FlexKV + Mooncake Transfer Engine 조합이 vllm-porter의 직접 이식 타깃이 된다. 비동기 스케줄러 기본화로 Activity A의 서브클래싱 이식 구조가 변경될 수 있다.

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [RelayCaching: Accelerating LLM Collaboration via Decoding KV Cache Reuse](https://arxiv.org/abs/2603.13289)
- **출처**: arXiv 2603.13289 (February 2026)
- **핵심 아이디어**: 멀티에이전트 파이프라인에서 이전 에이전트의 **디코딩 단계** KV 캐시를 다음 에이전트의 **프리필 단계**에 훈련 없이 직접 재주입한다. "동일 콘텐츠의 KV는 디코딩·프리필 간 높은 일관성을 보이되, 프리픽스 유도 편차(prefix-induced deviation)는 소수 레이어·소수 토큰에 집중된다"는 관찰에 기반한다. U자형 레이어 프로파일러로 핵심 레이어 범위를 식별하고, 편차 기반 + 영향 기반 토큰 선택기로 수정 대상 토큰만 재계산한다. 80% 이상 KV 재사용, 최대 4.7× 속도향상.
- **연구 목표와의 관련성**: high — 비연속 세그먼트(디코딩 KV)를 다른 컨텍스트(프리필)에서 재사용하는 가장 공격적인 형태의 비연속 재사용이다.
- **주목할 점**: KV Packet(학습 필요)과 달리 완전 훈련-무료(training-free)다. 정확도 보존에 핵심인 "편차 집중 레이어" 분석이 Activity C의 레이어별 중요도 구분과 시너지가 크다.

#### 2. [Self-Indexing KVCache: Predicting Sparse Attention from Compressed Keys](https://arxiv.org/abs/2603.14224)
- **출처**: arXiv 2603.14224 / AAAI 2026
- **핵심 아이디어**: 압축된 Key를 단순한 저장 구조가 아닌 희소 어텐션의 "자기 색인(self-index)"으로 활용한다. 부호 기반 1-bit 벡터 양자화(VQ)로 프리필 시 단일 패스로 코드북을 구성하고, 코사인 유사도 기반 top-k 검색으로 희소 어텐션 후보를 결정한다. 추가 훈련·보조 색인 구조 없이 5× 메모리 절감, 낮은 지연 달성.
- **연구 목표와의 관련성**: high — 압축(C)과 비연속 세그먼트 검색(B)을 단일 1-bit 표현으로 통합하는 B+C 융합 접근이다.
- **주목할 점**: AAAI 2026 채택으로 학계 인정. 1-bit 키 표현이 하드웨어 친화적(bitwise AND/OR 연산)이어서 실제 커널 구현 난이도가 낮다.

#### 3. [PolyKV: A Shared Asymmetrically-Compressed KV Cache Pool for Multi-Agent LLM Inference](https://arxiv.org/abs/2604.24971)
- **출처**: arXiv 2604.24971 (April 2026)
- **핵심 아이디어**: N개의 동시 에이전트가 **단일** 비대칭 압축 KV 풀을 공유한다. Key는 INT8(softmax 안정성 보존), Value는 FWHT 회전 후 3-bit Lloyd-Max 양자화(N(0,1) 최적). HuggingFace DynamicCache 객체를 통해 에이전트별 컨텍스트에 압축 KV를 주입한다. 15개 에이전트 공유 시 KV 메모리 97.7% 감소(19.8 GB → 0.45 GB).
- **연구 목표와의 관련성**: high — 멀티에이전트 시나리오에서 비연속 KV 공유(B)와 압축(C)을 직접 결합한 B+C 구현체다.
- **주목할 점**: SAW-INT4(동일 FWHT 계열)와 설계 원리가 일치하여 Activity C 구현체와 직접 통합 가능. 공유 풀 방식은 Activity A의 멀티에이전트 스케줄링 설계와도 연결된다.

#### 4. [Stochastic KV Routing: Enabling Adaptive Depth-Wise Cache Sharing](https://arxiv.org/abs/2604.22782)
- **출처**: arXiv 2604.22782 (April 2026)
- **핵심 아이디어**: 훈련 중 각 레이어가 자신의 KV 상태 또는 선행 레이어의 KV를 무작위로 선택해 어텐션을 수행하는 "랜덤 크로스-레이어 어텐션"을 적용한다. 이 확률적 훈련이 모델을 다양한 깊이-방향 KV 공유 전략에 강건하게 만들어, 추론 시 레이어 간 KV를 공유해 메모리 풋프린트를 줄인다. 데이터 제한 환경의 대형 모델에서 성능 유지 또는 개선.
- **연구 목표와의 관련성**: medium — 비연속성이 토큰 위치(Activity B의 주류)가 아닌 레이어 깊이(depth) 차원에서 발생하는 새로운 형태의 KV 공유다.
- **주목할 점**: 레이어 간 KV 공유는 토큰 퇴거·압축과 직교(orthogonal)하여 추가 압축과 결합 가능하다. 훈련 시 확률적 라우팅을 사용하므로 기존 모델에 적용하려면 파인튜닝이 필요하다.

| CacheBlend / Blending / KVLink | KV Packet(훈련 어댑터)이 재계산 없는 PIC 새 패러다임 | RelayCaching(훈련-무료)이 디코딩 KV의 직접 재주입으로 경쟁 진입 |

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [VQKV: High-Fidelity and High-Ratio Cache Compression via Vector-Quantization](https://arxiv.org/abs/2603.16435)
- **출처**: arXiv 2603.16435, LUMIA Lab / Shanghai Jiao Tong University (March 2026)
- **핵심 아이디어**: 훈련-무료(training-free) 벡터 양자화(VQ)로 KV 캐시를 코드북 인덱스로 치환한다. SimVQ 기반 공동 벡터-수준 양자화로 내재적 데이터 구조를 포착하고 표현을 점진적으로 정제한다. LLaMA3.1-8B에서 82.8% 압축률 + LongBench 98.6% 성능 유지, 4.3× 더 긴 생성 길이 지원.
- **연구 목표와의 관련성**: high — "정확도 보존(±1% 이내) + 메모리 −30% 이상" 목표를 동시에 달성하며, 저랭크·스칼라 양자화의 한계(고압축 시 품질 저하)를 극복한다.
- **주목할 점**: 훈련-무료 방식이 빠른 배포를 허용한다. Self-Indexing KVCache(1-bit 부호 VQ)와 비교 실험이 B+C 구현 방향 선택에 직접 도움이 된다.

#### 2. [CapKV: Rethinking KV Cache Eviction via a Unified Information-Theoretic Objective](https://arxiv.org/abs/2604.25975)
- **출처**: arXiv 2604.25975 (April 2026)
- **핵심 아이디어**: KV 캐시 퇴거를 정보 병목(Information Bottleneck) 원리로 재해석한다. 선형-가우시안 어텐션 대체 모델 하에서 보유 KV 부분집합의 유효 정보 용량을 특성화하는 닫힌 형태 상호 정보 목적함수를 유도한다. CapKV는 통계적 레버리지 스코어(leverage score) 기반 로그-행렬식 근사로 용량-최대화를 직접 타깃한다.
- **연구 목표와의 관련성**: high — 기존 경험적 퇴거 방법들을 "동일 용량-최대화 목적함수의 서로 다른 근사"로 통일하는 이론적 프레임워크. "정확도 보존 압축"의 수학적 기초를 제공한다.
- **주목할 점**: 메모리 예산이 줄어들수록 정확도 저하 곡선이 다른 방법 대비 현저히 완만(graceful degradation)하다. Activity B의 세그먼트 중요도 점수를 레버리지 스코어로 대체하면 B+C 통합의 이론적 기반이 된다.

#### 3. [LookaheadKV: Fast and Accurate KV Cache Eviction by Glimpsing into the Future without Generation](https://arxiv.org/abs/2603.10899)
- **출처**: arXiv 2603.10899 / ICLR 2026 (March 2026)
- **핵심 아이디어**: 미래 응답의 대리 표현(surrogate future response)을 명시적 드래프트 생성 없이 경량 파라미터-효율 모듈로 예측하여 KV 중요도를 사전 추정한다. 드래프트 생성 기반 방법의 프리필 오버헤드를 제거하면서도 정확도를 유지한다. 퇴거 비용 최대 14.5× 감소, TTFT 대폭 단축.
- **연구 목표와의 관련성**: high — ICLR 2026 채택으로 최고 수준 검증 완료. "미래 중요도 예측"이 Activity B의 세그먼트 재사용 우선순위 결정과 결합 가능하다.
- **주목할 점**: 기존 비용이 큰 미래-엿보기(SnapKV 계열) 방법들과 달리 추가 추론 오버헤드가 기존 경험적 방법 수준으로 낮다. 구현체 공개(Samsung Labs GitHub).

#### 4. [IceCache: Memory-efficient KV-cache Management for Long-Sequence LLMs](https://arxiv.org/abs/2604.10539)
- **출처**: arXiv 2604.10539 / OpenReview (April 2026)
- **핵심 아이디어**: 의미적으로 연관된 토큰을 연속 메모리 영역으로 클러스터링하는 계층적·동적 데이터 구조로 PagedAttention을 확장한다. 256-토큰 예산에서 원본 정확도의 99% 유지, 다른 오프로딩 기반 방법 대비 25% KV 예산으로 경쟁력 있는 지연-정확도 달성.
- **연구 목표와의 관련성**: high — 의미 기반 클러스터링이 Activity B의 세그먼트 경계 결정과 직접 연관되며, PagedAttention 통합 설계가 vLLM 이식을 용이하게 한다.
- **주목할 점**: Qwen3-32B 포함 4개 모델에서 6개 베이스라인 대비 일관된 우위. 오프로딩과 의미 클러스터링의 결합이 TTKV(시간 계층화)와 보완 관계.

#### 5. [ReCalKV: Low-Rank KV Cache Compression via Head Reordering and Offline Calibration](https://arxiv.org/abs/2505.24357)
- **출처**: arXiv 2505.24357 / OpenReview (May 2026)
- **핵심 아이디어**: Key와 Value를 분리 처리한다. Key에는 헤드-와이즈 유사도 인식 재정렬(HSR)로 구조적으로 유사한 헤드를 그룹화한 후 그룹화된 SVD로 저랭크 근사를 수행한다. Value에는 오프라인 교정 데이터로 Value 프로젝션 행렬을 효율적으로 교정(OVC)한다. LLaMA-2-7B, 50% 압축에서 정확도 2% 상대 저하.
- **연구 목표와의 관련성**: high — K/V 비대칭 처리가 Activity C의 "정확도 보존" 목표에 특히 효과적이며, 양자화(PolyKV의 INT8/3-bit)와 직교 결합 가능.
- **주목할 점**: K/V 각각의 수학적 특성(K: 유사 헤드 클러스터링, V: 컨텍스트 정보 보존)에 맞춰 서로 다른 압축 전략을 적용하는 체계적 설계가 이후 연구의 참조 기준이 될 수 있다.

#### 6. [Sequential KV Cache Compression via Probabilistic Language Tries: Beyond the Per-Vector Shannon Limit](https://arxiv.org/abs/2604.15356)
- **출처**: arXiv 2604.15356 (April 2026)
- **핵심 아이디어**: TurboQuant 등이 달성한 벡터 단위 Shannon 엔트로피 한계를 시퀀스 단위 엔트로피 한계로 돌파하는 2계층 순차적 KV 압축 구조를 제안한다. 1계층: 확률적 접두사 중복 제거(Probabilistic Language Tries, PLT)로 세션 간 공유 접두사를 식별. 2계층: 예측적 델타 코딩(predictive delta coding)으로 각 KV 벡터를 이전 토큰의 모델 예측 잔차만 저장. 이론적 압축비 TurboQuant 대비 ~914,000× 향상 주장.
- **연구 목표와의 관련성**: medium — 이론적으로 매우 공격적인 압축비를 제시하지만, 실시간 추론 오버헤드 검증이 아직 없다. 그러나 PLT의 크로스-세션 접두사 공유 개념이 Activity B의 비연속 세그먼트 재사용과 연결된다.
- **주목할 점**: PLT가 Activity B의 세그먼트 해시 캐시와 동일한 원리("공통 접두사 공유")를 정보이론적으로 형식화한 것이어서 B+C 통합 이론 기반으로 활용 가능하다.

## Activity 간 시너지 포인트

**A + B (스케줄링 + 비연속 재사용)**
- Crusoe MemoryAlloy의 P2P KV 패브릭 + KV-인식 TTFT 기반 라우팅이 RelayCaching의 "디코딩 KV → 프리필 재주입"을 멀티노드로 확장하는 직접적 인프라가 된다. 에이전트 A가 생성한 디코딩 KV를 에이전트 B의 프리필에 네트워크를 통해 재사용하는 시나리오가 현실화된다.
- CacheFlow의 3D 병렬 복원 프레임워크가 비연속 세그먼트 복원 병목을 해소함으로써 Activity B의 히트 시 TTFT 오버헤드를 줄인다.

**B + C (비연속 재사용 + 압축)**
- Self-Indexing KVCache(1-bit 부호 VQ)가 B와 C를 단일 구조로 통합한 직접적 사례: 압축된 Key가 동시에 희소 어텐션 색인으로 작동한다.
- PolyKV의 비대칭 압축 공유 풀(Key INT8 + Value 3-bit)을 Activity B의 세그먼트 해시 캐시와 결합하면 압축된 비연속 세그먼트 재사용 파이프라인이 완성된다.
- CapKV의 레버리지 스코어 기반 퇴거를 Activity B의 세그먼트 중요도 점수로 재해석하면, "중요한 세그먼트는 고정밀도로 캐시, 덜 중요한 세그먼트는 저정밀도 압축 후 캐시"하는 계층적 B+C 정책이 가능하다.

**A + C (스케줄링 + 압축)**
- Predictive Multi-Tier Memory Management(2604.26968)의 6-계층 메모리 위계 + RDMA 전송에서 VQKV나 PolyKV 압축 KV를 사용하면, 네트워크 전송 대역폭 요구를 대폭 줄이면서 멀티노드 KV 공유를 실현할 수 있다.
- Competitive Non-Clairvoyant 스케줄러(GSA)의 메모리 예산 모델에 CapKV의 정보 용량 목적함수를 통합하면, "이론적으로 보장된 정확도 보존 + 메모리 효율" 공동 최적화 스케줄러 설계가 가능하다.

**A + B + C (통합)**
- 핵심 통합 방향: RelayCaching(훈련-무료 비연속 재사용, B) → VQKV 또는 PolyKV 압축(C) → Crusoe MemoryAlloy P2P 패브릭 또는 CacheFlow 복원 스케줄러(A)로 이어지는 멀티에이전트 KV 공유 파이프라인. 각 에이전트가 생성한 디코딩 KV를 압축 후 클러스터 KV 풀에 저장하고, 후속 에이전트가 훈련-무료로 재사용하는 완전한 A+B+C 통합 시나리오.
- Sequential KV PLT(2604.15356)의 크로스-세션 접두사 공유가 Activity B의 세그먼트 해시 캐시 및 Activity A의 라우팅 힌트와 결합하면, 이론적으로 최적에 가까운 크로스-세션 KV 재사용 시스템이 가능하다.
