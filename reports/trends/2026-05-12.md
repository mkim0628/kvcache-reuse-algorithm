# 트렌드 리포트 — 2026-05-12

## 전체 요약

1. **Activity A (스케줄링)**: SSD 기반 KV 캐시 오프로딩의 병목이 CPU에서 GPU로 이동하는 구조 전환이 핵심 동향이다. Tutti(2605.03375)는 GPU-centric KV 오브젝트 스토어 + GPU io_uring으로 CPU를 크리티컬 패스에서 완전히 제거해 LMCache 대비 TTFT를 78.3% 단축하고 요청 처리율을 2×로 향상한다. SparKV(2604.21231)는 클라우드-에지 하이브리드 환경에서 KV 청크별로 "스트리밍 vs 로컬 연산"을 동적으로 선택하는 새로운 온-디맨드 KV 로딩 패러다임을 제시하며 TTFT를 1.3~5.1× 단축한다.

2. **Activity B (비연속 재사용)**: KV 캐시 재사용의 두 가지 새 방향이 등장했다. AdapShot(2605.03644)은 RoPE의 회전 속성을 활용한 위치 재인코딩(position re-encoding)으로 많은 샷(shot)을 비용 없이 재정렬해 Many-Shot ICL에서 비연속 KV 재사용을 실현한다. Louver(2605.06763)는 KV 검색을 반공간 범위 탐색(halfspace range searching) 문제로 재형식화해 "제로 거짓 음성(zero false negative)"을 보장하면서 FlashAttention보다 빠른 KV 인덱스를 제시한다.

3. **Activity C (압축)**: 세 가지 서로 다른 새 방향이 나란히 등장했다. MixedDimKV(2603.20616)는 토큰 퇴거(0 or 전체 차원)를 넘어 토큰별 최적 차원 수를 할당하는 혼합-차원 예산 배분으로 A100에서 디코딩 지연을 55% 수준으로 줄인다. Sequential PLT(2604.15356)는 확률적 언어 트라이(PLT)로 세션 간 공통 접두사를 예측적으로 중복 제거하고 델타 코딩으로 잔차를 압축해 벡터-단위 섀넌 한계를 넘는 압축률을 달성한다. MISA(2605.07363)는 DeepSeek 희소 어텐션 인덱서의 헤드를 MoE 라우터로 선택해 긴 컨텍스트에서 인덱서 비용을 대폭 줄인다.

4. **A+B 시너지**: Tutti의 GPU-centric 벌크 KV 전송과 Louver의 제로 거짓 음성 KV 검색이 결합되면, SSD에서 로드할 세그먼트를 Louver가 정확히 선별하고 Tutti가 GPU-native 전송으로 빠르게 복원하는 "정확-고속 비연속 세그먼트 SSD 복원" 파이프라인이 가능하다.

5. **B+C 시너지**: AdapShot의 RoPE 위치 재인코딩이 Activity B의 비연속 세그먼트 재조합에서 위치 불일치 문제를 해소하는 경량 접근을 제공하며, MixedDimKV의 혼합-차원 압축을 각 세그먼트에 적용하면 세그먼트별 중요도에 따라 차원 수를 달리 할당하는 "세그먼트-인식 혼합-차원 압축" 방향이 가능하다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [Tutti: Making SSD-Backed KV Cache Practical for Long-Context LLM Serving](https://arxiv.org/abs/2605.03375)
- **출처**: arXiv 2605.03375 (2026-05-05)
- **핵심 아이디어**: 기존 SSD 기반 KV 캐시 오프로딩 시스템(GPU Direct Storage 포함)이 모두 CPU-centric 구조여서 파편화된 GPU 메모리 레이아웃이 수많은 작은 랜덤 I/O를 유발하고 CPU를 극심한 병목으로 만드는 문제를 해결한다. 세 가지 설계 기여를 통해 CPU를 크리티컬 데이터·I/O 제어 경로에서 완전히 제거한다: (1) GPU-native 오브젝트 추상화 — KV 캐시를 GPU가 직접 관리하는 오브젝트로 표현해 벌크 전송 및 관리 가능, (2) GPU io_uring — GPU가 비동기적으로 직접 I/O를 개시·완료할 수 있도록 GPU 스토리지 스택 재설계, (3) slack-aware I/O 스케줄링 — GPU 연산의 유휴 슬랙을 감지해 I/O와 컴퓨팅을 오버랩하면서 GPU 자원 경쟁 최소화. CPU는 레이어당 한 번 I/O 커널을 GPU에 비동기 로드하는 역할만 담당한다. GPU-centric KV 캐시 오브젝트 스토어가 핵심.
- **연구 목표와의 관련성**: high
- **주목할 점**: 최신 GDS 기반 LMCache 대비 TTFT 78.3% 단축, 요청 처리율 2×, 서빙 비용 27% 절감. DRAM 기반 LMCache와 거의 동등한 추론 성능을 무한에 가까운 SSD 용량으로 제공. Activity A의 "스케줄링 오버헤드 TTFT p50 +5% 이내" 목표를 SSD 오프로딩 환경에서도 달성하는 새로운 기준. Dual-Blade(이미 보고)의 NVMe-Direct 이중 경로가 파일시스템 오버헤드 제거에 집중한 반면, Tutti는 CPU를 제어 경로에서 완전 제거하는 더 근본적 재설계. TTKV(이미 보고)의 HBM/DRAM 시간 기반 계층화와 결합 시 "HBM 최신 → DRAM 중간 → SSD 고용량" 3-티어에서 각 계층별 전송 메커니즘을 최적화하는 완전한 계층 KV 스토리지 스택이 가능하다. vLLM 기반 구현 여부와 오픈소스 공개 상태 확인 필요.

#### 2. [SparKV: Overhead-Aware KV Cache Loading for Efficient On-Device LLM Inference](https://arxiv.org/abs/2604.21231)
- **출처**: arXiv 2604.21231 (2026-04-23)
- **핵심 아이디어**: 에지 기기의 LLM 추론에서 프리필 단계가 전체 KV 캐시를 로컬에서 계산해야 해 TTFT가 과도하게 길어지는 문제를 클라우드-에지 하이브리드 KV 로딩으로 해결한다. 클라우드가 KV 캐시를 토큰·어텐션 헤드·레이어 차원으로 분할한 청크(indexed chunks)로 인덱싱하고, 에지 스케줄러가 각 청크에 대해 (1) 클라우드에서 스트리밍할지 (2) 로컬에서 계산할지를 동적으로 결정하는 "의존성-인식 로딩 결정(dependency-aware loading decision)" 구조. 경량 MLP 예측기가 어텐션 희소성 특성에서 청크별 로컬 계산 지연을 추정한다. 적응형 컨트롤러가 슬라이딩 윈도우로 무선 처리량과 에지 연산 여유를 모니터링해 스트리밍↔계산 경로를 동적으로 전환하며 두 경로를 오버랩한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 다양한 데이터셋·LLM·에지 기기 실험에서 TTFT 1.3~5.1× 단축, 요청당 에너지 소비 1.5~3.3× 절감. Dual-Blade(이미 보고)가 에지 NVMe-Direct 오프로딩으로 로컬 I/O를 가속한 반면, SparKV는 클라우드의 사전 계산 KV를 스트리밍해 에지 계산 자체를 대체하는 근본적으로 다른 방향. Activity A의 "비연속 세그먼트 로드" 최적화에서 "어떤 세그먼트를 로컬에서 계산하고 어떤 것을 원격에서 가져올 것인가"를 어텐션 희소성 신호로 결정하는 설계가 Activity B 비연속 재사용과 직접 연결된다 — 세그먼트 히트 예상 시 원격 스트리밍, 미스 예상 시 로컬 계산하는 캐시-인식 SparKV 변형이 가능하다.

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [AdapShot: Adaptive Many-Shot In-Context Learning with Semantic-Aware KV Cache Reuse](https://arxiv.org/abs/2605.03644)
- **출처**: arXiv 2605.03644 (2026-05-05, University of Electronic Science and Technology of China)
- **핵심 아이디어**: Many-Shot In-Context Learning(ICL)에서 기존 방법이 고정된 샷 수를 사전 결정해 쿼리 난이도에 적응하지 못하고, 긴 컨텍스트의 과도한 프리필 비용이 실용화를 막는 두 가지 문제를 동시에 해결한다. 세 가지 핵심 기여: (1) **탐침 기반 적응 샷 선택** — 출력 엔트로피를 기준으로 최적 샷 수를 경량 탐침 실행으로 결정, (2) **의미-인식 KV 재사용** — 탐침 단계와 추론 단계에서 중복 프리필 계산을 우회하는 KV 캐시 재사용 전략, (3) **위치 재인코딩(position re-encoding)** — RoPE의 회전 속성을 활용해 샷의 저비용·무손실·유연한 재순서화를 실현. 위치 분리(position decoupling) 후 RoPE 회전으로 임의 순서의 샷을 원하는 위치에 배치한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 프리필 비용이 사전 고정 방법 대비 대폭 절감되면서 많은 샷의 정보를 활용하는 QA 정확도를 유지. Activity B의 "위치-독립 비연속 KV 재사용"에서 RoPE 위치 재인코딩이 세그먼트 순서 재조합 시 발생하는 위치 오류를 해소하는 경량 접근으로 직접 활용 가능하다. InfoFlowKV(이미 보고)가 정보 흐름 기반 청크 재순서화를 형식화했다면, AdapShot은 RoPE 속성으로 실제 재순서화를 비용 없이 구현하는 방법을 제공한다 — 두 접근의 결합("정보 흐름 최적 순서 → RoPE 재인코딩으로 실현")이 Activity B의 비연속 세그먼트 조합 정확도를 향상시킨다. 에이전틱 워크로드에서 반복되는 도구 설명/예제를 Many-Shot으로 취급하면 KEEP(이미 보고)의 정적-동적 분리와 AdapShot의 적응 샷 선택을 통합하는 "에이전트-메모리 적응 KV 샘플링" 방향이 도출된다.

#### 2. [Sparse Attention as a Range Searching Problem: Towards an Inference-Efficient Index for KV Cache (Louver)](https://arxiv.org/abs/2605.06763)
- **출처**: arXiv 2605.06763 (2026-05-08)
- **핵심 아이디어**: 기존 KV 캐시 희소 어텐션 방법들이 고정 또는 적응형 토큰 예산 하에서 거짓 음성(false negative — 중요한 KV를 놓치는 것)을 허용해 출력 오류가 발생하는 문제를 해결한다. 희소 어텐션을 **반공간 범위 탐색(halfspace range searching)** 문제로 재형식화한다. 주어진 임계값에 대해 중요한 KV를 하나도 놓치지 않는 "제로 거짓 음성(zero false negative)"을 이론과 실험 모두에서 보장하는 새로운 인덱스 구조 Louver를 제안한다. 기존 LLM 파이프라인에 경량 통합이 가능하며 CPU와 GPU 양쪽에 대한 하드웨어-인식 최적화를 포함한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 기존 희소 어텐션 방법 대비 정확도와 런타임 양쪽에서 우수. 고도로 최적화된 FlashAttention 밀집 어텐션보다도 빠름. Activity B의 비연속 세그먼트 검색에서 가장 중요한 제약 — "중요한 KV를 놓쳐서 발생하는 정확도 저하" — 을 이론적으로 제거하는 첫 번째 접근. LycheeCluster(이미 보고)의 삼각부등식 기반 계층 인덱스가 O(N)→로그 시간 검색을 달성했지만 거짓 음성을 허용하는 반면, Louver는 제로 거짓 음성을 보장하면서 실용 속도를 달성한다. Activity B의 Radix 트리 캐시(src/cache/radix.py)에서 세그먼트 검색 정확도를 높이는 이론적 기반으로 Louver의 범위 탐색 접근을 통합하면 "비연속 세그먼트를 놓치지 않고 찾는" 고정밀 검색이 가능하다. DASH-KV(이미 보고)의 비대칭 딥 해싱이 O(N) 근사 검색을 제공한 반면, Louver는 정확 검색(exact) 수준의 품질을 희소성으로 달성한다.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [Beyond Token Eviction: Mixed-Dimension Budget Allocation for Efficient KV Cache Compression (MixedDimKV)](https://arxiv.org/abs/2603.20616)
- **출처**: arXiv 2603.20616 (2026-03-21)
- **핵심 아이디어**: 기존 토큰 퇴거 방법들이 각 토큰에 "전체 차원 유지 또는 완전 퇴거(0 차원)"라는 이진 결정만 내릴 수 있어 세분성이 지나치게 거칠다는 문제를 해결한다. 각 토큰을 서로 다른 최적 차원 수로 압축하는 **혼합-차원 예산 배분(mixed-dimension budget allocation)**을 제안한다. 토큰의 어텐션 유의미도(attention significance), 값 크기(value magnitude), 차원별 고유 압축 가능성(compressibility)을 결합한 손실 점수(loss score)를 정의하고, 이를 바탕으로 제약 최적화 문제를 이분 탐색(bisection-based search)으로 효율 풀어 토큰별 최적 차원을 결정한다. MixedDimKV-H는 헤드 수준 중요도 정보를 추가 통합한 변형.
- **연구 목표와의 관련성**: high
- **주목할 점**: 장문 컨텍스트 벤치마크(LongBench)에서 헤드-레벨 중요도 정보를 사용하는 기존 방법(HeadKV 등)을 지속 초과. A100에서 디코딩 지연을 전체 어텐션 대비 55%로 감소. Activity C의 "메모리 −30% 이상" 목표를 기존 이진 퇴거 대비 더 정밀한 예산 활용으로 달성하는 새로운 접근. LKV(이미 보고)가 예산과 토큰 선택을 엔드-투-엔드로 학습한 반면, MixedDimKV는 학습 없이(inference-time) 차원 할당을 최적화해 즉시 배포 가능하다. Activity B의 비연속 세그먼트에 MixedDimKV를 적용하면 세그먼트별 중요도(히트 빈도, 접근 시간 등)에 따라 각 세그먼트의 KV 차원 수를 달리 할당하는 "세그먼트-레벨 혼합-차원 압축"이 가능하다.

#### 2. [Sequential KV Cache Compression via Probabilistic Language Tries: Beyond the Per-Vector Shannon Limit](https://arxiv.org/abs/2604.15356)
- **출처**: arXiv 2604.15356 (2026-04-10, Gregory Magarshak)
- **핵심 아이디어**: 기존 KV 캐시 압축 연구가 개별 KV 벡터를 독립적으로 압축하는 "벡터 단위(per-vector)" 접근에 집중해 벡터 단위 섀넌 엔트로피 한계를 극복하지 못한다는 문제를 해결한다. KV 캐시에 저장된 토큰들이 모델이 훈련된 정확한 형식 언어(formal language)의 샘플임을 활용해 시퀀스 단위 압축을 실현한다. 2계층 구조: (1) **확률적 언어 트라이(Probabilistic Language Tries, PLT)** — 세션 간 의미적으로 동등한 공통 접두사를 PLT로 예측적으로 중복 제거. (2) **예측 델타 코딩(Predictive Delta Coding)** — 각 새 KV 벡터를 모델 자신의 예측으로부터의 잔차(residual)로만 저장. 모델이 언어의 근-최적 예측기라는 사실이 잔차 엔트로피를 이론적으로 최소화한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 세션 간 접두사 중복 제거(cross-session prefix deduplication)와 벡터-단위 델타 코딩을 결합해 벡터-단위 섀넌 한계를 초과하는 압축률 달성. Activity C의 "동일 메모리에서 2× 이상 유효 컨텍스트" 목표를 세션-레벨 압축으로 달성하는 새로운 경로. eOptShrinkQ(이미 보고)의 공유 저랭크 성분 추출이 단일 세션 내 공통성을 활용한다면, PLT는 세션 간 공통 접두사를 확률 모델로 예측·중복 제거해 교차-세션 효율을 달성한다. KVTC(이미 보고)의 PCA+엔트로피 코딩과 설계 방향이 상호 보완적 — KVTC가 변환 후 엔트로피를 낮추는 반면 PLT는 언어 모델 예측으로 델타 자체를 최소화한다. Activity B의 다중 세션 비연속 KV 재사용에서 PLT를 세그먼트 인덱스로 활용하면 "의미적으로 동등한 세그먼트를 확률 모델로 자동 중복 제거"하는 시맨틱 해시 대안이 가능하다.

#### 3. [MISA: Mixture of Indexer Sparse Attention for Long-Context LLM Inference](https://arxiv.org/abs/2605.07363)
- **출처**: arXiv 2605.07363 (2026-05-08)
- **핵심 아이디어**: DeepSeek Sparse Attention(DSA)의 인덱서가 많은 헤드(예: DeepSeek-V3.2에서 64개)를 사용해 긴 컨텍스트에서 인덱서 자체가 지배적 계산 비용이 되는 문제를 MoE 라우터로 해결한다. MISA는 DSA 인덱서를 "전문가 풀(mixture-of-experts)"로 취급한다. 경량 라우터가 블록 단위 통계(block-level statistics)로 각 쿼리에 대해 소수의 활성 헤드만 선택하고, 선택된 헤드들만 토큰 단위 중요도 점수를 계산한다. 원본 DSA의 다양한 헤드 풀을 보존하면서 쿼리별 비용을 대폭 줄인다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 긴 컨텍스트에서 인덱서 비용이 전체 어텐션 비용의 지배적 요소가 되는 문제를 MoE 라우팅으로 근본 해소. 기존 DSA 대비 인덱서 계산량 대폭 감소. Activity C의 희소 어텐션 기반 KV 압축에서 "어텐션 헤드 자체의 계산 비용"을 줄이는 새로운 차원을 제시 — 압축 알고리즘의 인덱서가 곧 새로운 병목임을 지적하고 이를 MoE로 해소. Louver(이번 주 Activity B에 등재)의 범위 탐색 기반 제로 거짓 음성 보장과 MISA의 MoE 라우터 기반 인덱서 가속이 결합되면, "정확한 KV 선택(Louver) + 저비용 인덱서(MISA)"의 희소 어텐션 파이프라인이 가능하다. eOptShrinkQ(이미 보고)의 공유 저랭크 성분이 인덱서 헤드 간 공통 정보를 추출한다면, MISA는 쿼리별로 인덱서 헤드를 선택해 헤드별 전문화를 유지하는 상보적 접근이다.

## Activity 간 시너지 포인트

**A + B**
- Tutti의 GPU-centric KV 오브젝트 스토어가 Activity B의 비연속 세그먼트 저장소로 직접 활용 가능하다. 비연속 세그먼트를 각각 GPU 오브젝트로 등록하면 Tutti의 벌크 전송 메커니즘으로 다수의 세그먼트를 병렬로 SSD에서 GPU HBM으로 로드하는 "비연속 세그먼트 GPU-native 벌크 복원"이 가능하다. CacheFlow(이미 보고)의 3D 병렬 복원과 결합 시 토큰·레이어·GPU 3축 병렬화 + GPU io_uring 비동기 I/O의 이중 가속으로 세그먼트 복원 지연을 최소화한다.
- SparKV의 "스트리밍 vs 로컬 계산" 청크 단위 의사결정을 Activity B의 비연속 세그먼트 스케줄러에 통합하면, 캐시 히트가 예상되는 세그먼트는 원격 스트리밍, 미스가 예상되는 세그먼트는 로컬 계산하는 "캐시-인식 SparKV 라우터"가 Activity A+B 통합 스케줄러의 에지 변형으로 가능하다.

**B + C**
- AdapShot의 RoPE 위치 재인코딩이 Activity B의 비연속 세그먼트 재조합에서 MixedDimKV의 차원-압축 세그먼트를 임의 순서로 재배치할 때 발생하는 위치 오류를 무손실로 해소하는 사전 처리 단계가 된다. "MixedDimKV 차원 압축(C) → AdapShot RoPE 재인코딩(B) → 재조합 세그먼트로 추론"하는 B+C 통합 파이프라인이 자연스럽게 도출된다.
- Louver의 제로 거짓 음성 KV 인덱스가 Activity B의 비연속 세그먼트 중요도 판단에서 "중요한 세그먼트를 놓치지 않는" 검색을 보장하고, MISA의 MoE 인덱서가 이 검색을 저비용으로 수행하는 결합이 B+C 통합의 정확도-효율 목표를 동시에 달성한다.
- Sequential PLT(C)를 Activity B의 세그먼트 인덱스로 활용하면, 의미적으로 동등한 비연속 세그먼트를 PLT가 확률 모델로 자동 식별·중복 제거하는 "시맨틱 PLT 세그먼트 인덱스"가 가능하다. 기존 해시 기반 인덱스가 byte-identical 접두사만 인식하는 반면, PLT는 의미적 유사성까지 포괄한다.

**A + C**
- Tutti의 SSD 오프로딩에 MixedDimKV의 혼합-차원 압축을 결합하면, SSD에 저장되는 KV의 전체 차원이 아닌 토큰별 최적 차원만 저장해 SSD 용량 효율이 추가 향상된다. "중요 토큰은 높은 차원, 중요도 낮은 토큰은 낮은 차원으로 SSD에 저장"하는 Tutti + MixedDimKV 통합 스토리지가 Activity A의 "캐시 퇴거율 최소화"에 기여한다.
- SparKV의 어텐션 희소성 신호(MLP 예측기)가 Activity C의 MISA 인덱서 라우터와 연결된다. SparKV가 청크별 어텐션 희소성을 추정할 때 MISA 스타일 MoE 인덱서를 사용하면 경량하면서도 정확한 청크 중요도 예측이 가능하며, 이를 스트리밍 vs 로컬 계산 결정에 활용하는 "MISA-guided SparKV 청크 스케줄러"가 가능하다.

**A + B + C**
- 통합 파이프라인 (장문 컨텍스트 비연속 재사용 + 에지 특화): SparKV 클라우드-에지 하이브리드 KV 로딩(A) → AdapShot RoPE 재인코딩으로 비연속 세그먼트 재조합(B) → MixedDimKV 혼합-차원 압축(C) → Louver 제로 거짓 음성 검색(B) → Tutti GPU-centric SSD 복원(A). 클라우드에서 사전 계산한 MixedDimKV 압축 세그먼트를 SparKV가 에지로 스트리밍하고, AdapShot이 RoPE 재인코딩으로 최적 순서로 재조합하며, Louver가 중요한 KV를 놓치지 않고 검색하는 완전한 에지-클라우드 비연속 재사용 파이프라인.
- Sequential PLT가 이 파이프라인에서 클라우드 세그먼트 저장소에서 세션 간 공통 접두사를 자동 제거해 스트리밍 필요 데이터량을 PLT 압축률만큼 추가 절감하는 프론트엔드 압축기로 작동한다.
