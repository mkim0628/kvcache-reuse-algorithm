# 트렌드 리포트 — 2026-05-03

## 전체 요약

1. **Activity A (스케줄링)**: NVIDIA Dynamo 1.0이 2026-03-16 정식 출시되어 멀티노드 KV-인식 라우팅이 프로덕션 OS 수준으로 격상됐다. DualMap의 이중 해시 매핑이 캐시 친화성(cache affinity)과 부하 균형(load balancing)을 동시에 달성하는 첫 이론적 해법을 제시했고, DualPath(DeepSeek)는 프리필 NIC 포화 문제를 RDMA 스토리지-to-decode 경로로 우회해 1.96× 처리량 향상을 달성했다. Mooncake-Conductor RFC는 KV-캐시 중심 분산 아키텍처의 글로벌 스케줄러 표준을 논의 중이다.
2. **Activity B (비연속 재사용)**: KVShare가 의미 유사도 기반 멀티테넌트 KV 공유로 캐시 히트율 60% 이상 향상 + TTFT 9.39× 감소를 달성했다. StepCache는 스텝 수준 KV 재사용과 경량 검증으로 79.7% 요청에서 재사용-전용 패스트 경로를 실현한다. FreeKV는 투기적 검색(speculative retrieval)으로 기존 KV 검색 대비 최대 13× 속도향상을 달성했다.
3. **Activity C (압축)**: Google의 TurboQuant(ICLR 2026)가 PolarQuant + QJL 2단계 벡터 양자화로 3비트에서 6× 메모리 절감 + 8× 어텐션 연산 가속을 달성하며 정보이론 한계에 근접했다. DepthKV와 StructKV는 각각 레이어별 민감도 기반 예산 할당과 전역 어텐션 in-degree 중심성으로 구조-인식 KV 압축의 새 방향을 제시했다.
4. **A+B 시너지**: DualMap의 SLO-인식 라우팅과 KVShare의 의미 기반 히트율이 결합되면 "의미 히트율 가중 라우팅"이 가능하다.
5. **B+C 시너지**: FreqKV의 주파수 도메인 압축과 FreeKV의 투기적 검색을 결합하면 압축된 KV의 비연속 세그먼트 검색 효율을 동시에 높일 수 있다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [DualMap: Enabling Both Cache Affinity and Load Balancing for Distributed LLM Serving](https://arxiv.org/abs/2602.06502)
- **출처**: arXiv 2602.06502 (2026-02-06)
- **핵심 아이디어**: 각 요청을 두 개의 독립 해시 함수로 두 후보 인스턴스에 매핑하고, 현재 시스템 상태에 따라 더 유리한 인스턴스를 선택한다. SLO-인식 라우팅(TTFT SLO 위반 시 부하 우선 전환), 핫스팟-인식 재균형, 경량 이중 해시링 스케일링 세 기법을 통합한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 기존 캐시 친화성 스케줄링과 부하 균형 스케줄링의 상충 문제를 이중 매핑으로 원리적으로 해결한 첫 논문. 실제 워크로드에서 SOTA 대비 유효 요청 처리 용량 2.25× 향상. Activity A의 "히트율 가중 부하" 복합 스코어 아이디어를 실용적으로 구현한 레퍼런스로 활용 가능.

#### 2. [DualPath: Breaking the Storage Bandwidth Bottleneck in Agentic LLM Inference](https://arxiv.org/abs/2602.21548)
- **출처**: arXiv 2602.21548 (2026-02-26, DeepSeek AI / 북경대 / 청화대)
- **핵심 아이디어**: 에이전틱 추론에서 KV 캐시 스토리지 I/O가 컴퓨팅보다 먼저 병목이 되는 문제를 해결한다. 기존 스토리지→프리필 경로에 더해 스토리지→디코드 경로를 추가하고, 디코드 엔진이 KV를 먼저 로드한 후 RDMA로 프리필 엔진에 전달한다. 전용 NIC 트래픽 격리 + 동적 부하 균형 스케줄러를 결합한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: DeepSeek RL 훈련에서 98.7% KV 캐시 히트율 달성 경험 기반. 오프라인 처리량 1.87×, 온라인 서빙 1.96× 개선. 멀티노드 KV 라우팅 설계에서 "스토리지 NIC 포화"라는 새로운 병목 유형을 발굴한 점이 novelty.

#### 3. [NVIDIA Dynamo 1.0: Production-Ready Distributed Inference OS](https://developer.nvidia.com/blog/nvidia-dynamo-1-production-ready/)
- **출처**: NVIDIA 기술 블로그 (2026-03-16 공식 출시)
- **핵심 아이디어**: 단일 GPU부터 수천 GPU 클러스터까지 KV 캐시 인식 라우팅, P/D 분리, KV 블록 관리(KVBM), 오브젝트 스토리지 계층화, 전역 KV 이벤트 방출을 통합한 오픈소스 추론 OS. GTC 2026에서 GB200 NVL72 Blackwell에서 7× 처리량 향상 시연.
- **연구 목표와의 관련성**: high
- **주목할 점**: vLLM·SGLang·llm-d·TensorRT-LLM·LMCache 등 주요 프레임워크와 직접 통합. KVBM이 KV 블록을 GPU HBM → CPU DRAM → NVMe → 오브젝트 스토리지로 자동 계층화하며 전역 KV 이벤트 스트림을 방출 — Activity A의 멀티노드 KV 상태 추적 인프라로 직접 활용 가능.

#### 4. [Mooncake-Conductor: Global Scheduler for KV-Cache-Centric Disaggregated Arch](https://github.com/kvcache-ai/Mooncake/issues/977)
- **출처**: GitHub kvcache-ai/Mooncake RFC (2026, 진행 중)
- **핵심 아이디어**: Mooncake KV-캐시 중심 분산 아키텍처에서 "brain" 역할을 하는 글로벌 스케줄러 모듈을 공식 설계·구현하는 RFC. 캐시-인식 + 부하 균형 + 예측적 스케줄링을 통합한 공식 표준 스케줄링 정책을 목표로 한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Mooncake가 이미 vLLM v1 KV Connector, SGLang Mooncake Store로 프로덕션 통합된 상황에서 글로벌 스케줄러가 추가되면 비연속 세그먼트 인식 라우팅(Activity B)을 Mooncake 생태계에 자연스럽게 삽입할 수 있다.

#### 5. [TraCT: Disaggregated LLM Serving with CXL Shared Memory KV Cache at Rack-Scale](https://arxiv.org/abs/2512.18194)
- **출처**: arXiv 2512.18194 (2025-12-20)
- **핵심 아이디어**: CXL 공유 메모리를 KV 전송 기판(substrate)이자 랙-와이드 접두사-인식 KV 캐시로 활용한다. GPU가 CXL load/store 및 DMA로 KV 블록을 직접 읽고 써서 NIC 홉(hop)을 완전 제거한다. NVIDIA Dynamo 프레임워크 위에 구현.
- **연구 목표와의 관련성**: high
- **주목할 점**: RDMA 및 DRAM 기반 캐싱 대비 TTFT 9.8×, P99 지연 6.2×, 피크 처리량 1.6× 개선. CXL 메모리가 RDMA의 대안으로 부상하는 신호 — Activity A의 네트워크-인식 KV 경로 선택에 CXL을 추가 고려해야 할 시점.

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [KVShare: An LLM Service System with Efficient and Effective Multi-Tenant KV Cache Reuse](https://arxiv.org/abs/2503.16525)
- **출처**: arXiv 2503.16525 (2026-03-17, v2: 2026-05-16)
- **핵심 아이디어**: 텍스트 접두사 매칭의 한계를 넘어 의미 유사도 기반 KV 공유를 실현한다. Dual-Stage High Deviation(DHD) 알고리즘이 프리필과 디코드 양 단계에서 편차가 큰 KV 소수만 선택 재계산하고, 캐시-인식 스케줄러가 KV 히트율 기반으로 요청 우선순위를 결정하여 연속 배칭을 오케스트레이션한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 실제 사용자 대화 데이터셋에서 KV 히트율 60% 이상 향상, TTFT 9.39× 감소, SOTA 대비 정확도 20.38% 향상. Qwen2.5-7B, Llama3.1-8B, Yi1.5-9B에서 검증. 의미 기반 비연속 재사용이 Activity B의 핵심 - 비정형 RAG 시나리오에서 "접두사 불일치라도 의미적으로 유사하면 재사용"하는 패러다임이 확립됨.

#### 2. [StepCache: Step-Level Reuse with Lightweight Verification and Selective Patching for LLM Serving](https://arxiv.org/abs/2603.28795)
- **출처**: arXiv 2603.28795 (2026-03-24)
- **핵심 아이디어**: 이전 출력을 스텝(step) 단위로 분할하고, 캐시된 요청과 최고-매칭 요청을 찾아 캐시된 스텝을 경량 태스크-인식 검증기로 확인한 후 실패 구간만 연속 블록 패칭(contiguous block patching)으로 재생성한다. 모델 내부 KV 상태에 의존하지 않아 백엔드-비종속적(backend-agnostic)으로 동작.
- **연구 목표와의 관련성**: high
- **주목할 점**: 전체 요청의 79.7%가 재사용-전용 패스트 경로를 취하고 5.4%만 패칭 필요. 평균 레이턴시 2.13s → 0.67s (중앙값 2.42s → 0.01s) 감소. 동일 솔루션 구조를 공유하지만 지역적 제약이 다른 반복 요청(코드 생성, JSON 출력 등)에 특화. Activity B의 비연속 재사용을 "출력 레벨"로 확장하는 새로운 관점.

#### 3. [FreeKV: Boosting KV Cache Retrieval for Efficient LLM Inference](https://arxiv.org/abs/2505.13109)
- **출처**: arXiv 2505.13109 (2025-05, OpenReview 채택)
- **핵심 아이디어**: 알고리즘 측: 투기적 검색(speculative retrieval)으로 KV 선택·복원 과정을 크리티컬 패스 밖으로 이동시키고, 세밀한 보정(fine-grained correction)으로 정확도 유지. 시스템 측: CPU-GPU 하이브리드 KV 레이아웃으로 단편화 전송 제거, 이중 버퍼 스트리밍 복원으로 효율 향상.
- **연구 목표와의 관련성**: high
- **주목할 점**: SOTA KV 검색 방법 대비 최대 13× 속도향상, near-lossless 정확도 유지. Activity B의 비연속 세그먼트 검색이 크리티컬 패스에서 제외되면 TTFT 오버헤드 목표(+5% 이내)를 달성하기 훨씬 용이해짐. 오픈소스(github.com/sjtu-zhao-lab/FreeKV).

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Segmented Hash / Radix | CacheFlow 3D 복원 프레임워크가 비연속 세그먼트 복원 병목 해소 | StepCache가 출력-레벨 스텝 단위 재사용으로 Activity B 적용 범위를 "KV 내부"에서 "KV 외부"로 확장 |
| CacheBlend / Blending / KVLink | RelayCaching(훈련-무료)이 디코딩 KV의 직접 재주입으로 경쟁 진입 | KVShare DHD 알고리즘이 의미 기반 KV 선택 재계산으로 정확도 대폭 향상 |
| Position-independent caching | PolyKV 멀티에이전트 공유 풀 방식 등장 | FreeKV 투기적 검색이 KV 검색 지연을 크리티컬 패스에서 제거하는 시스템 패러다임 제시 |

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [TurboQuant: Online Vector Quantization with Near-optimal Distortion Rate](https://arxiv.org/abs/2504.19874)
- **출처**: arXiv 2504.19874 / ICLR 2026 (Google Research / NYU / KAIST)
- **핵심 아이디어**: 2단계 벡터 양자화 파이프라인. 1단계 PolarQuant: KV 벡터에 랜덤 회전 행렬을 적용해 분산을 균일하게 재분배한 후 스칼라 양자화. 2단계 QJL: 잔차 양자화 오류를 1비트 Quantized Johnson-Lindenstrauss 변환으로 보정. 훈련·캘리브레이션·파인튜닝 불필요.
- **연구 목표와의 관련성**: high
- **주목할 점**: 3비트에서 6× 메모리 절감, H100에서 8× 어텐션 연산 가속, 모델 정확도 완전 보존. 정보이론 최적 한계의 ~2.7배 내에서 동작하여 기존 SAW-INT4·VQKV 대비 이론적 완성도가 가장 높음. Gemma, Mistral에서 검증됨. Activity C의 "accuracy ±1% 이내" 제약을 3비트에서도 달성하는 새 기준선.

#### 2. [DepthKV: Layer-Dependent KV Cache Pruning for Long-Context LLM Inference](https://arxiv.org/abs/2604.24647)
- **출처**: arXiv 2604.24647 (2026-04-27)
- **핵심 아이디어**: 기존 방법들이 모든 레이어에 균일 압축률을 적용하는 문제를 해결한다. 레이어별 민감도에 따라 고정 전역 KV 예산을 차등 할당: 위치-의존 보호(중간 레이어 보존), 메트릭-유도 할당, 혼합 전략을 지원하는 통합 프레임워크.
- **연구 목표와의 관련성**: high
- **주목할 점**: 동일 메모리 예산 하에서 LongBench 및 RULER 벤치마크에서 균일 KV 압축을 일관되게 초과. Activity C의 "레이어별 중요도 인식 압축" 방향을 체계화하며, SAW-INT4나 TurboQuant 같은 양자화 방법과 직교 결합 가능.

#### 3. [CAOTE: KV Cache Selection for LLMs via Attention Output Error-Based Token Eviction](https://arxiv.org/abs/2504.14051)
- **출처**: arXiv 2504.14051 (2026-04, v5까지 업데이트)
- **핵심 아이디어**: 기존 어텐션 스코어 기반 퇴거의 한계를 지적하고, 토큰이 어텐션 출력에 기여하는 정도를 닫힌 형태(closed-form)로 계산하는 새 퇴거 기준을 제안한다. 어텐션 스코어와 Value 벡터를 동시에 활용한 최초의 방법.
- **연구 목표와의 관련성**: high
- **주목할 점**: 기존 어텐션 스코어 기반 방법들과 결합 시 항상 정확도 개선. LLAMA3, QWEN2.5에서 검증. Activity C의 "정확도 보존" 제약 달성에서 Value 정보를 무시했던 기존 방법들의 맹점을 해소하는 핵심 보완책.

#### 4. [FastKV: Decoupling of Context Reduction and KV Cache Compression for Prefill-Decoding Acceleration](https://arxiv.org/abs/2502.01068)
- **출처**: arXiv 2502.01068 / ACL Findings 2026 (GitHub: dongwonjo/FastKV)
- **핵심 아이디어**: 프리필 컨텍스트 압축과 디코딩 KV 보존을 완전히 분리한다. 초기 레이어는 전체 컨텍스트를 처리하고, 안정화 시점 이후 레이어부터 핵심 토큰만 전파하는 Token-Selective Propagation(TSP) 2단계 프리필 전략을 사용한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 프리필 1.82×, 디코딩 2.87× 가속, 디코딩-전용 가속 방법과 동등한 정확도. 프리필 압축 예산과 디코딩 KV 예산을 독립 조정할 수 있어 Activity C의 다양한 메모리 예산 시나리오에 유연하게 적용 가능.

#### 5. [StructKV: Preserving the Structural Skeleton for Scalable Long-Context Inference](https://arxiv.org/abs/2604.06746)
- **출처**: arXiv 2604.06746 (2026-04-08)
- **핵심 아이디어**: 로컬 현저성(saliency) 메트릭으로 토큰을 선택하는 기존 방법이 레이어 전반의 "글로벌 정보 허브" 토큰을 놓치는 문제를 해결한다. Global In-Degree Centrality로 네트워크 깊이 전반의 어텐션 패턴을 집계하고, Dynamic Pivot Detection으로 최적 압축 레이어를 정보이론 메트릭으로 탐색한다. 계산 예산과 메모리 예산을 분리(Structural Propagation and Decoupling)한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: LongBench·RULER에서 일관된 우위. Activity B의 세그먼트 중요도 점수에 글로벌 in-degree centrality를 적용하면 비연속 세그먼트 선택의 이론적 기반이 강화됨. CapKV(레버리지 스코어), DepthKV(레이어별 예산)와 함께 구조-인식 압축 3부작을 형성.

#### 6. [FreqKV: Key-Value Compression in Frequency Domain for Context Window Extension](https://arxiv.org/abs/2505.00570)
- **출처**: arXiv 2505.00570 (2025-05, OpenReview 채택)
- **핵심 아이디어**: KV 상태를 주파수 도메인으로 변환하면 에너지가 저주파 성분에 집중됨을 관찰하고, 고주파 성분을 필터링해 KV를 압축한다. 반복적 압축 단계마다 오래된 컨텍스트의 압축률을 높이고 최신 토큰은 덜 압축하는 점진적 방식. 파라미터 추가 없음.
- **연구 목표와의 관련성**: high
- **주목할 점**: 8K 길이 훈련 후 LLaMA-2-7B의 컨텍스트 윈도우를 256K로 확장하면서 perplexity 안정 유지. Activity C의 "동일 메모리에서 더 긴 컨텍스트 지원" 목표(유효 컨텍스트 2× 이상)와 직결. 기존 양자화·퇴거와 직교 결합 가능.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Quantization (INT4/FP8) | PolyKV 비대칭 압축(Key INT8 + Value 3-bit FWHT) 조합이 새 벤치마크 | TurboQuant(ICLR 2026)이 3비트 PolarQuant+QJL로 정보이론 한계 내 근접, 새로운 압축 기준 제시 |
| Token eviction (H2O, SnapKV) | CapKV 정보이론 기반, LookaheadKV ICLR 2026 채택 | CAOTE가 Value 기여도 포함한 어텐션 출력 오류 기반 퇴거로 기존 방법 일관 개선 |
| Low-rank / structure-aware | ReCalKV K/V 비대칭 저랭크(HSR+OVC) 체계적 K/V 분리 처리 표준 제시 | StructKV 글로벌 in-degree centrality가 레이어 교차 구조 정보 보존 프레임워크 제시 |
| Frequency-domain compression | 실험적 연구 단계 | FreqKV가 8K→256K 컨텍스트 확장에서 perplexity 안정 유지, 실용화 가능성 입증 |

## Activity 간 시너지 포인트

**A + B**
- DualMap의 이중 해시 매핑 기반 라우팅에서 "히트율"을 단순 접두사 매칭이 아닌 KVShare의 의미 유사도 기반 히트율로 대체하면 "의미 히트율 가중 이중 해시 라우팅"이 가능하다. 부분 의미 히트도 라우팅 스코어에 포함시켜 비연속 세그먼트 재사용률을 극대화할 수 있다.
- DualPath의 스토리지-to-decode RDMA 경로가 StepCache의 스텝별 KV를 노드 간으로 전달하는 전송 기반이 될 수 있다. 에이전틱 파이프라인에서 스텝 단위로 캐시된 KV를 RDMA로 공유하는 조합이 가능하다.

**B + C**
- FreqKV의 주파수 도메인 압축과 FreeKV의 투기적 검색을 결합하면 "압축된 주파수-도메인 KV 세그먼트를 비동기적으로 미리 검색"하는 파이프라인이 가능하다. 검색 지연이 크리티컬 패스 밖에 있으므로 압축 오버헤드도 숨길 수 있다.
- CAOTE의 어텐션 출력 오류 기반 퇴거를 Activity B의 세그먼트 중요도 점수에 적용하면 "비연속 세그먼트 중 어텐션 출력 기여도가 낮은 세그먼트"를 우선 퇴거하는 이론적으로 더 엄밀한 B+C 정책이 가능하다.

**A + C**
- NVIDIA Dynamo 1.0의 전역 KV 이벤트 스트림에 TurboQuant 압축 상태(3비트 KV)를 태그로 추가하면 스케줄러가 "압축된 KV를 가진 노드 vs 원본 KV를 가진 노드" 중 정확도 요구사항에 따라 라우팅 결정을 내릴 수 있다.
- DepthKV의 레이어별 예산 할당 결과를 스케줄러가 메모리 압력 신호로 수신하면, 레이어별 압축률이 높아지는 시점을 예측해 선제적 KV 전송·퇴거를 결정할 수 있다.

**A + B + C**
- 통합 파이프라인: KVShare 의미 유사도 기반 비연속 세그먼트 매칭(B) → TurboQuant 3비트 압축(C) → DualMap 이중 해시 라우팅 또는 Dynamo 1.0 KV-인식 라우팅(A). 의미 히트 KV를 3비트로 압축 후 최적 노드로 라우팅하는 완전한 A+B+C 통합 경로.
- FreqKV 기반 장문 컨텍스트(256K)를 FreeKV 투기적 검색으로 비연속 세그먼트를 추출하고, DualPath RDMA로 노드 간 공유하는 시나리오: 극장문 컨텍스트 + 비연속 재사용 + 멀티노드 공유를 동시에 달성.

## idea-generator에게 전달할 포인트

### Activity A
- **DualMap의 핵심 교훈**: 두 후보 인스턴스 중 선택하는 이중 매핑 접근이 캐시 친화성-부하 균형 상충을 해소한다. Activity A 구현 시 단일 최적 노드를 탐색하는 대신 이중 후보 + 런타임 선택 구조를 고려할 것.
- **DualPath의 발견**: 에이전틱 워크로드에서 KV 스토리지 I/O가 실제 병목임을 실증. Activity A의 멀티노드 스케줄링 설계 시 컴퓨팅 병목이 아닌 I/O 병목 시나리오를 별도 모델링할 것.
- **Dynamo 1.0**: 전역 KV 이벤트 스트림(KVBM 방출)이 Activity A의 멀티노드 KV 상태 추적 인프라로 즉시 활용 가능. 오픈소스이므로 vLLM 이식(6단계)에서 Dynamo KV Router를 참조 구현으로 채택할 것.

### Activity B
- **KVShare**: 의미 유사도 기반 KV 공유 + DHD 선택 재계산이 Activity B의 "접두사 불일치 시 비연속 재사용" 목표와 가장 직접적으로 일치하는 최신 구현. TTFT 9.39× 감소는 Activity B 목표 지표 달성의 새 벤치마크.
- **StepCache의 접근**: 출력-레벨 스텝 재사용은 KV-레벨 비연속 재사용과 다른 층위이지만, "가장 매칭되는 캐시 요청 검색 + 부분 재생성" 패턴이 Activity B 세그먼트 매칭의 구현 참조가 된다.
- **FreeKV의 시스템 최적화**: 비연속 KV 검색을 크리티컬 패스 밖으로 이동시키는 투기적 검색 패턴을 Activity B에 적용하면 TTFT +5% 이내 목표 달성이 가능해진다.

### Activity C
- **TurboQuant(ICLR 2026)**: 훈련-무료 3비트 압축으로 정보이론 한계 ~2.7배 내에서 정확도 완전 보존. Activity C의 최우선 구현 후보로 격상. vLLM 통합 구현(github.com/0xSero/turboquant)이 이미 존재해 6단계 이식 비용 최소화 가능.
- **CAOTE**: Value 기여도를 퇴거 기준에 포함시키는 것이 "정확도 보존 퇴거"의 필수 구성요소임이 확인됨. 기존 H2O·SnapKV에 CAOTE 오류 기준을 결합하면 Activity C의 ±1% perplexity 제약 달성 가능성이 높아짐.
- **FreqKV + DepthKV 조합**: 주파수 도메인 압축(FreqKV)으로 장문 컨텍스트 지원 확장(목표: 2× 이상)하고 레이어별 차등 예산(DepthKV)으로 메모리 절감(목표: -30% 이상)하는 두 목표를 병렬로 달성하는 조합이 유망.
