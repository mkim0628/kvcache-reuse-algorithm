# 트렌드 리포트 — 2026-05-15

## 전체 요약

1. **Activity A (스케줄링)**: 분산 KV 캐시 풀이 에이전틱 서빙의 핵심 인프라로 확립됐다. vLLM x Mooncake Store 블로그(2026-05-06)가 80K+ 토큰 에이전틱 트레이스에서 캐시 히트율 1.7% → 92.2%, TTFT 46× 단축을 실증하며 "단일 인스턴스 로컬 KV" 한계를 공식 선언했다. Feather(2605.06046)는 배치 크기 대 프리픽스 동질성 간의 근본 트레이드오프를 처음으로 형식화하고, 강화학습 기반 배치 중단 정책으로 균형을 자동 결정한다.

2. **Activity B (비연속 재사용)**: 다중 에이전트·다중 모델 환경에서 KV 캐시를 크로스-에이전트, 크로스-모델로 공유하는 방법론이 급속히 성숙하고 있다. RelayCaching(2603.13289)은 이전 에이전트의 디코딩 KV를 다음 에이전트의 프리필에 직접 재사용해 4.7× TTFT 단축을 훈련 없이 달성한다. ICaRus(2603.13281)는 아키텍처 수준에서 복수 모델이 동일 KV 캐시를 공유하도록 파인튜닝 구조를 재설계해 8-모델 멀티에이전트에서 3.8× 처리량을 달성한다. QKVShare(2605.03884)는 에지 기기의 에이전트 간 양자화 KV 핸드오프를 온-디바이스 맥락에서 최초로 실현한다.

3. **Activity C (압축)**: LookaheadKV(2603.10899, ICLR 2026)가 학습 가능한 룩어헤드 토큰으로 미래 응답의 어텐션 패턴을 예측하는 새로운 KV 퇴거 패러다임을 제시했다. 기존 Lookahead 계열이 비용이 큰 드래프트 생성을 요구한 반면, LookaheadKV는 파라미터-효율적 모듈로 14.5× 오버헤드 절감을 달성한다. LMCache AMD MI300X 벤치마크(2026-05-12)는 CPU-DRAM KV 오프로딩이 에이전틱 실제 트레이스에서 HBM-only 대비 3.0× TTFT 개선을 실증하며 AMD 에코시스템 지원을 확인했다.

4. **A+B 시너지**: vLLM x Mooncake Store의 분산 KV 풀(A)과 RelayCaching의 크로스-에이전트 KV 재사용(B)이 결합되면, 분산 풀에 저장된 이전 에이전트 디코딩 KV를 다음 에이전트가 직접 참조하는 "분산-분산 크로스-에이전트 KV 공유" 시스템이 가능하다.

5. **B+C 시너지**: QKVShare의 에지 양자화 KV 핸드오프(B+C 중간)와 ICaRus의 크로스-모델 KV 공유(B)가 결합되면, 에지에서 복수 전문화 모델이 양자화 공유 KV로 작동하는 에지-특화 멀티모델 비연속 KV 시스템이 가능하다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [Serving Agentic Workloads at Scale with vLLM x Mooncake](https://vllm.ai/blog/2026-05-06-mooncake-store)
- **출처**: vLLM 공식 블로그 (2026-05-06, vLLM 팀 + Moonshot AI / InferAct)
- **핵심 아이디어**: 에이전틱 트레이스가 80K+ 토큰으로 성장하고 94%+ 재사용 가능한 프리픽스를 가지지만, 단일 vLLM 인스턴스의 로컬 KV 캐시는 이를 퇴거시키고 크로스-인스턴스 라우팅은 이를 놓친다는 두 가지 구조적 한계를 해결한다. Mooncake Store를 분산 KV 캐시 풀로 vLLM에 통합해, 복수의 vLLM 인스턴스가 Mooncake 클라이언트를 임베드하고 클러스터-와이드 Mooncake Store를 공유한다. Mooncake 마스터가 KV 블록 메타데이터와 서비스 디스커버리를 관리하고, 워커가 RDMA를 통해 KV 블록을 GPU HBM과 분산 DRAM/SSD 풀 사이에 전송한다. 캐시 히트 시 GPU HBM에서 직접 로드하고 미스 시 DRAM/SSD에서 복원하는 계층 구조다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 현실적 에이전틱 트레이스에서 캐시 히트율 1.7% → 92.2% 향상. TTFT 46× 단축, E2E 지연 8.6× 단축, 처리량 3.8× 향상을 60 GB200 GPU 환경에서 거의 선형 스케일로 달성. Activity A의 "캐시 퇴거율 최소화" 및 "TTFT p50 +5% 이내" 목표를 분산 풀로 에이전틱 워크로드에서 실증한 첫 대규모 실사례. 기존 Tutti(SSD 오프로딩 GPU-centric)가 단일 인스턴스 내 SSD 계층화를 다뤘다면, 이 통합은 인스턴스 경계를 넘어 클러스터 수준 KV 풀을 구성하는 근본적으로 다른 수준의 해결책이다. PBKV(이미 보고)의 예측 기반 에이전트 KV 관리와 결합 시 "예측기가 클러스터 KV 풀에서 어느 인스턴스의 캐시를 재사용할지 결정"하는 클러스터-인식 에이전트 스케줄러가 가능하다. SGLang Q2 2026 로드맵의 "에이전틱 분산 KVCache 시스템"이 이와 동일한 방향으로 수렴 중임을 확인.

#### 2. [Requests of a Feather Must Flock Together: Batch Size vs. Prefix Homogeneity in LLM Inference (Feather)](https://arxiv.org/abs/2605.06046)
- **출처**: arXiv 2605.06046 (2026-05-09)
- **핵심 아이디어**: 프리픽스 공유 워크로드에서 배치 크기와 프리픽스 동질성(prefix homogeneity) 사이의 근본 트레이드오프를 최초로 형식화한다. 직관에 반하여, 모든 요청이 동일 프리픽스를 공유하는 작은 동질 배치가 더 큰 이종 배치보다 높은 디코딩 처리량을 달성할 수 있음을 실험으로 입증한다. 더 큰 배치는 GPU 이용률을 높이지만 공유 프리픽스 정렬을 감소시켜 KV 캐시 지역성이 저하된다. Feather 스케줄러는 배치 구성을 순차적 의사결정 과정으로 모델링하고, 강화학습(RL)으로 배치 크기 대 프리픽스 동질성 사이의 최적 중단 정책을 학습한다. Chunked Hash Tree(C++로 구현)로 공유 프리픽스 체인을 추적하고, 스케줄러 수준에서 동작해 기존 시스템 변경 없이 플러그인으로 적용 가능하다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 배치 구성 시점에 프리픽스 동질성을 스케줄링 신호로 활용한다는 개념이 기존 시스템(vLLM, SGLang)에 명시적으로 없었던 새로운 차원을 추가한다. 동질 소배치가 더 빠른 TBT(time-between-tokens)를 달성하는 반면 이종 대배치는 처리량에서 유리한 복잡한 트레이드오프를 RL로 자동 해결. Activity A의 "prefix-aware batching" 방향에서 기존 방법(가장 긴 프리픽스 매치, DFS 가중치 등)이 정적 휴리스틱인 반면, Feather는 학습된 동적 배치 중단 정책이라는 근본적 차별화가 있다. Helium(이미 보고)의 쿼리-플랜 스케줄러가 사전-정의 구조를 활용했다면, Feather는 구조 없이 배치 동질성 신호만으로 작동한다. Activity B의 비연속 세그먼트 재사용에서 "동일 세그먼트를 공유하는 요청들을 Feather로 배치"하는 세그먼트-동질 배칭이 히트율을 극대화하는 방향이 가능하다.

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [RelayCaching: Accelerating LLM Collaboration via Decoding KV Cache Reuse](https://arxiv.org/abs/2603.13289)
- **출처**: arXiv 2603.13289 (2026-02-28, Yingsheng Geng, Yuchong Gao, Weihong Wu, Guyue Liu, Jiang Liu)
- **핵심 아이디어**: 다중 에이전트 LLM 협업에서 각 에이전트가 이전 에이전트가 생성한 공유 콘텐츠를 프리필 단계에서 처음부터 재계산해야 하는 이중 부담 — 높은 TTFT와 KV 메모리 폭발 — 을 해결한다. 핵심 통찰: 동일 콘텐츠에 대한 KV 캐시는 프리필/디코딩 단계 간 고도로 일관되며, 프리픽스-유도 편차는 드물고 제한된 레이어·토큰 위치에 국소화된다. RelayCaching은 두 가지 핵심 구성 요소로 이를 활용한다: (1) **레이어-범위 프로파일러** — U자형 프로파일을 이용해 중요 레이어 범위를 식별하고 레이어 간 상관관계로 탐지 레이어를 선택, (2) **토큰 선택기** — 편차 기반 + 영향 기반 선택을 결합해 재계산이 필요한 토큰을 정확히 식별. 이 최소 재계산 세트만 선택적으로 갱신하고 나머지는 이전 에이전트의 디코딩 KV를 그대로 재사용한다. 훈련 없이 적용 가능하다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 수학 추론·일반 지식·코드 생성 등 다양한 태스크에서 표준 프리필 파이프라인과 동등한 생성 품질을 유지하면서 80% 이상 KV 캐시 재사용, 최대 4.7× TTFT 단축 달성. 기존 prefix caching이 byte-identical 프리픽스에만 작동하는 반면, RelayCaching은 비동일 콘텐츠(이전 에이전트 디코딩 출력)에 대해서도 KV 재사용을 가능하게 한다 — Activity B의 "비연속·비접두사 KV 재사용"의 정확한 실용 사례. 프리픽스-유도 편차가 U자형으로 국소화된다는 발견은 비연속 세그먼트 재사용 시 어떤 레이어/위치에서 재계산이 필요한지를 예측하는 "선택적 세그먼트 갱신 정책" 설계에 직접 활용 가능하다. AdapShot(이미 보고)의 RoPE 위치 재인코딩과 결합 시 "RelayCaching이 재사용할 KV를 선택하고, AdapShot이 위치 재인코딩으로 새 위치에 배치"하는 결합 파이프라인이 에이전트 간 비연속 KV 재사용의 정확도를 높인다.

#### 2. [ICaRus: Identical Cache Reuse for Efficient Multi Model Inference](https://arxiv.org/abs/2603.13281)
- **출처**: arXiv 2603.13281 (2026-02-27, Sunghyeon Woo, Jaeeun Kil 외 10인 — NAVER Cloud)
- **핵심 아이디어**: 다중 모델 추론(multi-model inference)에서 각 전문화 모델이 동일 프롬프트에 대해 자체 KV 캐시를 유지해야 해 메모리가 폭발적으로 증가하고, 기존 프리픽스 캐싱은 서로 다른 모델 간에 적용할 수 없다는 근본 한계를 아키텍처 수준에서 해결한다. 핵심 통찰: 디코더-온리 트랜스포머를 "논리적 인코더(KV 캐시 생성)" + "논리적 디코더(토큰 예측)"로 개념 분해하고, 논리적 인코더를 동결(freeze)한 채 논리적 디코더만 파인튜닝한다. 이 설계가 여러 태스크-특화 모델들이 동일한 KV 캐시를 모든 레이어에 걸쳐 공유할 수 있게 한다. LoRA 같은 경량 어댑터로 논리적 디코더를 각 태스크에 특화하고, 디코딩 중 KV 캐시 생성과 다음-토큰 예측을 병렬화한다. 모델이 공유 KV 캐시에 새 입력 토큰을 추가 처리할 때도 크로스-모델 KV 재사용을 허용해 리-프리필 오버헤드를 제거한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 8개 다른 모델이 참여하는 멀티에이전트 워크플로우에서 3.8× 처리량 향상. 각 태스크-특화 파인튜닝 모델과 동등한 정확도를 유지하면서 KV 캐시 폭발 문제를 아키텍처 수준에서 근본 제거. Activity B의 "비연속 KV 재사용"에서 기존 접근이 "동일 모델의 다른 요청 간" 재사용에 집중한 반면, ICaRus는 "다른 모델들 간" KV 재사용이라는 새로운 차원을 최초로 열었다. 멀티모델 에이전틱 워크플로우에서 8 모델이 동일 KV를 공유하므로 비연속 세그먼트 저장 공간이 1/8로 감소하는 극단적 메모리 효율이 달성된다. PBKV(이미 보고)의 예측 기반 에이전트 호출 예측이 ICaRus의 공유 KV 풀에서 어느 태스크의 어댑터를 활성화할지 결정하는 "PBKV-guided ICaRus 어댑터 선택기"로 결합 가능하다. OpenReview 채택 기록이 있어 상위 학회 논문으로 확인됨.

#### 3. [QKVShare: Quantized KV-Cache Handoff for Multi-Agent On-Device LLMs](https://arxiv.org/abs/2605.03884)
- **출처**: arXiv 2605.03884 (2026-05-05)
- **핵심 아이디어**: 에지 기기의 다중 에이전트 LLM 시스템에서 에이전트 간 컨텍스트를 전달할 때 전체 재-프리필(costly)과 전체 정밀도 KV 전송(high memory) 사이의 비효율적 선택만 존재하는 문제를 해결한다. QKVShare는 세 가지 구성 요소를 통합한다: (1) **토큰-레벨 혼합 정밀도 할당** — 토큰 중요도에 따라 KV를 차별화된 비트폭으로 양자화해 핵심 토큰은 고정밀도, 덜 중요한 토큰은 저정밀도로 저장, (2) **자기-완결 CacheCard 표현** — 양자화 메타데이터를 캐시 페이로드에 패키징해 수신 에이전트가 의존성 없이 즉시 주입 가능, (3) **HuggingFace 호환 캐시 주입 경로** — 기존 모델 코드 변경 없이 CacheCard를 모델 KV 캐시에 직접 주입. 균일 양자화 대비 적응형 양자화가 더 깊은 호핑·높은 예산 설정에서 뚜렷한 우위를 보인다.
- **연구 목표와의 관련성**: high
- **주목할 점**: GSM8K 150개 문제, Llama-3.1-8B-Instruct에서 적응형 양자화가 반복 핸드오프에서 균일 양자화 대비 경쟁력 있는 정확도 유지. QKVShare 경로가 전체 재-프리필 대비 모든 컨텍스트 길이에서 TTFT 단축. Activity B의 비연속 KV 재사용 문제를 에지 기기에서 에이전트 간 KV 전달이라는 구체적 시나리오로 실현하는 최초 온-디바이스 접근. RelayCaching(이번 주, B)이 서버 환경에서 80% 이상 KV 재사용을 달성했다면, QKVShare는 에지 기기에서 양자화-압축 KV 전달로 유사한 효과를 얻는 상보적 방향이다. When Quantization Is Free(이미 보고, C)의 SRFT+INT4 퓨즈드 커널이 QKVShare의 토큰-레벨 양자화 백엔드로 결합되면 에지에서 속도와 정밀도를 동시에 최적화할 수 있다.

#### 1. [LookaheadKV: Fast and Accurate KV Cache Eviction by Glimpsing into the Future without Generation](https://arxiv.org/abs/2603.10899)
- **출처**: arXiv 2603.10899 / ICLR 2026 포스터 (Samsung AI Research)
- **핵심 아이디어**: KV 캐시 퇴거 정확도를 높이기 위해 미래 응답을 "엿보는(glimpsing)" 방법들이 드래프트 생성에 의존해 상당한 프리필 오버헤드를 유발하는 문제를 해결한다. LookaheadKV는 드래프트 생성 없이 미래 응답의 어텐션 패턴을 직접 예측하는 파라미터-효율적 모듈을 도입한다: (1) **룩어헤드 토큰(Lookahead Tokens)** — 각 어텐션 헤드의 쿼리로 사용되도록 설계된 훈련 가능한 소프트 토큰 집합을 입력 시퀀스 끝에 추가, (2) **룩어헤드 LoRA** — 각 레이어에서 룩어헤드 쿼리가 실제 모델 응답의 어텐션 패턴과 정렬되도록 훈련되는 경량 LoRA 어댑터. 프리필 단계에서 이 모듈이 토큰 중요도를 예측하고 KV를 퇴거한 후, 보존된 KV로 자기회귀 디코딩을 수행한다. 퇴거 결정이 미래 응답 패턴에 기반하므로 현재 어텐션 통계보다 근본적으로 더 정확한 퇴거가 가능하다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 퇴거 비용을 최대 14.5× 절감해 TTFT를 크게 단축하면서 기존 비싼 근사 방법보다 높은 정확도 달성. 기존 inexpensive heuristic(H2O, SnapKV 등)와 동등한 런타임 오버헤드로 훨씬 우수한 퇴거 품질 보장. Activity C의 "정확도 ±1% 이내" 제약과 "메모리 −30% 이상" 목표를 ICLR 2026 학회 논문으로 검증한 첫 미래-인식 퇴거 프레임워크. LaProx(이미 보고)의 출력-인식 레이어 단위 중요도 점수가 현재 어텐션 통계로 중요도를 계산하는 반면, LookaheadKV는 미래 응답 패턴 예측으로 중요도를 계산하는 직교적 접근 — 두 방법을 결합해 "레이어 단위 현재 + 미래-예측 복합 중요도 점수"를 만드는 통합이 가능하다. LoRA 어댑터만 추가하면 기존 모델 가중치 변경 없이 배포 가능하므로 vLLM 이식 친화성이 높다. LKV(이미 보고)의 학습 기반 예산+선택과 결합 시 "LKV가 헤드별 예산을 결정, LookaheadKV가 미래-인식 선택을 실행"하는 학습-예측 이중 최적화 퇴거 파이프라인이 가능하다.

### Activity B
- **RelayCaching(2603.13289) — 비접두사 에이전트 간 KV 재사용**: 에이전트 디코딩 KV를 다음 에이전트 프리필에 직접 재사용하는 원칙이 Activity B의 핵심 문제 — "byte-identical 접두사 없이도 KV를 재사용할 수 있는가" — 에 대한 실용적 해답이다. 편차가 U자형으로 레이어/위치 국소화된다는 관찰은 Activity B의 비연속 세그먼트 재사용에서 "어떤 레이어를 재계산해야 하는가"를 예측하는 레이어-선택적 갱신 정책 설계에 직접 기여한다. src/cache/segmented.py의 세그먼트 재사용 로직에 RelayCaching 스타일 레이어 범위 프로파일러를 추가하면 "세그먼트가 일치하지 않을 때도 레이어별로 재사용 가능 여부를 판별"하는 그레이디드 세그먼트 재사용이 가능하다.
- **ICaRus(2603.13281) — 크로스-모델 KV 아키텍처**: Activity B에서 "동일 세그먼트를 여러 모델이 공유"하는 시나리오를 구현할 때 ICaRus의 논리적 인코더-디코더 분리 원칙이 설계 기반이 된다. 특히 Activity B의 src/cache/radix.py에서 모델 ID를 레이어로 추가한 Radix 트리 인덱스를 구성하면 크로스-모델 세그먼트 공유를 지원하는 "멀티-모델 Radix 캐시"가 가능하다.
- **QKVShare(2605.03884) — 온-디바이스 양자화 KV 핸드오프**: Activity B의 에지 배포 시나리오에서 에이전트 간 비연속 KV 전달을 양자화와 결합하는 최초 실용 프레임워크다. CacheCard 자기-완결 표현이 Activity B의 비연속 세그먼트를 에이전트 간 교환 가능한 표준 형식으로 정의하는 "비연속 세그먼트 CacheCard 표준"으로 발전 가능하다.

### Activity C
- **LookaheadKV(2603.10899, ICLR 2026) — 미래-인식 드래프트-프리 퇴거**: Activity C 구현에서 기존 H2O·SnapKV·LaProx의 현재-인식 퇴거를 LookaheadKV의 미래-인식 퇴거로 업그레이드하는 것이 "정확도 ±1% 이내" 제약 달성 신뢰도를 가장 높이는 단기 우선순위다. LoRA 어댑터만 추가하면 기존 모델 변경 없이 배포 가능하므로 6단계 vLLM 이식에서 즉시 실험 가능하다. Samsung Research의 공개 GitHub(SamsungLabs/LookaheadKV)에서 코드를 직접 참조할 것. LaProx(레이어-단위 중요도) + LookaheadKV(미래 패턴 예측)의 앙상블이 Activity C의 최고 정확도 보존 퇴거 파이프라인 후보다.
