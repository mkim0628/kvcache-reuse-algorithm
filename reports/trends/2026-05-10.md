# 트렌드 리포트 — 2026-05-10

## 전체 요약

1. **Activity A (스케줄링)**: KV 캐시 복원(cache restoration)을 병렬화하는 새로운 시스템 차원의 접근이 등장했다. CacheFlow(2604.25080, UIUC/NUS)는 토큰·레이어·GPU 세 축으로 KV 복원을 3D 병렬화하여 TTFT를 10~62% 단축한다. 이론 측면에서는 Flow-Controlled Scheduling(2604.11001)이 KV 캐시 제약 스케줄러의 안정성 필요조건을 증명하는 첫 번째 이론 체계를 제시했으며, Valve(2604.07874)는 8,054 GPU 규모 프로덕션 클러스터에서 온-오프라인 추론 동시 운영으로 GPU 이용률을 34.6% 향상시켰다.

2. **Activity B (비연속 재사용)**: 재계산 없이(recomputation-free) 문서 KV를 맥락 독립적으로 재사용하는 두 가지 상이한 접근이 나란히 등장했다. KV Packet(2604.13226)은 trainable soft-token 어댑터로 캐시된 문서 KV를 래핑해 맥락 불연속을 해소하며 TTFT를 크게 낮춘다. Persistent Q4 KV Cache(2603.04428)는 에이전트별 KV를 4비트 양자화 후 디스크에 영속화해 에지 멀티에이전트 환경에서 15.7초의 재프리필을 제거한다.

3. **Activity C (압축)**: 벡터 양자화(VQ) 기반 압축이 스칼라 양자화를 능가하는 새로운 패러다임으로 확립됐다. VQKV(2603.16435)는 훈련 없이 VQ를 적용해 LLaMA3.1-8B에서 82.8% 압축률로 LongBench 성능의 98.6%를 유지한다. TTKV(2604.19769)는 시간적 거리를 기준으로 HBM/DRAM 계층에 KV를 배분하는 temporal tiering으로 128K 컨텍스트에서 크로스-계층 트래픽을 5.94× 감소시킨다. KVTC(ICLR 2026, NVIDIA)는 PCA+적응 양자화+엔트로피 코딩 파이프라인으로 20× 압축 후 TTFT를 최대 8× 단축한다.

4. **A+B 시너지**: CacheFlow의 3D 병렬 KV 복원이 Activity B의 비연속 세그먼트 다중 로드 상황(여러 세그먼트를 병렬로 복원)에 직접 적용된다. KV Packet의 어댑터-래핑이 CacheFlow의 I/O 병렬화와 결합되면 비연속 세그먼트들을 병렬로 로드·복원하는 완전한 파이프라인이 가능하다.

5. **B+C 시너지**: Persistent Q4 KV Cache와 VQKV가 모두 "저비트 압축 + 디스크/DRAM 영속화"라는 공통 방향을 가리킨다. Activity B의 비연속 세그먼트를 VQKV로 VQ 압축 후 영속화하면 에지-클라우드 양쪽에서 비연속 캐시 용량을 대폭 확장할 수 있다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [CacheFlow: Efficient LLM Serving with 3D-Parallel KV Cache Restoration](https://arxiv.org/abs/2604.25080)
- **출처**: arXiv 2604.25080 (2026-04-28, Sean Nian, Jiahao Fang 외 — UIUC / NUS)
- **핵심 아이디어**: 기존 KV 캐시 복원 방법들이 토큰·레이어·GPU 세 축 중 하나씩만 활용하고, 배치 단위 서빙에서 요청 간 자원 경쟁을 무시하는 문제를 해결한다. 통합 3D 병렬화 추상화(unified 3D parallelism abstraction)가 토큰-축(청크 단위 재계산 + I/O 중첩), 레이어-축(선행 레이어 I/O와 후행 레이어 연산 중첩), GPU-축(분산 멀티-GPU 병렬 복원)을 동시에 활용한다. 배치-인식 투-포인터 스케줄러(batch-aware two-pointer scheduler)가 재계산 비용 감소 한계편익이 가장 높은 요청을 우선 처리하며, 처리 후 남은 비용을 동적으로 갱신해 자원 할당을 적응적으로 조정한다. FlashAttention 선호 블록 크기(512 토큰)에 청크 크기를 정렬해 커널 효율을 극대화한다. vLLM + LMCache 위에 구현돼 기존 서빙 스택과 seamless 통합된다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 다양한 모델·워크로드·하드웨어에서 기존 최신 방법 대비 TTFT 10~62% 감소. Activity A의 "스케줄링 오버헤드 TTFT p50 +5% 이내" 목표를 KV 복원 단계에서 달성하는 새로운 레퍼런스. ContiguousKV(이미 보고)의 세분화 정렬이 토큰-축 청크 크기와 직접 연결되며, TTKV(이번 주 Activity C에 등재)의 계층 구조와 결합 시 계층별 복원 우선순위 설정이 가능하다. vLLM Q2 2026 로드맵의 "KV 캐시 매니저 재설계"와 동일 방향을 가리킨다.

#### 2. [Flow-Controlled Scheduling for LLM Inference with Provable Stability Guarantees](https://arxiv.org/abs/2604.11001)
- **출처**: arXiv 2604.11001 (2026-04, v1)
- **핵심 아이디어**: LLM 추론에서 디코드 길이가 사전에 알 수 없고 요청별 메모리 사용량이 생성 중 동적으로 증가하는 특성이 KV 캐시 오버플로우와 시스템 불안정을 야기한다는 문제를 이론적으로 형식화한다. 활성화 예산(activation budget)으로 각 타임스텝에서 활성 집합(active set)에 진입 가능한 요청 수를 제한하는 흐름 제어(flow control) 프레임워크를 제안한다. 이론적 분석이 세 가지 기여를 제공한다: (1) 안정 시스템의 필요조건(기대 워크로드 < KV 캐시 용량) 증명, (2) 이 조건을 충족하는 일정한 활성화 예산이 안정성을 달성함을 증명, (3) 무제한 레이턴시를 야기하는 과부하 조건을 정량화.
- **연구 목표와의 관련성**: high
- **주목할 점**: 활성화율 제어만으로 KV 캐시 사용량 궤적이 예측 가능해지고 시스템 안정성이 향상됨을 실험으로 확인. Online Scheduling(2502.07115, 이미 보고)이 평균 레이턴시 최적화를 다룬 반면, 이 논문은 시스템 안정성(stability)의 필요충분조건을 증명하는 보완적 이론 기여. Fluid-Guided WAIT(이미 보고)가 유체역학 근사로 실용 알고리즘을 제시했다면, Flow-Controlled Scheduling은 그 기저의 안정성 이론을 형식화한다. Activity A의 스케줄러 설계에서 "임계 부하 조건에서의 안정성 보장"이라는 새로운 설계 제약을 추가한다.

#### 3. [Valve: Production Online-Offline Inference Colocation with Jointly-Bounded Preemption Latency and Rate](https://arxiv.org/abs/2604.07874)
- **출처**: arXiv 2604.07874 (2026-04-09)
- **핵심 아이디어**: 프로덕션 LLM 추론이 트래픽 폭주(burst)로 인해 과도하게 프로비저닝되어 자원 낭비가 발생하는 문제를 온-오프라인 추론 동시 운영(colocation)으로 해결한다. 세 가지 GPU 런타임 기술을 결합한다: (1) 채널 제어 컴퓨트 격리(channel-controlled compute isolation)로 요청당 최대 1회의 서브-밀리초 선점, (2) 페이지 폴트 없는 메모리 회수(page-fault-free memory reclamation)로 KV 캐시 메모리를 오프라인 작업에 안전하게 재할당, (3) 동적 메모리 예약(dynamic memory reservation)으로 온라인 요청의 KV 증가 예산을 실시간 보장. 선점 레이턴시와 선점률을 공동으로 유계(jointly bounded)하는 최초 시스템.
- **연구 목표와의 관련성**: high
- **주목할 점**: 8,054 GPU 프로덕션 클러스터에 배포, 평균 GPU 이용률 34.6% 향상(2,170 GPU 절감 상당), TTFT 증가 <5%, TPOT 증가 <2%. Activity A의 "스케줄링 오버헤드 TTFT p50 +5% 이내" 목표를 프로덕션 규모에서 실증한 유일한 논문. TokenFlow(이미 보고)가 요청 폭주 시 선점으로 TTFT를 80.2% 감소시킨 반면, Valve는 온-오프라인 동시 운영이라는 추가 차원에서 선점의 레이턴시·율(rate) 상한을 동시에 보장한다. KV 메모리 회수와 예약의 공동 관리가 Activity A의 "캐시 퇴거율 최소화"와 직접 연결된다.

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [KV Packet: Recomputation-Free Context-Independent KV Caching for LLMs](https://arxiv.org/abs/2604.13226)
- **출처**: arXiv 2604.13226 (2026-04-14; Hugging Face Papers 수록; GitHub 오픈소스 공개)
- **핵심 아이디어**: 표준 KV 캐시의 핵심 제약인 "맥락 의존성(context-dependence)"을 근본적으로 제거한다. 캐시된 문서 KV를 재계산 없이 새 맥락에서 직접 재사용할 수 없는 이유는 어텐션 분포가 맥락에 따라 달라지기 때문이다. KV Packet은 각 문서 KV 블록을 경량 훈련 가능 소프트-토큰 어댑터(trainable soft-token adapter)로 래핑해 맥락 불연속을 해소하며, 자기지도 증류(self-supervised distillation)로 어댑터를 훈련한다. 추론 시에는 검색된 문서 KV Packet들을 단순 연결(concatenation)로 조합해 추가 재계산 없이 서빙한다. 모델 파라미터 수정이 불필요하며 훈련된 어댑터는 범용(universal)으로 다양한 맥락에서 재사용 가능하다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Llama-3.1 및 Qwen2.5에서 재계산 기반 기준선과 동등한 F1 점수를 유지하면서 TTFT를 재계산 대비 대폭 단축, 연산량(FLOPs) 거의 제로. Activity B의 "비연속 세그먼트를 위치 독립적으로 재사용"하는 핵심 목표를 "소프트-토큰 어댑터로 맥락 의존성 자체를 제거"하는 방식으로 달성하는 최초의 접근. 기존 CacheBlend·KVLink·ProphetKV가 선택적 재계산으로 맥락 의존성을 보정한 반면, KV Packet은 재계산 없이 맥락 독립적 캐시를 구현하는 질적으로 다른 방향이다. 어댑터 래핑이 세그먼트 저장 크기를 거의 증가시키지 않으므로 Activity B의 비연속 세그먼트 해시 캐시에 즉시 통합 가능하다.

#### 2. [Agent Memory Below the Prompt: Persistent Q4 KV Cache for Multi-Agent LLM Inference on Edge Devices](https://arxiv.org/abs/2603.04428)
- **출처**: arXiv 2603.04428 (2026-02-17; GitHub: yshk-mxim/agent-memory)
- **핵심 아이디어**: 에지 기기(Apple M4 Pro, 10.2 GB 캐시 예산)에서 멀티에이전트 워크플로우가 에이전트 수 증가에 따라 KV 캐시를 상시 퇴거·재로드해야 하는 문제를 에이전트별 KV 캐시의 디스크 영속화로 해결한다. 핵심 설계: (1) 각 에이전트의 KV 캐시를 4비트 양자화(Q4)로 압축 후 safetensors 포맷으로 디스크에 저장하는 영속 블록 풀(persistent block pool), (2) 퇴거 후 재진입 시 O(n) 풀 재프리필(15.7초) 대신 압축 KV를 어텐션 레이어에 직접 주입해 복원, (3) 배치 Q4 추론을 지원하는 BatchQuantizedKVCache, (4) 인터리브드 프리필+디코드 스케줄러. 서버 재시작·기기 재부팅 후에도 에이전트 상태가 복원된다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 4K 컨텍스트 기준 에이전트당 재프리필 지연 15.7초를 캐시 직접 주입으로 대폭 단축. Activity B의 "비연속 세그먼트 재사용"을 에지-에이전트 영속 저장 관점에서 구현하는 최초 논문. KEEP(이미 보고)의 정적-동적 메모리 분리가 "언제 재계산할 것인가"를 결정한다면, Persistent Q4 KV Cache는 "재계산 없이 캐시를 어디에 저장하고 어떻게 복원할 것인가"를 결정한다. Dual-Blade(이미 보고)의 NVMe-Direct 오프로딩과 설계 방향을 공유하면서 멀티에이전트 관점을 추가한다. Activity B의 장기 기억(long-term memory) 지원 목표와 직접 연결된다.

|------|---------|------------|
| Segmented Hash / Radix | LycheeCluster 삼각부등식 기반 계층 인덱스로 세그먼트 검색 O(N)→로그 시간 | 변화 없음 — 이번 주 신규 세그먼트 해시/Radix 논문 없음; 비연속 세그먼트 저장 방향으로 무게중심 이동 |
| CacheBlend / Blending | SemantiCache 의미 청킹 + 클러스터 병합으로 TPOT 2.61× 가속 | KV Packet(2604.13226)이 소프트-토큰 어댑터로 맥락 의존성 자체를 제거하는 질적으로 새로운 방향 추가; 기존 CacheBlend 계열의 "선택적 재계산"에서 "재계산 제로"로의 패러다임 전환 |
| Position-independent caching | InfoFlowKV 청크 재순서화 전략 형식화 | Persistent Q4 KV Cache(2603.04428)가 에이전트 KV를 4비트 압축 후 영속 저장·직접 복원하는 에지 PIC 구현 경로 추가; 서버 재시작 후에도 위치 독립 재사용 가능한 최초 에지 시스템 |

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [VQKV: High-Fidelity and High-Ratio Cache Compression via Vector-Quantization](https://arxiv.org/abs/2603.16435)
- **출처**: arXiv 2603.16435 (2026-03-17; OpenReview 게재)
- **핵심 아이디어**: 기존 스칼라 양자화와 저랭크 근사가 높은 압축률과 높은 재구성 충실도를 동시에 달성하지 못하는 한계를 벡터 양자화(VQ)로 극복한다. Residual Simple Vector Quantization(RSimVQ)을 통해 KV 벡터를 다중 코드북에 매핑하고 정수 인덱스만 저장한다. RoPE의 이종 주파수 특성으로 Key 캐시의 RoPE-후 표현 구조가 이질적으로 변하는 문제를 pre-RoPE 공간에서 코드북을 학습한 후 적용해 해결한다. 디코딩 시에는 온-디맨드 재구성으로 슬라이딩 윈도우 내 최신 토큰만 로컬 캐시로 유지한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: LLaMA3.1-8B에서 82.8% 압축률(약 5.8× 메모리 감소)로 LongBench 성능의 98.6% 유지, 동일 메모리 예산에서 4.3× 긴 컨텍스트 지원. Activity C의 "메모리 −30% 이상" 목표를 −82.8%로 대폭 초과하면서 "정확도 ±1% 이내" 조건을 충족(98.6% 성능 유지). 훈련 불필요(training-free)하여 즉시 배포 가능. ManifoldKV(이미 보고)의 스케일-인식 퇴거와 달리 KV 벡터 자체의 표현 공간을 압축해 메모리를 줄인다. eOptShrinkQ(이미 보고)의 자동 저랭크 성분 추출과 결합 시 "공유 저랭크 성분은 스칼라 양자화, 잔차는 VQ 압축"하는 하이브리드 파이프라인이 가능하다.

#### 2. [TTKV: Temporal-Tiered KV Cache for Long-Context LLM Inference](https://arxiv.org/abs/2604.19769)
- **출처**: arXiv 2604.19769 (2026-03-27, Gradwell Dzikanyanga, Weihao Yang 외 — Harbin Institute of Technology / Guangzhou University)
- **핵심 아이디어**: KV 캐시 토큰 중요도가 시간적 거리에 따라 변화한다는 관찰에 기반한다. 최근 상태는 단기 기억(short-term memory)으로 생성에 더 중요하고, 오래된 상태는 장기 기억(long-term memory)으로 소수만 현재 쿼리와 관련된다. 이 비대칭성을 계층적 메모리 구조로 실현한다: (1) Tier Layout — HBM(고속)과 DRAM(저속)의 이중 계층, (2) Tier Content — 시간적 근접성에 따라 최신 KV를 HBM 고정밀도 계층에, 오래된 KV를 DRAM 저정밀도 계층에 배치, (3) Tier Interaction — 블록-단위 스트리밍 어텐션(block-wise streaming attention)으로 저속 계층 접근 시 통신과 연산을 오버랩. 압축과 오프로딩을 계층 구조로 통합하는 것이 핵심.
- **연구 목표와의 관련성**: high
- **주목할 점**: 128K 컨텍스트 태스크에서 크로스-계층 트래픽 5.94× 감소, 레이턴시 76% 단축, 처리량 2× 향상. Activity C의 "동일 메모리 예산에서 유효 컨텍스트 2× 이상" 목표를 메모리 계층화와 통신 오버랩으로 달성하는 새로운 접근. TTKV의 시간 기반 계층화가 CacheFlow(이번 주 Activity A)의 배치-인식 복원 우선순위와 시너지를 낸다 — 최근 HBM 계층 KV는 복원 우선순위를 높이고, 오래된 DRAM 계층 KV는 배치 레벨에서 지연 복원 허용. Activity A의 ContiguousKV(이미 보고)의 청크 I/O 정렬 원칙이 TTKV의 블록-단위 스트리밍 어텐션 청크 크기 선택에 직접 적용된다.

#### 3. [KV Cache Transform Coding for Compact Storage in LLM Inference](https://arxiv.org/abs/2511.01815)
- **출처**: arXiv 2511.01815 / ICLR 2026 (NVIDIA; GitHub: OnlyTerp/kvtc)
- **핵심 아이디어**: KV 캐시를 멀티스텝 변환 코딩(transform coding) 파이프라인으로 압축하는 최초의 ICLR 채택 방법이다. 세 단계로 구성된다: (1) PCA 기반 특성 비상관화(decorrelation) — 교정 데이터셋에서 단 1회 PCA 기저 행렬 V를 계산하고 모든 미래 캐시에 재사용, (2) 적응 양자화(adaptive quantization) — 동적 프로그래밍(DP)으로 잔여 재구성 오류를 최소화하는 최적 비트 할당 계산 (DP가 후반 주성분에 종종 0비트 할당해 조기 차원 감소 실현), (3) 엔트로피 코딩(entropy coding) — DEFLATE 알고리즘으로 심볼 패킹. 다른 방법들과 달리 프롬프트별 별도 분해를 계산하지 않아 추론 오버헤드가 매우 낮다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 20× 압축 시 AIME25, GSM8K, LiveCodeBench, LongBench, MATH-500, MMLU 등에서 기준 모델 대비 점수 1점 이내 정확도 유지. 특정 유스케이스에서 40×+ 압축 달성. 장문 컨텍스트 KV 오프로딩 대비 TTFT 최대 8× 단축. Llama3, Mistral NeMo, R1-Qwen2.5에서 검증. Activity C의 "정확도 ±1% 이내" 제약과 "메모리 −30% 이상" 목표를 20× 압축(−95%)에서 달성하는 가장 공격적인 정확도-보존 압축. KVTC는 eOptShrinkQ(이미 보고)의 랜덤 행렬 이론 기반 자동 저랭크 추출과 상호 보완적이다 — eOptShrinkQ가 공유 성분을 자동 추출한다면, KVTC는 나머지를 추가 엔트로피 코딩으로 더 압축한다. ICLR 2026 채택으로 최고 수준 학계 인정. 오픈소스 구현(kvtc) 제공.

## Activity 간 시너지 포인트

**A + B**
- CacheFlow의 3D 병렬 KV 복원을 Activity B의 비연속 세그먼트 로드에 적용하면, 다수의 비연속 세그먼트를 병렬로 복원하는 "비연속 세그먼트 3D 병렬 로드" 아키텍처가 가능하다. 토큰-축 병렬화는 긴 세그먼트 내 청크를 병렬 재계산하고, 레이어-축 병렬화는 서로 다른 세그먼트의 레이어를 동시에 복원하며, GPU-축 병렬화는 세그먼트를 여러 GPU에 분산 복원한다.
- KV Packet의 어댑터-래핑 비연속 세그먼트와 CacheFlow의 배치-인식 복원 스케줄러를 결합하면, RAG 요청에서 검색된 K개 문서 KV Packet을 CacheFlow가 한계편익 기반으로 복원 순서를 최적화하는 통합 파이프라인이 가능하다. 재계산이 없으므로 CacheFlow의 재계산-I/O 중첩 대신 어댑터 주입 병렬화에 집중할 수 있다.
- Valve의 페이지 폴트 없는 메모리 회수가 Activity B의 세그먼트 캐시에서 저사용 세그먼트를 오프라인 배치 작업 중 안전하게 회수하고, 온라인 요청 폭주 시 즉시 복원하는 "세그먼트 캐시 탄성 확장" 정책으로 활용 가능하다.

**B + C**
- KV Packet의 소프트-토큰 어댑터가 맥락 의존성을 해소한 후, VQKV의 VQ 압축을 각 KV Packet에 적용하면 "맥락 독립 + 고압축" 비연속 세그먼트 저장이 실현된다. 어댑터 크기는 미미하므로 패킷 전체를 82.8% 압축해도 어댑터 정보를 손실 없이 보존할 수 있다.
- Persistent Q4 KV Cache의 에이전트별 영속화 구조에 TTKV의 시간 기반 계층화를 결합하면, 최근 에이전트 KV는 HBM 고정밀도로, 오래된 에이전트 KV는 DRAM Q4로, 가장 오래된 에이전트 KV는 디스크 Q4로 자동 계층화하는 3-티어 에이전트 KV 영속화 시스템이 가능하다. 에지 기기에서 에이전트 메모리의 효율적 장기 관리 경로를 제공한다.
- KVTC의 PCA 기저가 교정 데이터셋에서 1회만 계산된다는 특성이 Activity B의 세그먼트 캐시에 적용될 때 장점을 갖는다 — 세그먼트 유형(시스템 프롬프트, 사용자 쿼리, 도구 출력)별로 한 번 PCA 기저를 계산해 두면, 같은 유형의 새 세그먼트를 동일 기저로 즉시 압축 가능하다.

**A + C**
- CacheFlow의 배치-인식 복원 스케줄러가 TTKV의 계층 정보를 활용하면, HBM 계층 KV는 복원 비용이 낮아 한계편익이 높고 DRAM 계층 KV는 복원 비용이 높아 한계편익이 낮으므로, 복원 우선순위를 계층에 연동하는 "계층-인식 CacheFlow" 스케줄러가 가능하다.
- Valve의 KV 메모리 예약(dynamic memory reservation)에 VQKV의 VQ 압축을 통합하면, 오프라인 작업에 메모리를 해제할 때 선점된 KV를 즉시 VQKV로 압축해 5.8× 작은 크기로 저장하고, 온라인 요청 시 압축 해제 후 주입하는 "선점-VQ 압축 파이프라인"이 가능하다. 선점 오버헤드를 압축 크기만큼 추가로 줄인다.
- KVTC의 20× 압축을 PrfaaS(이미 보고)의 크로스-데이터센터 KV 전송에 적용하면, 이더넷으로 전송되는 KV 크기가 기존 MLA 압축 대비 추가로 20×(~10비트) 더 감소해 저대역폭 WAN 환경에서의 크로스-데이터센터 P/D 분리의 실용성이 획기적으로 향상된다.

**A + B + C**
- 통합 파이프라인 (RAG 서빙 특화): Valve 온-오프라인 동시 운영(A) → KV Packet 어댑터 래핑 비연속 세그먼트(B) → VQKV VQ 압축 + KVTC 엔트로피 코딩(C) → CacheFlow 3D 병렬 복원(A). 온라인 RAG 요청이 도착하면 Valve가 KV 메모리를 동적 예약하고, KV Packet이 맥락 독립 세그먼트 K개를 즉시 조합하며, VQKV+KVTC로 압축된 세그먼트를 CacheFlow가 3D 병렬로 복원해 TTFT를 최소화한다.
- TTKV의 HBM/DRAM 계층 구조가 이 파이프라인에서 KV Packet의 "자주 재사용되는 세그먼트 → HBM 고정밀, 드물게 재사용 → DRAM VQKV 압축"하는 접근 빈도 기반 계층 배치 정책을 실현하는 메타-스케줄러로 작동할 수 있다.

## idea-generator에게 전달할 포인트

### Activity A
- **CacheFlow의 3D 병렬화 원칙**: 토큰·레이어·GPU 세 축으로 KV 복원을 병렬화하는 "meet-in-the-middle" 투-포인터 전략이 Activity A의 P/D 분리 복원 병목 해소의 직접적 구현 경로. 현재 ContiguousKV(청크 I/O 정렬) + CacheFlow(3D 병렬화) 조합이 "가장 실용적인 KV 복원 가속 스택"으로 확립됐다. 두 논문 모두 vLLM 기반 구현이므로 6단계 vLLM 이식에 즉시 활용 가능.
- **Valve의 프로덕션 설계 원칙**: 선점 레이턴시 <1ms + 선점 횟수 요청당 최대 1회 + KV 메모리 페이지 폴트 없는 회수 세 가지 보장이 Activity A의 "스케줄링 오버헤드 TTFT p50 +5% 이내" 제약을 프로덕션에서 만족하는 충분조건임을 8K GPU 실증 데이터로 뒷받침한다. 단일 노드 Activity A 구현에서도 이 세 가지 설계 원칙을 채택할 것.
- **Flow-Controlled Scheduling의 안정성 이론**: 활성화 예산(activation budget) 파라미터가 Activity A 스케줄러의 핵심 하이퍼파라미터가 됨을 이론으로 증명. Fluid-Guided WAIT의 임계값 알고리즘과 Flow-Controlled Scheduling의 안정성 조건을 통합하면 "임계값이 안정성 조건을 항상 만족하도록 동적 조정"하는 이론-보장 WAIT 변형이 가능하다.

### Activity B
- **KV Packet의 패러다임 전환**: "선택적 재계산으로 맥락 의존성 보정"에서 "소프트-토큰 어댑터로 맥락 의존성 자체 제거"로의 전환이 Activity B의 비연속 세그먼트 재사용 정확도 천장(accuracy ceiling)을 근본적으로 높인다. Activity B의 segmented.py 구현에서 각 세그먼트를 KV Packet처럼 "어댑터-래핑 캐시 유닛"으로 취급하는 설계 변경을 탐색할 것. 어댑터 훈련에 필요한 데이터량이 소규모(자기지도 증류)라 Activity B의 연구 사이클 내 구현이 현실적이다.
- **Persistent Q4 KV Cache의 에지 영속화**: Activity B의 "장기 기억(long-term memory) 지원" 목표에서 에이전트 KV를 디스크에 영속화하는 것이 실용적으로 검증됐다. 현재 src/cache/segmented.py에 영속화 레이어를 추가하면 서버 재시작 후에도 세그먼트 캐시가 유지되는 내구성(durability) 기능이 추가된다. 4비트 양자화가 디스크 저장과 직접 재주입 양쪽에서 유리하다는 점은 Activity C 통합의 자연스러운 진입점.

### Activity C
