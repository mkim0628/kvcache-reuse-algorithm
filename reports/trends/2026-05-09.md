# 트렌드 리포트 — 2026-05-09

## 전체 요약

1. **Activity A (스케줄링)**: P/D 분리 아키텍처의 두 가지 새로운 방향이 동시에 등장했다. PPD Disaggregation(2603.13358)은 멀티턴 서빙에서 디코드 노드가 직접 append-prefill을 처리해 KV 전송 병목을 제거하고 Turn 2+ TTFT를 68% 단축한다. Prefill-as-a-Service(PrfaaS, 2604.15039)는 데이터센터 경계를 초월해 장문 프리필만 전용 클러스터에 위탁하고 결과 KV를 일반 이더넷으로 전송하는 크로스-데이터센터 P/D 확장 설계로 처리량 54% 향상을 달성했다. 에지 측에서는 Dual-Blade(2604.26557)가 NVMe-Direct 이중 경로 오프로딩으로 단말 GPU에서 KV 메모리 예산을 동적으로 확장한다.

2. **Activity B (비연속 재사용)**: 청크-레벨 캐시 재사용의 "무엇이 효과적인가"를 실증적으로 정리하는 연구가 등장했다. arXiv 2603.20218은 CLC 시스템들의 교차-청크 어텐션 의존성 문제를 체계적으로 실험하며 기존 방법들의 보완 가능성을 형식화했다. LycheeCluster(2603.08453)는 삼각부등식 기반 계층적 KV 인덱스로 청크 검색을 O(N) → 로그 시간으로 단축하며 3.6× 속도향상을 달성한다. SemantiCache(2603.14303)는 의미 청킹과 클러스터-병합으로 TPOT 2.61× 가속을 달성한다.

3. **Activity C (압축)**: 훈련 기반 레이어 간 KV 공유가 새로운 압축 방향으로 부상했다. Stochastic KV Routing(2604.22782, Apple ML Research)은 학습 중 계층별 KV를 확률적으로 공유해 배포 시 다양한 depth-wise 공유 전략에 적응 가능한 모델을 만든다. SpecKV(2605.02888)는 압축 수준에 따른 최적 추측 길이(γ)를 draft 신호로 자동 결정해 고정 γ=4 대비 56.0% 가속을 달성한다. 에지 특화로는 Dual-Blade가 오프로딩 지연을 33.1~42.4% 단축한다.

4. **A+B 시너지**: PPD의 decode-node 로컬 append-prefill이 Activity B의 "캐시된 KV 세그먼트를 현 노드에서 직접 재사용"하는 비연속 재사용 최적 경로와 정확히 일치한다. LycheeCluster의 로그 시간 세그먼트 검색이 PPD의 실시간 라우팅 결정에서 append 대 full prefill 선택을 지원할 수 있다.

5. **B+C 시너지**: Stochastic KV Routing의 레이어 간 KV 공유를 SemantiCache의 의미 청크 경계와 결합하면 의미 경계를 기준으로 레이어 공유 범위를 제한하는 "의미-인식 depth-wise 공유" 구조가 가능하다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [Not All Prefills Are Equal: PPD Disaggregation for Multi-turn LLM Serving](https://arxiv.org/abs/2603.13358)
- **출처**: arXiv 2603.13358 (2026-03-09, Zongze Li, Jingyu Liu 외 — 제출 후 주목 누적, 2026-05-05 전후 주요 서빙 커뮤니티 인용 급증)
- **핵심 아이디어**: 전통 P/D 분리의 두 가지 비효율을 해결한다. (1) 멀티턴에서 매 턴마다 이전 응답 KV를 P 노드로 재전송 후 재프리필, (2) P→D KV 전송이 대역폭을 포화시켜 TTFT가 악화된다. PPD(Prefill Prefill-capable Decode) 분리는 Turn 2+ 요청을 D 노드에서 직접 처리하는 동적 라우팅 시스템이다. append-prefill(새 입력 토큰만 처리, 캐시된 KV 재사용)이 full prefill보다 디코딩 속도저하를 한 자릿수 줄인다는 관찰에 기반한다. SLO 제약에 따라 가중치를 조정하는 구성 가능한 라우터를 포함한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Turn 2+ TTFT 약 68% 감소, TPOT 경쟁력 유지, KV 전송 혼잡 대폭 완화. Activity A의 "캐시 인식 스케줄링"을 P/D 분리 아키텍처에서 "어떤 노드에서 어떤 종류의 프리필을 처리할 것인가"로 확장하는 최신 실용 구현. 기존 ContiguousKV의 Re-Prefill 가속(이미 보고)이 PPD의 append-prefill에 직접 적용 가능하다. vLLM GitHub Issue #32733에서 비방향성 KV 전송 RFC가 PPD와 동일한 문제를 논의 중이다.

#### 2. [Prefill-as-a-Service: KVCache of Next-Generation Models Could Go Cross-Datacenter](https://arxiv.org/abs/2604.15039)
- **출처**: arXiv 2604.15039 (2026-04-16, Moonshot AI + Tsinghua; Hacker News 주목)
- **핵심 아이디어**: 기존 P/D 분리가 낮은 레이턴시 RDMA 패브릭을 가정해 단일 데이터센터 내에 묶여 있던 제약을 깬다. 하이브리드 어텐션 모델(MLA 등)로 KV 크기를 먼저 줄이고, 이를 기반으로 장문 프리필만 독립적인 연산-집약형 프리필 클러스터에 위탁한다. 결과 KV를 일반 이더넷으로 디코드 클러스터에 전송하는 크로스-데이터센터 P/D 아키텍처를 실현한다. 길이 기반 임계값 라우팅(짧은 요청은 로컬 PD 유지), 대역폭 인식 스케줄러(링크 상태에 반응), 하이브리드 프리픽스 캐시 풀의 세 가지 시스템-측 기여를 결합한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 1T 파라미터 내부 하이브리드 모델 기준 동종 PD 대비 처리량 54%, P90 TTFT 64% 개선. Activity A의 "네트워크-인식 KV 라우팅" 방향에서 GORGO(이미 보고)가 지리적 멀티-리전 라우팅을 다룬 반면, PrfaaS는 이더넷 환경에서 크로스-데이터센터 KV 이동의 실현 가능성을 증명한다. MLA/하이브리드 어텐션 모델이 KV 크기를 충분히 줄여야만 실용적임을 제약 조건으로 명시한다.

#### 3. [DUAL-BLADE: Dual-Path NVMe-Direct KV-Cache Offloading for Edge LLM Inference](https://arxiv.org/abs/2604.26557)
- **출처**: arXiv 2604.26557 (2026-04-29, Bodon Jeong, Hongsu Byun, Youngjae Kim 외)
- **핵심 아이디어**: 에지 기기에서 KV 캐시가 가용 메모리를 초과할 때 기존 파일 기반 오프로딩이 커널 페이지 캐시 스래싱·예측 불가 지연·높은 소프트웨어 오버헤드를 유발하는 문제를 이중 경로 KV 잔류 프레임워크로 해결한다. 런타임 메모리 가용성에 따라 KV 텐서를 (1) 페이지-캐시 경로 또는 (2) NVMe-Direct 경로 중 동적으로 할당한다. NVMe-Direct 경로는 파일시스템을 우회하고 KV 텐서를 연속 LBA 영역에 매핑해 저오버헤드 직접 스토리지 접근을 구현한다. 적응형 파이프라인 병렬성이 스토리지 I/O와 GPU DMA를 오버랩한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 프리필 지연 최대 33.1%, 디코드 지연 최대 42.4% 감소, SSD 이용률 2.2× 향상. Activity A의 "캐시 퇴거율 최소화"와 "스케줄링 오버헤드 TTFT +5% 이내" 목표를 에지 환경에서 실현하는 시스템. ScoutAttention(이미 보고)의 레이어-선행 CPU 선계산 원리를 NVMe 오프로딩으로 확장한 방향. 단말 기기에서 "KV 오프로딩 = 에지 캐시 분리"로 Activity A의 적용 범위를 확장한다.

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [LycheeCluster: Efficient Long-Context Inference with Structure-Aware Chunking and Hierarchical KV Indexing](https://arxiv.org/abs/2603.08453)
- **출처**: arXiv 2603.08453 (2026-03-09, Dongfang Li 외 — Harbin Institute of Technology / UESTC)
- **핵심 아이디어**: 기존 검색 기반 KV 캐시 관리 방법이 고정 크기 청킹으로 의미 완결성을 훼손하고 선형 스캔으로 효율이 낮다는 문제를 해결한다. 경계-인식 청킹(boundary-aware chunking)으로 로컬 의미 응집력을 유지하고, 삼각부등식 기반의 재귀 계층 인덱스를 구성해 캐시 검색을 선형 스캔에서 이론적으로 한정된 로그 시간 가지치기 탐색으로 전환한다. 삼각부등식을 활용하면 현재 쿼리와 유사하지 않은 서브트리 전체를 한 번의 비교로 제거할 수 있다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Quest, ClusterKV 대비 최대 3.6× 엔드-투-엔드 추론 속도향상, 모델 성능 저하 무시. Activity B의 "비연속 세그먼트 검색 효율" 문제를 O(N) 선형에서 O(log N) 로그 시간으로 근본적으로 해소한다. DASH-KV(이미 보고)의 비대칭 딥 해싱이 O(N)을 달성한 반면, LycheeCluster는 삼각부등식 기반 가지치기로 보다 이론적으로 강건한 경계를 제공한다. Activity B의 Radix 트리 캐시에서 삼각부등식 기반 서브트리 가지치기를 추가하면 세그먼트 검색 시간을 대폭 단축할 수 있다.

#### 2. [An experimental study of KV cache reuse strategies in chunk-level caching systems](https://arxiv.org/abs/2603.20218)
- **출처**: arXiv 2603.20218 (2026-03-03)
- **핵심 아이디어**: RAG 청크-레벨 캐싱(CLC)이 교차-청크 어텐션 의존성을 무시해 출력 품질이 저하되는 근본 원인을 체계적으로 실험한다. 기존 CLC 접근들의 한계를 (1) 정확도 제약 또는 (2) 적용 범위 제약으로 분류하고, 기존 방법들이 상호 보완적이라는 관찰을 형식화한다. CLC 시스템 실험 평가를 통해 교차-청크 의존성 해소 전략의 비교 기준을 확립한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Activity B의 "비연속 세그먼트를 조합할 때 교차-세그먼트 의존성을 어떻게 처리할 것인가"라는 핵심 문제를 실험적으로 분석한 최초의 종합 연구. ProphetKV·QCFuse·InfoFlowKV(이미 보고)가 개별 해법을 제시한 반면, 이 논문은 CLC 방법론 전체를 비교 분석해 어떤 조합이 효과적인지를 실험 기반으로 도출한다. Activity B의 세그먼트 해시 캐시 구현에서 교차-세그먼트 의존성 처리 전략 설계에 직접 활용 가능한 실증 데이터를 제공한다.

#### 3. [SemantiCache: Efficient KV Cache Compression via Semantic Chunking and Clustered Merging](https://arxiv.org/abs/2603.14303)
- **출처**: arXiv 2603.14303 (2026-03-17)
- **핵심 아이디어**: 기존 KV 캐시 압축 방법이 이산 토큰 또는 비의미적 청크 단위로 작동해 의미 단편화가 발생하는 문제를 해결한다. 자연적 의미 경계(구분자)로 캐시를 의미 응집 청크로 분할하고, 효율적인 Greedy Seed-Based Clustering(GSC) 알고리즘으로 각 청크 내 토큰을 클러스터링한 후 시맨틱 코어로 병합한다. 병합된 토큰의 감소된 어텐션 기여를 재조정하는 비례 어텐션(Proportional Attention) 메커니즘을 도입한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 다양한 벤치마크와 모델에서 TPOT 최대 2.61× 가속 및 메모리 대폭 감소, 원본 모델과 동등한 성능 유지. Activity B의 세그먼트 분할에서 "의미 경계를 기준으로 청크를 구성"하는 SemantiCache의 원칙이 세그먼트 해시 경계 설정에 직접 활용 가능하다. LycheeCluster(경계-인식 청킹 + 계층 인덱스)와 SemantiCache(의미 청킹 + 클러스터 병합)가 상호 보완적 설계를 제공한다 — LycheeCluster가 효율적 검색을, SemantiCache가 의미 보존 청크 설계를 담당하는 조합.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [Stochastic KV Routing: Enabling Adaptive Depth-Wise Cache Sharing](https://arxiv.org/abs/2604.22782)
- **출처**: arXiv 2604.22782 (2026-04-03; Apple Machine Learning Research; Hugging Face Daily Papers 수록)
- **핵심 아이디어**: KV 캐시 최적화가 주로 시간 축(어느 토큰을 유지할 것인가)에 집중한 반면, 이 논문은 깊이 축(어느 레이어의 KV를 공유할 것인가)에서 새로운 압축 방향을 제시한다. 학습 중 각 레이어가 확률적으로 자신의 KV 상태 또는 선행 레이어의 KV 상태를 선택하는 무작위 교차-레이어 어텐션(random cross-layer attention)을 적용한다. 이 확률적 학습이 모델을 다양한 depth-wise 공유 전략에 강건하게 만들어 배포 시 하드웨어 제약에 따라 공유 전략을 유연하게 선택할 수 있게 한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 대규모 모델의 데이터-제약 환경에서 성능을 유지하거나 향상하면서 KV 캐시 메모리 풋프린트를 크게 감소시킨다. Apple ML Research의 연구로 에지 기기 배포 맥락에서 특히 의미 있다. Activity C의 압축이 사후(post-hoc) 방식이 아닌 학습 중 내재화된다는 새로운 접근. KV-CoRE(이미 보고)의 Key-Value 비대칭 압축 가능성과 결합하면 "Key 레이어는 더 적극적으로 공유, Value 레이어는 선택적으로 공유"하는 비대칭 depth-wise 라우팅이 가능하다.

#### 2. [SpecKV: Adaptive Speculative Decoding with Compression-Aware Gamma Selection](https://arxiv.org/abs/2605.02888)
- **출처**: arXiv 2605.02888 (2026-05-05, 최신; GitHub: 오픈소스 공개)
- **핵심 아이디어**: 추측적 디코딩에서 추측 길이(γ)를 고정(보통 γ=4)하는 것이 태스크 유형과 압축 수준에 따라 최적이 아님을 실증한다. draft 모델 자체의 신호(min draft confidence, max draft entropy 등)를 활용해 압축 수준별 최적 γ를 경량 MLP로 자동 선택하는 적응형 컨트롤러를 제안한다. 프로파일링 대상: 4개 태스크 카테고리 × 4개 추측 길이 × 3개 압축 수준(FP16, INT8, NF4)의 5,112개 스텝-레벨 레코드.
- **연구 목표와의 관련성**: high
- **주목할 점**: 고정 γ=4 대비 56.0% 개선, 추가 계산 오버헤드 0.34ms. Activity C의 KV 압축과 추측적 디코딩이 독립 최적화 차원이 아닌 결합 최적화 차원임을 형식화한 첫 번째 논문. 압축 수준이 높아질수록 γ를 줄여야 정확도 저하 없이 가속을 유지할 수 있다는 실증이 Activity C의 "정확도 ±1% 이내" 제약 하에서 추측적 디코딩 통합 시 필수 고려 사항을 제공한다.

#### 3. [KV Cache Offloading for Context-Intensive Tasks](https://arxiv.org/abs/2604.08426)
- **출처**: arXiv 2604.08426 (2026-04-09, Andrey Bocharnikov 외 — HSE/Yandex)
- **핵심 아이디어**: 기존 KV 오프로딩 평가가 컨텍스트에서 적은 정보만 추출하는 태스크에 집중해 "컨텍스트-집약적(context-intensive)" 태스크에서의 성능 저하를 간과했음을 지적한다. 대량 구조화 지식 추출이 필요한 Text2JSON 벤치마크를 새로 구성하고, 최신 KV 오프로딩 방법들이 Llama 3와 Qwen 3 모델에서 컨텍스트-집약적 태스크에서 유의미한 성능 저하를 겪음을 체계적으로 문서화한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Activity C의 "정확도 보존(±1% 이내)" 제약을 컨텍스트-집약적 태스크에서 검증하는 최초의 체계적 벤치마크. ScoutAttention·TailorKV(이미 보고) 등 기존 오프로딩 방법들이 이 벤치마크에서 얼마나 정확도를 유지하는지를 평가하는 기준 데이터셋 역할. Activity C 구현 평가 시 일반 벤치마크(LongBench, RULER 등)뿐만 아니라 Text2JSON처럼 밀도 높은 구조 추출이 필요한 태스크를 반드시 포함해야 함을 강조한다.

#### 4. [Is KV Cache Quantization Sabotaging Your Context?](https://dasroot.net/posts/2026/05/kv-cache-quantization-agentic-coding-long-horizon/)
- **출처**: dasroot.net 기술 블로그 (2026-05, 에이전틱 코딩 장기 작업 실험 포스트)
- **핵심 아이디어**: TurboQuant(이미 보고)가 KV 메모리를 최대 6× 감소시키지만, 50회 이상의 반복 작업이 필요한 장기 에이전틱 코딩 태스크에서 공격적 저비트 양자화가 성능 저하를 유발하는 구체적 사례를 실험한다. KV 캐시 양자화가 컨텍스트 유지가 필수적인 작업에서 누적 오류를 발생시킴을 독립적으로 검증한다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: KV Cache Offloading for Context-Intensive Tasks(2604.08426)의 시스템-측 실험과 이 블로그의 적용 측 실험이 같은 문제(장기/집약적 컨텍스트에서의 KV 압축 정확도 저하)를 서로 다른 관점에서 확인한다. Activity C의 "정확도 ±1% 이내" 제약이 에이전틱 장기 작업에서 더욱 엄격해짐을 실증. 에이전틱 워크로드를 타겟으로 하는 Activity A+C 통합 설계 시 이 누적 오류 문제를 핵심 설계 제약으로 반영해야 한다.

## Activity 간 시너지 포인트

**A + B**
- PPD의 D 노드 로컬 append-prefill이 Activity B의 비연속 세그먼트 재사용과 완전히 맞닿는다. D 노드가 이미 캐시한 KV 세그먼트(이전 턴의 시스템 프롬프트, 공통 맥락)를 append-prefill에서 직접 재사용하면 LycheeCluster의 로그 시간 계층 인덱스가 "어떤 세그먼트가 이번 append에 관련 있는가"를 빠르게 결정하는 검색 엔진으로 작동한다. PPD 라우팅 결정(append vs full prefill)을 LycheeCluster의 세그먼트 히트 확인과 결합하면 "히트 예상 시 D 노드 append, 미스 예상 시 P 노드 full prefill"하는 캐시-인식 PPD 라우터가 가능하다.
- Dual-Blade의 NVMe-Direct 오프로딩이 에지 환경의 Activity B 세그먼트 캐시에 직접 적용된다. 비연속 세그먼트를 NVMe LBA 영역에 직접 매핑하면 파일시스템 오버헤드 없이 세그먼트 단위 KV를 고속으로 로드·저장할 수 있어 에지 디바이스에서의 비연속 캐시 실용화 경로를 제공한다.

**B + C**
- SemantiCache의 의미 청킹 경계가 Stochastic KV Routing의 depth-wise 공유 범위를 제한하는 기준으로 활용 가능하다. 의미 경계 내에서는 레이어 간 KV 공유(낮은 압축 비율), 경계를 넘는 세그먼트 간에는 독립 KV 저장(높은 보존율)하는 "의미-인식 depth-wise 공유" 구조가 B+C 통합의 새 방향을 제시한다.
- LycheeCluster의 삼각부등식 기반 계층 인덱스를 SemantiCache의 의미 클러스터 코어에 적용하면, 의미 코어를 계층 인덱스 노드로 활용해 "의미 코어끼리의 삼각부등식으로 관련 없는 세그먼트를 단번에 제거"하는 고속 의미-보존 검색이 가능하다.
- SpecKV의 압축 수준-γ 동적 결합이 Activity B의 비연속 세그먼트 재사용률과 연동될 수 있다. 세그먼트 히트율이 높은(재사용 많은) 상황에서는 KV 압축을 더 공격적으로 적용해도 γ를 낮춰 정확도를 보상하는 "재사용-압축-γ 공동 최적화" 정책이 실현 가능하다.

**A + C**
- PrfaaS의 크로스-데이터센터 KV 전송에 eOptShrinkQ(이미 보고)의 2.2비트 압축을 결합하면 이더넷 전송 KV 크기가 추가로 ~60% 감소해 일반 이더넷의 대역폭 제약을 더욱 완화한다. MLA 모델의 낮은 KV 크기 + eOptShrinkQ 압축의 이중 효과로 크로스-데이터센터 KV 전송의 실용 가능성이 크게 향상된다.
- Dual-Blade의 에지 NVMe-Direct 오프로딩에 Stochastic KV Routing의 depth-wise 공유를 학습 단계에서 내재화하면, 에지 기기에서 저장해야 할 KV 레이어 수 자체가 줄어 Dual-Blade의 오프로딩 효율이 추가 향상된다.

**A + B + C**
- 통합 파이프라인 (멀티턴 에이전틱 + 에지 특화): PPD 라우팅(A) → LycheeCluster 세그먼트 히트 확인(B) → Stochastic KV Routing으로 학습된 depth-wise 압축 KV(C) → Dual-Blade NVMe 오프로딩(A+C). D 노드에서 PPD append-prefill 처리 시 LycheeCluster가 관련 세그먼트를 로그 시간에 검색하고, depth-wise 공유로 압축된 KV를 Dual-Blade가 NVMe에서 빠르게 로드하는 완전한 에지-친화 멀티턴 비연속 재사용 파이프라인.
- SemantiCache의 의미 코어 병합이 이 파이프라인에서 D 노드 로컬 캐시의 스토리지 풋프린트를 줄여 Dual-Blade의 NVMe 오프로딩이 필요한 KV 범위를 최소화하는 메타-압축 단계로 작동할 수 있다.

## idea-generator에게 전달할 포인트

### Activity A
- **PPD의 핵심 교훈**: "모든 프리필이 동등하지 않다"는 원칙이 Activity A의 스케줄러 설계에 직접 적용된다. 요청 유형(새 세션 vs 멀티턴 append)에 따라 P 노드 전달 vs D 노드 로컬 처리를 동적으로 선택하는 스케줄러가 KV 전송 병목을 근본적으로 해소한다. Activity B의 비연속 세그먼트 히트 예측과 결합하면 "세그먼트 히트 시 → D 노드 로컬 append, 미스 시 → P 노드 full prefill"하는 히트-인식 PPD 라우터가 가능하다.
- **PrfaaS의 설계 원칙**: 모델-측 KV 크기 감소(MLA/하이브리드 어텐션)가 시스템-측 크로스-데이터센터 전송의 실현 가능성을 결정한다. Activity A에서 KV 압축(Activity C)을 전제로 멀티-클러스터 라우팅 설계를 고려할 때 "압축 비율이 전송 임계값에 미치는 영향"을 핵심 변수로 모델링할 것.
- **Dual-Blade의 시스템 교훈**: NVMe-Direct 경로가 파일시스템 오버헤드를 제거하는 접근이 Activity A의 KV 오프로딩 스케줄러에서 "I/O 블록 단위 직접 접근"을 설계 원칙으로 채택하는 근거를 강화한다. ContiguousKV(이미 보고)의 I/O 블록 정렬 원칙 + Dual-Blade의 NVMe-Direct 접근이 Activity A+C의 오프로딩 파이프라인 설계에서 결합 가능하다.

### Activity B
- **LycheeCluster의 핵심 기여**: 삼각부등식 기반 재귀 계층 인덱스가 Activity B의 세그먼트 캐시 검색을 O(N) → O(log N)으로 단축한다. 현재 Radix 트리 캐시(src/cache/radix.py)에 삼각부등식 기반 서브트리 가지치기를 추가하면 대규모 세그먼트 캐시에서 검색 효율이 획기적으로 향상된다. DASH-KV(이미 보고)의 비대칭 딥 해싱이 O(N) 해시-기반 검색을 제공한 반면, LycheeCluster는 더 강건한 거리 상한을 보장한다.
- **CLC 실험 연구(2603.20218)**: 교차-청크 의존성 처리 전략의 보완 가능성이 Activity B의 비연속 세그먼트 조합에서 단일 전략보다 두 전략(예: 선택적 재계산 + 병렬 처리)을 조합하는 것이 더 효과적임을 제안한다. ProphetKV(쿼리-중심 재계산) + InfoFlowKV(청크 재순서화)의 조합이 이미 보고된 가장 유망한 보완 쌍이다.
- **SemantiCache의 의미 경계**: 청크 크기를 고정 토큰 수 대신 의미 경계(문장/절 구분자)로 결정하면 Activity B의 세그먼트 해시 경계 설계에서 청크 내 의미 완결성이 보장돼 세그먼트 재사용 정확도가 향상된다. GSC 알고리즘의 시드-기반 클러스터링을 세그먼트 내 토큰 중요도 순위화에 응용할 것.

### Activity C
- **Stochastic KV Routing(2604.22782) — 새로운 학습 기반 압축 방향**: 사후 압축(post-hoc compression)이 아닌 학습 중 내재화된 depth-wise KV 공유가 Activity C의 "정확도 보존" 제약을 훈련 시점에 보장하는 새로운 접근. 활성 추론에서는 추가 연산 없이 배포 시 하드웨어에 맞춰 공유 전략을 선택만 하면 된다. 파인튜닝 가능한 모델 환경에서는 1K 스텝으로 DMS(이미 보고)와 유사한 절차로 Stochastic KV Routing을 적용해 Activity C의 구현 대안으로 탐색할 것.
