## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [WiCER: Wiki-memory Compile, Evaluate, Refine Iterative Knowledge Compilation for LLM Wiki Systems](https://arxiv.org/abs/2605.07068)
- **출처**: arXiv 2605.07068 (2026-05-08, Juan M. Huerta 외)
- **핵심 아이디어**: "LLM Wiki 패턴"이라는 새로운 패러다임을 정의하고 검증한다. 도메인 지식(문서·사실)을 전처리하여 "Wiki"라는 영속적 아티팩트로 압축하고, 추론 시 이 Wiki를 KV 캐시로 직접 서빙해 RAG의 검색 실패와 레이턴시 없이 지식에 접근한다. 문제는 "컴파일 갭(compilation gap)": 원시 문서에서 Wiki로 증류할 때 중요한 사실이 소실되는 현상이다. WiCER(Wiki-memory Compile, Evaluate, Refine)는 CEGAR(Counterexample-Guided Abstraction Refinement) 알고리즘에서 영감을 받아 Wiki를 반복적으로 컴파일·평가·정제한다. 평가 단계에서 LLM이 현재 Wiki KV 캐시만으로 답하지 못하는 질문들을 카운터예제로 식별하고, 정제 단계에서 해당 정보를 Wiki에 보강하는 루프를 수렴까지 반복한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 전체 컨텍스트 KV 캐시 추론이 선별된 지식에서 RAG 대비 4.38 vs 4.08/5 점수 + 7.3× 빠른 TTFT를 달성함을 17개 도메인·6,800개 질문으로 최초 입증. 그러나 대규모에서는 어텐션 희석(attention dilution)으로 RAG 아래로 떨어지고, 맹목적 컴파일은 53~60% 파국 실패율을 보인다는 문제를 WiCER가 해결한다. Activity B의 "장기 기억(long-term memory) 지원" 목표에서 "도메인 지식을 비연속 KV 세그먼트 집합으로 압축·정제 후 영속화"하는 구체적 방법론을 제시한다. Persistent Q4 KV Cache(이미 보고)의 에이전트별 KV 영속화가 단일 에이전트의 컨텍스트 저장이라면, WiCER는 도메인 지식 전체를 KV 캐시로 증류·보관하는 조직화된 지식 저장소 방향이다. KV Packet(이미 보고)의 맥락 독립 캐시 + WiCER의 반복 컴파일 정제 = 비연속 세그먼트 정확도 보장 프레임워크로 결합 가능하다.

#### 2. [LatentRAG: Latent Reasoning and Retrieval for Efficient Agentic RAG](https://arxiv.org/abs/2605.06285)
- **출처**: arXiv 2605.06285 (2026-05-07)
- **핵심 아이디어**: 기존 에이전틱 RAG가 중간 추론(thoughts)과 서브쿼리를 토큰-by-토큰 자기회귀 방식으로 생성해 대규모 지연을 유발하는 문제를 해결한다. LatentRAG는 추론과 검색을 이산 언어 공간에서 연속 잠재 공간(continuous latent space)으로 이동시킨다. 단일 순전파(single forward pass)로 숨겨진 상태에서 직접 잠재 토큰을 생성해 서브쿼리와 추론 중간 결과를 표현한다. LLM과 밀집 검색 모델(dense retrieval model)을 잠재 공간에서 정렬해 잠재 서브쿼리 토큰에 대한 검색을 지원하며, 엔드-투-엔드 공동 최적화가 가능하다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: 7개 벤치마크에서 명시적 에이전틱 RAG 방법과 동등한 성능을 유지하면서 추론 지연을 약 90% 단축, 단순 단계 RAG와의 지연 차이를 크게 좁힘. Activity B에서 비연속 KV 세그먼트 검색 자체를 잠재 공간으로 이동시키면 검색 오버헤드를 크리티컬 패스에서 완전히 제거하는 방향으로 이어진다. FreeKV(이미 보고)의 투기적 검색(검색을 크리티컬 패스 밖으로 이동)과 LatentRAG의 잠재 공간 검색(검색 자체를 단일 패스로 압축)이 상호보완적으로 비연속 KV 검색 지연을 최소화하는 경로를 제공한다.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Segmented Hash / Radix | LycheeCluster 삼각부등식 기반 계층 인덱스, O(N)→로그 시간 검색 | 변화 없음 — 이번 주 신규 세그먼트 해시/Radix 논문 없음 |
| CacheBlend / Blending | KV Packet 소프트-토큰 어댑터로 맥락 의존성 자체 제거, 재계산 제로 패러다임 | WiCER(2605.07068)가 반복 컴파일·정제로 도메인 지식을 KV 캐시 아티팩트로 보장하는 새로운 B 방향 추가; 기존 "즉각 재사용"을 넘어 "정제된 지식 KV 영속화" 패러다임 |
| Position-independent caching | Persistent Q4 KV Cache 에이전트 KV 4비트 디스크 영속화 | LatentRAG(2605.06285)가 검색 서브쿼리를 잠재 공간으로 이동시켜 RAG 기반 비연속 검색의 지연 병목을 90% 단축; 위치-독립 세그먼트 검색의 새로운 공간을 잠재 표현으로 확장 |

### 새로운 논문·구현체

#### 1. [Reformulating KV Cache Eviction Problem for Long-Context LLM Inference (LaProx)](https://arxiv.org/abs/2605.07234)
- **출처**: arXiv 2605.07234 (2026-05-08, Tho Mai & Joo-Young Kim — KAIST)
- **핵심 아이디어**: 기존 KV 퇴거 방법들이 헤드별 어텐션 가중치 평균에만 의존해 값 표현(value representation)·출력 투영(output projection)·헤드 간 의존성을 무시하는 근본 문제를 해결한다. KV 퇴거 문제를 "어텐션 맵과 투영된 값 상태 간의 곱 상호작용을 명시적으로 모델링하는 출력-인식(output-aware) 레이어 단위 행렬곱 근사 문제"로 재형식화한다. LaProx(Layer-wise Proximal KV Eviction)는 헤드별 로컬 결정 대신 레이어 전체에서 토큰을 글로벌 비교하는 통합 중요도 점수를 할당한다. 전통적 헤드별 독립 퇴거에서 레이어-와이즈 공동 최적화로의 전환이 핵심 기여다.
- **연구 목표와의 관련성**: high
- **주목할 점**: LongBench의 19개 데이터셋과 NIAH 벤치마크에서 기존 SOTA를 전 구성에서 일관되게 초과. 메모리 예산이 작아질수록(극단적 압축) LaProx와 기존 방법 간 성능 차이가 커져 고압축 환경에서 강건성이 높다. 단 5% KV 캐시 유지로 모델 성능 유지를 실험으로 확인. Activity C의 "정확도 ±1% 이내" 제약 하에서 "어텐션 맵 × 값 투영의 레이어 단위 공동 최적화"가 헤드별 독립 퇴거보다 근본적으로 우월함을 이론적으로 설명하는 첫 번째 논문. ManifoldKV(이미 보고)의 유클리드 아웃라이어 탐지가 스케일 정보를 통합했다면, LaProx는 출력 투영의 곱(product) 구조를 통합하는 더 완전한 근사를 제공한다.

#### 2. [LKV: End-to-End Learning of Head-wise Budgets and Token Selection for LLM KV Cache Eviction](https://arxiv.org/abs/2605.06676)
- **출처**: arXiv 2605.06676 (2026-05, Enshuai Zhou 외 10인 — ICT/CAS)
- **핵심 아이디어**: 기존 KV 캐시 압축 패러다임의 두 가지 근본 한계를 동시에 해결한다. (1) 휴리스틱 예산 배분: 통계적 사전지식(attention sink, 초기 토큰 우선 등)에 의존해 태스크 목표와 무관한 헤드별 자원 배분. (2) 휴리스틱 토큰 선택: 쿼리-키 상호작용 또는 정적 귀납적 편향에 의존해 글로벌 최적에서 이탈. LKV-H(글로벌 학습 예산 배분): 헤드 구조를 평탄화해 모든 헤드가 전체 예산 내에서 경쟁하도록 하고 학습 가능 임베딩으로 태스크-최적화된 글로벌 예산을 학습. LKV-T(내재적 KV 중요도 도출): 어텐션 행렬을 구체화하지 않고도 KV 중요도를 유도하는 학습 기반 선택기. 두 모듈을 결합한 엔드-투-엔드 미분 가능 KV 압축 프레임워크가 핵심.
- **연구 목표와의 관련성**: high
- **주목할 점**: LongBench와 RULER 벤치마크에서 높은 압축률에서 SOTA 달성. 15% KV 캐시 유지로 near-lossless 성능 달성. 기존 H2O, SnapKV 등이 어텐션 통계 기반 휴리스틱인 반면, LKV는 태스크 목표를 직접 학습하는 첫 번째 엔드-투-엔드 프레임워크. GVote(이미 보고)가 미래 쿼리 투표로 적응 예산을 결정했다면, LKV는 예산과 선택을 공동 학습해 태스크 목표에 더 직접 정렬한다. Activity C의 "정확도 ±1% 이내" 제약을 달성하는 데 학습 기반 예산 배분이 휴리스틱 대비 근본적으로 유리하다는 실증을 제공한다.

#### 3. [RateQuant: Optimal Mixed-Precision KV Cache Quantization via Rate-Distortion Theory](https://arxiv.org/abs/2605.06675)
- **출처**: arXiv 2605.06675 (2026-05, Fei Zuo 외 — BA TechWorks/BMW Group, NUS, Tsinghua)
- **핵심 아이디어**: 현재 KV 캐시 혼합-정밀도 양자화 방법들이 모든 어텐션 헤드에 동일한 비트폭을 할당하거나, 헤드별 왜곡 곡선(distortion curve)이 양자화 방식에 따라 다름을 무시해(왜곡 모델 불일치, distortion model mismatch) 최적에서 멀어지는 문제를 해결한다. 소규모 보정 세트에서 각 양자화기(quantizer)의 왜곡 모델을 피팅하고, 율-왜곡 이론의 역 워터필링(reverse waterfilling)으로 비트 할당 문제를 닫힌 형태(closed form)로 풀어 헤드별 최적 비트폭을 결정한다. 양자화 방식(KIVI, QuaRot 등)을 불문하고 일반적으로 적용 가능한 최적화 레이어로 작동한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Qwen3-8B에서 평균 2.5비트로 KIVI의 퍼플렉시티를 49.3에서 14.9로 70% 감소, QuaRot 대비 6.6 PPL 개선. 보정(calibration)은 단일 GPU에서 1.6초 소요, 추론 시 추가 오버헤드 없음. Activity C의 "정확도 ±1% 이내" 제약과 "메모리 효율" 목표를 수학적으로 최적인 비트 할당으로 달성하는 새로운 접근. eOptShrinkQ(이미 보고)의 랜덤 행렬 이론 기반 자동 랭크 선택이 구조적(structural) 최적화라면, RateQuant는 비트 할당(quantization budget) 최적화로 상호 보완적이다. 두 방법의 결합 — eOptShrinkQ로 저랭크 공유 성분 추출 후 RateQuant 최적 비트 할당 — 이 Activity C의 최적 2단계 압축 파이프라인 후보가 된다.

#### 4. [RetentiveKV: State-Space Memory for Uncertainty-Aware Multimodal KV Cache Eviction](https://arxiv.org/abs/2605.04075)
- **출처**: arXiv 2605.04075 (2026-05, Sihao Liu 외)
- **핵심 아이디어**: 멀티모달 LLM에서 시각 토큰이 긴 컨텍스트를 생성할 때 두 가지 독특한 문제가 발생한다: (1) "지연 중요성(deferred importance)" — 처음에는 어텐션이 낮지만 이후 디코딩에서 중요해지는 시각 토큰이 조기 퇴거되는 문제. (2) 이산적 프루닝으로 시각 단서의 공간 연속성이 파괴되는 문제. RetentiveKV는 KV 퇴거를 "이산 컨텍스트 절단"에서 "연속 기억 진화(continuous memory evolution)"로 재정의한다. 정보 엔트로피를 활용해 저어텐션 토큰의 정보 잠재력을 정량화하고, 퇴거 예정 토큰을 엔트로피-가이드 상태 전이(entropy-guided state transition)를 통해 연속 상태 공간(SSM)에 통합한다. 이후 디코딩에서 의미 관련성이 발생하면 동적으로 재활성화(reactivation)한다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: 5.0× KV 캐시 압축 + 1.5× 디코딩 가속 달성. 기존 멀티모달 퇴거 방법(WindowQuant, HybridKV — 이미 보고)이 어텐션 통계 기반인 반면, RetentiveKV는 상태 공간 모델(SSM)로 "퇴거된" 정보를 연속 표현으로 유지하는 근본적으로 다른 접근. Activity C의 "정확도 보존" 제약에서 정보를 완전히 버리는 대신 압축된 상태로 유지하는 설계 원칙이 텍스트 전용 LLM에도 전용 가능하다. Activity B의 비연속 세그먼트 재사용과 결합 시, 세그먼트가 당장 필요 없더라도 SSM 상태로 유지해 나중에 재활성화하는 "비연속 KV 잠재 보유" 방향이 가능하다.

#### 5. [Make Your LVLM KV Cache More Lightweight (LightKV)](https://arxiv.org/abs/2605.00789)
- **출처**: arXiv 2605.00789 / OpenReview (2026-05-01, Xihao Chen, Yangyang Guo, Roger Zimmermann)
- **핵심 아이디어**: 대형 비전-언어 모델(LVLM)에서 대량의 시각 토큰이 KV 캐시를 급증시키는 문제를 해결한다. 텍스트 프롬프트로 가이드된 교차-모달리티 메시지 패싱(cross-modality message passing)으로 시각 토큰 간 중복 정보를 집계하고, 프리필 단계에서 시각 토큰 KV 캐시를 점진적으로 압축한다. 순수 시각 중심 압축(vision-only compression)과 달리 텍스트 프롬프트가 어떤 시각 정보가 관련 있는지를 가이드하는 프롬프트-인식(prompt-aware) 압축이 핵심 차별점이다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: 8개 오픈소스 LVLM × 8개 공개 벤치마크(MME, SeedBench 등) 평가. 원본 시각 토큰의 55%만으로 시각 토큰 KV 캐시 크기 절반 감소, 연산 최대 40% 감소, 일반 성능 유지. 기존 WindowQuant(이미 보고)가 윈도우 단위 혼합-정밀도로 시각 토큰을 처리한 반면, LightKV는 텍스트 가이드 교차-모달 메시지 패싱으로 시각-텍스트 정렬 기반 압축을 실현. Activity B의 비연속 세그먼트 중요도 평가에서 "현재 쿼리(텍스트) + 세그먼트 내용(시각 또는 도구 출력) 교차 메시지 패싱"으로 세그먼트 중요도를 동적으로 결정하는 방향으로 전용 가능.

## Activity 간 시너지 포인트

**A + B**
- WiCER의 CEGAR 기반 반복 컴파일-평가-정제 루프가 Activity A의 스케줄러 설계에도 응용 가능하다. 스케줄러가 "현재 캐시 세그먼트로 요청을 얼마나 잘 서빙하는가"를 평가하고, 히트율이 낮은 세그먼트를 재컴파일(재캐싱)하는 "CEGAR 방식 캐시 정제 스케줄러" 개념이 도출된다. 반복 정제로 세그먼트 커버리지를 점진적으로 높이는 방향이 Activity A의 "캐시 퇴거율 최소화" 목표와 직결된다.
- LatentRAG의 잠재 공간 검색을 Activity A의 스케줄러가 인식하면, 요청 도착 시 잠재 서브쿼리 벡터로 비연속 세그먼트 히트 확률을 사전 추정하고 이를 라우팅 결정에 반영하는 "잠재-히트 예측 스케줄러"가 가능하다.

**B + C**
- WiCER의 KV 캐시 Wiki 아티팩트에 RateQuant의 헤드별 최적 비트 할당을 적용하면, 자주 접근되는 핵심 지식 헤드는 높은 비트폭으로 보존하고 부수적 정보 헤드는 낮은 비트폭으로 압축하는 "질의 빈도 기반 헤드 선택적 Wiki 양자화"가 가능하다. WiCER가 "무엇을 저장할 것인가"를 결정하고 RateQuant가 "어떻게 압축할 것인가"를 결정하는 역할 분리.
- RetentiveKV의 SSM 기반 연속 기억이 Activity B의 비연속 세그먼트 재사용과 결합하면, 당장 사용되지 않는 비연속 세그먼트를 완전 퇴거하는 대신 SSM 상태로 압축 유지하고 나중에 재활성화하는 "비연속 세그먼트 잠재 보유(latent retention)" 전략이 가능하다. 세그먼트 영속화(Persistent Q4 KV Cache, 이미 보고)의 디스크 I/O 없이 GPU 내 압축 보유하는 대안.
- LKV의 엔드-투-엔드 학습 기반 예산 배분을 Activity B의 세그먼트별 중요도 점수에 적용하면, 비연속 세그먼트 조합에서 각 세그먼트에 할당할 최적 KV 예산을 태스크 목표로 학습하는 "세그먼트-레벨 LKV" 변형이 가능하다.

**A + C**
- RateQuant의 헤드별 최적 비트 할당이 Activity A의 선점형 KV 관리(Valve, TokenFlow)와 결합하면, 메모리 압력이 높아 선점이 필요할 때 RateQuant가 실시간으로 헤드별 비트를 낮춰 압축하는 "선점-최적 비트 압축(preemptive rate-optimal compression)" 파이프라인이 가능하다. 선점 오버헤드를 최소화하면서 KV 메모리를 동적으로 축소한다.
- LaProx의 레이어 단위 출력-인식 중요도 점수가 Activity A의 배치 스케줄러에서 "어떤 요청의 KV를 복원할 것인가" 결정에 사용되면, 레이어별 출력 기여가 높은 KV를 우선 복원하는 "출력-인식 CacheFlow 스케줄러" 변형이 가능하다.

**A + B + C**
- 통합 파이프라인 (장기 지식 기반 RAG 서빙): WiCER 반복 컴파일 Wiki KV(B) → RateQuant 헤드별 최적 비트 압축(C) → LatentRAG 잠재 공간 검색(B) → Activity A 스케줄러 잠재-히트 예측 라우팅(A) → LaProx/LKV 기반 실시간 퇴거 관리(C). 도메인 지식이 WiCER로 정제된 Wiki KV로 압축 저장되고, LatentRAG가 잠재 공간에서 빠르게 관련 세그먼트를 검색하며, RateQuant가 메모리 예산에 맞게 비트를 최적 배분하는 완전한 지식-기반 비연속 재사용 파이프라인.
- RetentiveKV의 SSM 상태 유지가 이 파이프라인에서 WiCER Wiki KV에서 퇴거된 정보를 SSM으로 유지해 LatentRAG 검색 시 재활성화하는 "3-티어 기억 관리(즉시 접근: GPU KV / 압축 보유: SSM 상태 / 영속 저장: 디스크 Q4)" 아키텍처로 확장될 수 있다.
