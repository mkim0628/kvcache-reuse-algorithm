# 트렌드 리포트 — 2026-05-14

## 전체 요약

1. **Activity A (스케줄링)**: 이번 주 Activity A에서 주목할 새 논문은 제한적이다. CacheSolidarity(2603.10726)가 멀티테넌트 환경에서 Automatic Prefix Caching의 타이밍 사이드채널 문제를 최초로 체계적으로 해결하는 시스템을 제시했다. 선택적 프리픽스 격리로 기존 방어 기법 대비 캐시 재사용률을 70% 높이고 추론 지연을 30% 낮춘다. 멀티테넌트 KV 재사용이 서빙 인프라의 보안 문제와 맞닿는 새로운 연구 전선이다.

2. **Activity B (비연속 재사용)**: 이번 주 순수하게 Activity B에 해당하는 새로운 논문은 확인되지 않았다. 직전 주까지 보고된 KV-Fold(foldl 청크 재귀), AdapShot(RoPE 위치 재인코딩), Louver(제로 거짓 음성 KV 검색)가 현재 최전선이다.

3. **Activity C (압축)**: 이번 주 가장 활발한 진전이 Activity C에서 나타났다. FibQuant(2605.11478)는 구형-베타 소스에 매칭된 방사상-각도 코드북으로 기존 벡터 양자화 방법의 기하 구조 손실 문제를 해결하고 10× 이상 압축에서 유일한 범용 랜덤-접근 방법임을 실증한다. AB-Sparse(2605.12110)는 어텐션 헤드별 블록 크기 민감도가 다름을 발견해 보정-기반 헤드별 적응 블록 크기를 할당하고, 커스텀 GPU 커널로 5.43% 정확도 향상과 처리량 보존을 동시에 달성한다. Fluxion(2605.07719)는 CPU-상주 KV 캐시에 대한 하이브리드 스파스 어텐션에서 출력-인식 KV 예산 할당과 헤드별 희소성 구성으로 최강 기준선 대비 1.5~3.7× 속도향상을 달성한다.

4. **A+C 시너지**: Fluxion의 CPU-GPU 하이브리드 스파스 어텐션이 Activity A의 KV 오프로딩 스케줄러와 직접 연결된다. CPU에 오프로딩된 KV에 Fluxion이 출력-인식 예산 할당으로 필요한 KV만 GPU에 가져오는 "오프로딩-인식 스파스 어텐션 스케줄러"가 가능하다.

5. **B+C 시너지**: FibQuant의 방사상-각도 코드북이 Activity B의 비연속 세그먼트에 적용되면, 동일한 코드북을 공유하는 랜덤-접근 방식으로 임의 세그먼트를 독립적으로 압축·복원할 수 있어 비연속 재사용과 벡터 양자화 압축의 통합이 용이해진다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [CacheSolidarity: Preventing Prefix Caching Side Channels in Multi-tenant LLM Serving Systems](https://arxiv.org/abs/2603.10726)
- **출처**: arXiv 2603.10726 (2026-03-11)
- **핵심 아이디어**: 멀티테넌트 LLM 서빙에서 Automatic Prefix Caching(APC)이 캐시 히트 시 응답이 빠르고 미스 시 느리다는 타이밍 차이를 유발해 공격자가 다른 사용자의 요청 내용을 추론할 수 있는 "APC 사이드채널(side channel)"을 체계적으로 해결한다. 기존 방어 기법들이 사용자 전체를 격리해 캐시 재사용 자체를 차단하는 반면, CacheSolidarity는 캐시 재사용 패턴을 지속 모니터링해 여러 사용자 간 의심스럽게 공유되는 프리픽스만 선택적으로 격리한다. 핵심 통찰: 공격을 방지하는 데 전체 사용자 격리가 아닌 "공격 유발 가능 공유 프리픽스만 보호"하는 것으로 충분하다는 점이다. 경량 모니터링이 공유 프리픽스에 플래그를 달고 후속 요청에서 해당 프리픽스의 재사용만 제한한다. 전체 캐시 구조와 비-공격적 공유는 그대로 보존된다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: 기존 격리 방어 기법 대비 최대 70% 높은 캐시 재사용률과 30% 낮은 추론 지연 달성. Activity A의 "캐시 퇴거율 최소화" 및 "스케줄링 오버헤드 TTFT p50 +5% 이내" 목표와 직접 연결된다 — 보안 제약 하에서도 캐시 재사용 효율을 최대한 보존하는 접근이 멀티테넌트 프로덕션 환경에서 필수. KV 캐시 재사용이 확산되면서 보안(프라이버시 사이드채널)과 효율(캐시 히트율) 사이의 트레이드오프가 새로운 Activity A 설계 제약으로 등장했다는 것을 처음 형식화한 논문. Louver(이미 보고)의 제로 거짓 음성 KV 검색이 세그먼트 중요도 기반 선택이라면, CacheSolidarity는 멀티테넌트 환경에서 어떤 프리픽스를 공유 허용하고 어떤 것을 격리할 것인지의 정책 결정 문제를 다룬다. DualMap(이미 보고)의 캐시-친화성 라우팅과 결합 시 "보안 플래그가 없는 프리픽스는 캐시-친화적 라우팅, 플래그된 프리픽스는 격리 라우팅"하는 보안-인식 캐시 라우터 설계가 가능하다.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [FibQuant: Universal Vector Quantization for Random-Access KV-Cache Compression](https://arxiv.org/abs/2605.11478)
- **출처**: arXiv 2605.11478 (2026-05-12, Namyoon Lee, Yongjune Kim — POSTECH)
- **핵심 아이디어**: 기존 회전 기반 스칼라 양자화(VQKV 등)가 정규화 단계에서 형성된 구형 기하 구조를 무시하고 좌표를 개별 처리해 고압축률에서 품질이 급격히 저하되는 문제를 해결한다. FibQuant는 정규화-회전-저장 인터페이스를 유지하면서 스칼라 코드북을 **구형-베타(Spherical-Beta) 소스에 매칭된 방사상-각도(radial-angular) 공유 코드북**으로 대체한다. 세 가지 설계 혁신: (1) **베타-분위수 반경(Beta-quantile radii)** — 구형 정규화 후 각 벡터 블록의 반경 분포가 베타 분포를 따른다는 사실을 활용해 반경 양자화 격자를 수학적으로 최적화, (2) **피보나치/Roberts-Kronecker 준균일 방향(quasi-uniform directions)** — 고차원 구면에 균일하게 분산된 방향 집합을 피보나치 수열과 Roberts-Kronecker 격자로 구성해 방향 양자화 오류를 최소화, (3) **다중-재시작 Lloyd-Max 정제** — 보정 데이터로 코드북을 반복 정제해 실제 KV 분포에 적응. 결과적으로 단일 공유 코드북이 모든 레이어·헤드에 랜덤-접근 방식으로 적용 가능하다.
- **연구 목표와의 관련성**: high
- **주목할 점**: GPT-2 small KV 캐시에서 5× 압축에서 어텐션 출력 코사인 유사도 0.99, 34× 압축에서 0.95 달성. TinyLlama-1.1B에서 4× 압축 시 fp16 대비 퍼플렉시티 차이 0.10 이내. **10× 이상 압축에서 유일한 범용 랜덤-접근 VQ 방법**임을 실험으로 확인. Activity C의 "메모리 −30% 이상" 목표를 5×~34× 압축으로 대폭 초과하면서 "정확도 ±1% 이내" 제약을 만족. 이미 보고된 VQKV(훈련-프리 VQ, 82.8% 압축 + 98.6% 성능)가 코드북 학습에 pre-RoPE 공간 활용에 집중한 반면, FibQuant는 정규화 후 구형 분포를 직접 활용하는 수학적으로 더 근본적인 접근. **랜덤-접근(random-access)** 특성이 Activity B의 비연속 세그먼트 저장에서 임의 세그먼트를 독립적으로 압축·복원하는 사용 패턴과 완벽히 부합한다. RateQuant(이미 보고)의 헤드별 최적 비트 할당과 결합 시 "RateQuant가 FibQuant 코드북의 최적 비트 수를 헤드별로 결정, FibQuant가 해당 비트 수의 구형-베타 코드북으로 압축"하는 수학적으로 최적화된 VQ 파이프라인이 가능하다.

#### 2. [AB-Sparse: Sparse Attention with Adaptive Block Size for Accurate and Efficient Long-Context Inference](https://arxiv.org/abs/2605.12110)
- **출처**: arXiv 2605.12110 (2026-05-12, Di Liu, Ruitian Wang 외 — Shanghai Jiao Tong University / Ant Group)
- **핵심 아이디어**: 기존 블록-스파스 어텐션(block-sparse attention)이 모든 어텐션 헤드에 동일한 블록 크기를 적용하는 균일성 가정이 잘못됐음을 발견한다. 헤드마다 블록 크기 민감도(block size sensitivity)가 현저히 다르므로 균일한 블록 크기가 정확도를 희생한다. AB-Sparse는 세 가지 통합 접근으로 이를 해결한다: (1) **보정-기반 헤드별 블록 크기 프로파일링(calibration-driven profiling)** — 헤드별 블록 크기 민감도가 다양한 입력에 걸쳐 안정적이라는 관찰을 활용해, 소규모 보정 데이터로 각 헤드의 최적 블록 크기를 배포 전 1회 결정. (2) **중심 양자화(centroid quantization)** — 블록 중심(centroid)이 실제 어텐션 계산이 아닌 랭킹에만 사용되므로 정밀도에 민감하지 않다는 사실을 활용해 메모리 발자국을 무손실로 감소. (3) **적응형 블록 크기 지원 커스텀 GPU 커널** — 가변-길이 배치 실행을 위한 인덱싱 메커니즘과 표준 paged KV 캐시와 호환되는 페이지 매핑 메커니즘 구현.
- **연구 목표와의 관련성**: high
- **주목할 점**: 기존 블록-스파스 어텐션 기준선 대비 최대 **5.43% 정확도 향상** + 처리량 오버헤드 없음. Activity C의 "정확도 ±1% 이내" 제약을 스파스 어텐션 방법에서 달성하는 새로운 기준. Louver(이미 보고, Activity B)가 제로 거짓 음성을 이론으로 보장한다면, AB-Sparse는 헤드별 적응적 블록 크기로 실용적인 정확도-속도 균형을 달성하는 상보적 접근 — 두 방법은 "어떤 KV를 선택할 것인가(Louver)" + "어떤 블록 크기로 검색할 것인가(AB-Sparse)"를 각각 해결한다. MISA(이미 보고)의 MoE 인덱서 라우터가 인덱서 헤드 계산량을 줄인다면, AB-Sparse는 헤드별 블록 크기 적응으로 인덱서 정확도를 높이는 상보적 방향 — 두 기법의 결합이 "저비용 + 고정확도" 스파스 어텐션 인덱서를 가능하게 한다. paged KV 캐시와의 호환성이 vLLM 이식(6단계)에서 직접 활용 가능한 중요한 구현 장점.

#### 3. [An Efficient Hybrid Sparse Attention with CPU-GPU Parallelism for Long-Context Inference (Fluxion)](https://arxiv.org/abs/2605.07719)
- **출처**: arXiv 2605.07719 (2026-05-08, Feiyu Yao, Zhixiong Niu 외 — 여러 소속)
- **핵심 아이디어**: CPU-상주(CPU-resident) KV 캐시에 대한 스파스 어텐션을 수행할 때, GPU-전용 설계는 PCIe 대역폭과 메타데이터 메모리 병목에 갇히고, CPU-GPU 하이브리드 설계는 GPU 유휴 시간과 CPU 측 상위-k 선택·스파스 어텐션 계산 병목을 겪는 문제를 해결한다. Fluxion은 세 가지 핵심 통찰로 이를 극복한다: (1) **출력-인식 KV 예산 할당(output-aware KV budgeting)** — 어텐션 점수만으로 중요도를 판단하면 실제 출력에 미치는 영향을 잘못 추정하므로, 헤드별 출력 편차와 헤드의 상대적 기여도를 함께 고려해 KV 예산을 배분함으로써 CPU 측 불필요한 스파스 어텐션 계산을 감소, (2) **헤드별·세분성-인식 희소성 구성(head-specific & granularity-aware sparse config)** — 어텐션 헤드마다 검색 수요가 다르고 top-k 세분성이 CPU 측 오버헤드와 강하게 결합돼 있음을 활용해 헤드별 최적 (예산, 블록 크기) 쌍을 보정으로 결정, (3) **크로스-디바이스 조율 실행(cross-device coordinated execution)** — CPU와 GPU가 독립적으로 KV 로드와 스파스 어텐션 계산을 병렬 실행해 GPU 유휴 시간을 70.65% → 45.78%로 대폭 감소.
- **연구 목표와의 관련성**: high
- **주목할 점**: 2개 모델, 3개 벤치마크, 40개 태스크에서 최강 고정-스파스 하이브리드 기준선(KV 예산 0.05) 대비 **1.5~3.7× 속도향상**, 평균 품질 저하 -0.26에 불과. 32K 컨텍스트 Qwen 환경에서 GPU 유휴율 70.65% → 45.78% 감소. CPU-상주 KV에 대한 스파스 어텐션이 단순 토큰 퇴거·양자화를 넘어 Activity C 압축의 새로운 시스템 차원 방향을 제시한다. SparKV(이미 보고)가 클라우드-에지 하이브리드 KV 로딩에서 "스트리밍 vs 로컬 계산" 선택을 다룬다면, Fluxion은 CPU에 이미 오프로딩된 KV에 대한 효율적인 스파스 접근을 다루는 상보적 문제를 해결한다. Tutti(이미 보고)의 GPU-centric KV 오브젝트 스토어가 SSD→GPU 복원을 최적화한다면, Fluxion은 CPU→GPU 경로에서 필요한 KV만 선택적으로 가져오는 반대 방향의 최적화 — 두 시스템을 결합하면 "SSD에서 CPU로(Tutti GPU-centric 대신 CPU 상주) → CPU에서 GPU로(Fluxion 스파스 선택)"하는 3-계층 KV 접근 파이프라인이 완성된다. Activity A의 KV 오프로딩 스케줄러와 Fluxion을 통합하면, 오프로딩된 KV를 전량 복원하지 않고 스파스하게 접근하는 "오프로딩-인식 스파스 복원 스케줄러"가 Activity A+C 통합의 실용 방향이다.

## Activity 간 시너지 포인트

**A + C**
- Fluxion(C)의 출력-인식 KV 예산 할당이 Activity A의 KV 오프로딩 스케줄러와 결합되면, 스케줄러가 요청별로 CPU에 오프로딩할 KV 양을 결정할 때 Fluxion이 실제 어텐션 계산 시 그 중에서 다시 필요한 KV만 스파스 선택하는 "2단계 KV 필터링 스케줄러"가 가능하다. Flow-Controlled Scheduling(이미 보고)의 활성화 예산 이론과 결합하면 "활성화 예산 → CPU 오프로딩 비율 결정 → Fluxion 출력-인식 스파스 복원"의 이론-보장 통합 파이프라인이 설계 가능하다.
- CacheSolidarity(A)의 선택적 프리픽스 격리와 Fluxion(C)의 헤드별 스파스 KV 선택을 결합하면, 격리되지 않은 공유 프리픽스 KV는 Fluxion으로 스파스 접근하고 격리된 KV는 전량 복원하는 "보안-인식 스파스 복원 정책"이 가능하다.

**A + B**
- CacheSolidarity(A)의 공유-격리 판단 메커니즘이 Activity B의 비연속 세그먼트 재사용 정책에 통합되면, 보안 플래그가 없는 세그먼트는 크로스-요청 비연속 재사용 허용, 플래그된 세그먼트는 사용자별 독립 캐싱하는 "보안-인식 세그먼트 재사용 정책"이 멀티테넌트 환경에서 Activity B의 실용 배포 경로가 된다.

**B + C**
- FibQuant(C)의 랜덤-접근 특성이 Activity B의 비연속 세그먼트 저장과 이상적으로 부합한다. 각 비연속 세그먼트를 FibQuant로 독립 압축하고, 재사용 시 필요한 세그먼트만 선택적으로 복원하는 "FibQuant-압축 비연속 세그먼트 캐시"가 Activity B의 메모리 효율을 Activity C 수준(4~10× 압축)으로 향상시키면서 랜덤-접근 복원의 오버헤드를 최소화한다.
- AB-Sparse(C)의 헤드별 블록 크기 보정이 Activity B에서 비연속 세그먼트 검색 정확도를 헤드별로 최적화하는 기반이 된다. Louver(이미 보고, B)의 제로 거짓 음성 보장과 AB-Sparse의 헤드별 최적 블록 크기를 결합하면 "헤드별로 블록 크기를 조절해 Louver 검색 범위를 최적화"하는 이중 최적화가 가능하다.

**A + B + C**
- 통합 파이프라인 (멀티테넌트 장문 컨텍스트 비연속 재사용): CacheSolidarity 보안-인식 프리픽스 관리(A) → 비격리 세그먼트 비연속 재사용(B) → FibQuant 랜덤-접근 VQ 압축(C) → Fluxion 출력-인식 CPU-GPU 스파스 복원(C) → AB-Sparse 헤드별 블록 크기 최적 선택(C). 멀티테넌트 환경에서 CacheSolidarity가 안전하게 공유할 수 있는 세그먼트를 결정하고, 공유 허용 세그먼트는 FibQuant로 압축 저장되며, 재사용 시 Fluxion이 CPU에서 스파스하게 복원하고 AB-Sparse가 헤드별 최적 블록 크기로 어텐션을 계산하는 완전한 보안-효율 통합 파이프라인.
